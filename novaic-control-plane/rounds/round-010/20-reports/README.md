# Reports

Each team submits one report file with:
- implemented work
- exact commands + pass/fail summary
- artifact/doc paths
- acceptance mapping
- risks/blockers
- decision needed (if any): issue/options/recommendation/impact/owner/target_round
- self status
- canonical `repo_url`: `https://github.com/<org>/<repo>` only
- full 40-char `commit_sha`

Hard rule:
- understanding-only updates are invalid
- no evidence => cannot mark DONE
- local-only repo addressing (`file:///`, `local:`) is invalid
