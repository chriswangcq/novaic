# Reports

Each team submits one report file with:
- problem fixed (what failed in previous round)
- solution applied (what was changed)
- target state proof (what is now true)
- task result
- exact command + expected marker
- canonical `repo_url`
- `commit_sha` + `migrated_paths`
- replay output for non-authors
- artifact/doc paths

Hard rule:
- no template placeholders
- no ambiguous `repo_url`
