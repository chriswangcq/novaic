# QMP：击穿 Linux 黑盒的宿主通信管线

> 路径参考：各种底仓设备唤醒或 VNC 操作截取控制面。

## 1. 为什么不用网络 SSH 或者 API 通往系统？
设想我们要控制一个虚拟机里面的 Linux 去跑一波被认为是破坏级别的测试甚至由于它的应用崩溃网卡驱动都不再响应了！
如果你走局域网 SSH 控制——它由于假死或脱网已经断接！VMuse 控制组不可能像残废一样失去联络。我们需要走到“拔它插头机房人员”的权限高度。

## 2. QEMU Machine Protocol (QMP) 套接字操控
这就是整个项目能够提供无赖级操作的神器：它并未直接与处于虚拟机“内部”那挂逼的各种软件发 HTTP；它是直奔那个包绕着这个虚拟机的宇宙墙壁：`QEMU` 母体本身搭话。

这基于由于启动它时它在操作系统层面挂出的 `Unix Domain Socket`，这种 Socket 利用极其底层的数据总线暴露了这台电脑一切模拟硬件的能力：
- **外挂热插入（Hardware Insertion）**：即使里面网断了，我们可以直接热查虚拟设备挂进去；
- **强制抓屏获取视觉 (Screendump via VNC)**：通过发送强信号的拉屏 dump 获取内存，直接拿到显存里的真实验报帧画用于检查其死锁情况。
- **拔插头重启重启 (Hard Power Reset)**：指令直接关停它的进程宿主达到毫无讲价与依赖的硬件级操控。

这个大招正是 Gateway 及各种外部动作管用这层将死去的机器在开机预热前置入监控，并通过它的监控获取生命周期待命成功的完美证明途径闭环！
