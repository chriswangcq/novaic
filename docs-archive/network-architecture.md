# 边缘网络与穿透调度节点 (Quic Service)

在所有复杂的端接端业务与远控指令之外。在互联网那些最昏暗且具有严酷隔离网关、对称 NAT 的路由背后。
是 `novaic-quic-service` 撑起了大后方的通讯中转站（Relay）打洞打不穿时候的救生衣地位。

---

## 1. 结构定位
作为纯粹偏向网络基建底层的一个完全无业务认知的独立仓库：
它是全线唯一的 P2P (Peer To Peer) 连接保障卫士。基于极低时延与握手开销的 QUIC 架构提供。

由于现代运营商的宽带层层叠加剥削公网 IP 以及防火墙横飞。
当你用公司的前端机器（挂在 Gateway 前端下面），试图向位于另一座家里机器上搭载运行在一套封闭虚拟机中的 `VmControl` 要求打一通建立共享的音视频 WebRTC 时点的时候，因为互不相认和各种打洞失效极大面临完全无法握手的状况报错死屏：这在旧款架构中引发了极惨的体验事故。
这就迫使我们一定要部署这套基础设施（常伴随 COTURN/STUN 同服运行）。

## 2. 核心职能分遣

- **全球中继器 (`Relay`)**：当 P2P 穿透的各种努力被确认是无能为力后，将由这个服务用纯私有网络流量代打中点！
- **STUN 认证发放（TURN 联合校验）**：网关在配置这部分资源的时候把配置和密码一并发起作为连入票据保护，防止白嫖带宽打挂 CDN 流量资源带宽资源。
- **静态前端的高速挂载下发 (Frontend CDN 卸载)**：由于这台极其特例化的机器就该位于所有网关之前做流量漏斗。我们把原来打包进极其笨重的网关自身（亦或者在其他业务层里）的静态 HTML 前端 `dist` 等构建发版物，改为了在它这个最靠边缘的最优节点的 Web 服务中兼职进行文件发牌加载送往外网与 OTA 更新。彻底净化主业务服务器集群，也实现了所谓按需部署 OTA Web 热更体系前端的最外衣职能！
