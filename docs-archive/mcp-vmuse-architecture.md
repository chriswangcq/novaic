# MCP VMuse 工具集架构总览

> 模块对应：`novaic-mcp-vmuse`

`MCP VMuse` 并非运行在云端的虚妄服务，而是一个真正落地、直插到用户侧操作系统甚至直接接管 QEMU 虚拟机的实体兵器库。其核心定位是：利用大模型支持的标准化工具协议（Model Context Protocol / MCP），将对真实世界软件和网页的一举一动操控权“交棒”给 Agent 大脑。

---

## 1. 突破沙盒：走向本地接管

在云端跑 Agent 最大的弱点是你无法触碰用户面前那台电脑或者那些独立设备真正跑着的画面。
VMuse 被打造出来弥合最后这断层的：它提供了一套可以由 Runtime (Saga 步骤 / Context) 请求调用并在对应端接续动作的网络接口组或者直调二进制方法集阵列：
它可以：
1. **直接操办 Chrome/Chromium**：通过控制调试协议打开那些充满验证登录并且恶心防抓取的网站，替你在真屏幕底下拖滚动条看。
2. **硬件断电/操作管理**：它如果生在云服务器跑着的模拟器母机边上，甚至能执行底层虚拟机电源重启或者键盘盲敲的暴力行为。

## 2. 核心架构面片

我们依照着 Anthropic 对外公开的那一套模型工具调度框架（MCP），把各种复杂的动作统合编排成了规范 JSON Schema 的服务方法。这让我们甚至不需要额外写什么专属给前端的数据格式（详细请参看 `docs/mcp-vmuse/` 系列专题篇）：

- **无缝对接底层虚拟系统 (`QMP & VNC`)**：通过与 QEMU monitor 通道接力，绕过什么层层嵌套代理去直接把指令投放到黑框虚拟机，并以毫秒级别将一幅幅截屏抽离后上发。
- **Browser-Automation (浏览器智控化机制)**：不再写愚蠢且容易因网页变更立刻崩溃的 xpath；通过注入 JS 取走带有节点框线树的简化版网页 DOM 层（例如只抽取了含有关联信息的纯净 Tag 附带相对坐标信息），扔给模型分析视觉点再指定点阵回打。这就是让大语言模型突然“长了眼睛和拥有鼠标准星”的核心魔术所在！
