## 1. 产品概述

### 1.1 一句话定位

**降低 Claude Code 能力的使用门槛** —— Claude Code 的能力，Manus 的易用性。

### 1.2 产品愿景

让每个人都能拥有一个「有自己办公室的 AI 员工」，你随时能去看他干活、接管操作，而不需要懂任何技术。

### 1.3 核心价值主张

| 价值点 | 描述 |
|--------|------|
| **能力强** | 本地代码执行、持久化工作空间、完整开发环境 |
| **看得见** | GUI 可视化，能看到 AI 在做什么 |
| **够安全** | 代码在本地虚拟机跑，数据不上云 |
| **易上手** | 不需要会终端，下载安装就能用，像普通 App 一样 |

### 1.4 产品定位比喻

```
Claude Code = 厉害的程序员，但只会用终端跟你交流
Manus      = 友好的助手，但每次见面都失忆
NovAIC      = 有自己办公室的 AI 员工，你随时能去看他干活
```

## 2. 市场分析

### 2.1 竞品对比

| 维度 | Claude Code | NovAIC | Manus |
|------|-------------|-------|-------|
| **核心能力** | 本地代码执行+持久化 | ≈同等能力 | 云端临时环境 |
| **使用方式** | 终端 CLI | 桌面 GUI | 网页 GUI |
| **目标用户** | 开发者 | 所有人 | 所有人 |
| **学习成本** | 高（需会终端） | 低 | 低 |
| **环境隔离** | ❌ 直接操作本地 | ✅ 沙盒隔离 | ✅ 云端隔离 |
| **环境持久化** | ✅ | ✅ 永久保留 | ❌ 每次重建 |
| **GUI 可视化** | ❌ | ✅ | ✅ |
| **浏览器操作** | ❌ | ✅ | ✅ |
| **数据位置** | 本地 | 本地 | 云端 |

### 2.2 市场定位

```
                    使用门槛
                      低
                       │
        ┌──────────────┼──────────────┐
        │     Manus    │    NovAIC     │
        │   (云端/不持久) │  (目标位置)   │
        │              │              │
能力弱 ─┼──────────────┼──────────────┼─ 能力强
        │              │              │
        │  通用网页AI   │  Claude Code │
        │              │  (本地/CLI)  │
        └──────────────┼──────────────┘
                       │
                      高
```

### 2.3 差异化优势

#### 比 Claude Code 强的地方

| 优势 | 说明 |
|------|------|
| **GUI 可视化** | 用户能看到 AI 在干什么，心理安全感 |
| **安全沙盒** | Claude Code 直接改你文件，我们是隔离的 |
| **浏览器自动化** | Claude Code 做不了 |
| **使用门槛** | 不需要会用终端 |
| **非开发者友好** | 数据分析师、运营、产品经理也能用 |

#### 比 Manus 强的地方

| 优势 | 说明 |
|------|------|
| **环境持久化** | 核心优势，Manus 最大痛点是每次重建环境 |
| **本地数据** | 数据不上云，更安全 |
| **无网络依赖** | 离线也能用（模型调用除外） |
| **完全接管** | 可以随时自己操作虚拟机 |

## 3. 目标用户

### 3.1 用户画像

#### 核心用户（优先级从高到低）

| 优先级 | 用户群体 | 痛点 | 为什么选我们 |
|--------|----------|------|--------------|
| P0 | **非技术人员** | 想用 AI 能力但不会用终端 | Claude Code 无法覆盖，我们能 |
| P1 | **隐私敏感用户** | 不想数据上云 | Manus 无法满足，我们本地执行 |
| P2 | **自动化玩家** | 想让 AI 帮忙操作电脑 | 爬虫、批量处理、浏览器自动化 |
| P3 | **开发者** | 想要 Claude Code 能力 + GUI | 更直观的开发体验 |

#### 典型用户场景

**场景 1：数据分析师小王**
```
背景：每周要处理 10+ 个 Excel 报表，手动操作繁琐
痛点：不会写 Python，Claude Code 看不懂终端
使用 NovAIC：
  1. 拖入 Excel 文件
  2. 说"帮我汇总销售数据，按地区生成图表"
  3. 看着 AI 在虚拟机里操作 Python
  4. 下载生成的报告
```

**场景 2：运营小李**
```
背景：需要批量处理社交媒体内容，收集竞品信息
痛点：Manus 每次都要重新配置环境
使用 NovAIC：
  1. 第一次配置好爬虫环境
  2. 之后每次打开，环境还在
  3. 说"帮我抓取 XX 网站的最新内容"
  4. 看着 AI 操作浏览器
```

**场景 3：律师老张**
```
背景：需要 AI 帮忙处理合同文档
痛点：合同内容敏感，不能上传到云端
使用 NovAIC：
  1. 所有处理都在本地虚拟机
  2. 数据不出电脑
  3. 说"帮我提取这份合同的关键条款"
```

#### 可视化执行设计

**核心原则**：用户能「看到」AI 在做什么

| 执行类型 | 可视化方式 | 说明 |
|----------|------------|------|
| **代码执行** | VNC 终端窗口 | 所有命令在可见终端中运行 |
| **浏览器操作** | VNC 浏览器窗口 | 用户看到浏览器被操作 |
| **文件操作** | 实时日志面板 | 显示读写进度 |
| **任意操作** | 执行日志面板 | 补充显示详细信息 |

**双重可视化**：
1. **VNC 画面**：虚拟机桌面，Agent 所有操作都在可见窗口中执行
2. **执行日志**：实时显示当前工具、状态、输出，作为 VNC 的补充

#### 关键交互说明

| 交互 | 说明 |
|------|------|
| **发送消息** | 输入框输入 + 回车/点击发送 |
| **上传文件** | 点击上传按钮 或 拖拽到对话区 |
| **下载文件** | 点击消息中的下载按钮 |
| **查看画面** | 右侧上方实时显示虚拟机桌面（终端窗口可见） |
| **查看日志** | 右侧下方显示实时执行日志（工具名、状态、输出） |
| **展开日志** | 点击「展开日志」查看完整输出 |
| **接管操作** | 点击「接管操作」后可直接在画面中操作 |
| **全屏模式** | 点击「全屏」放大虚拟机画面 |

### 5.2 技术选型

| 模块 | 技术选型 | 理由 |
|------|----------|------|
| **桌面框架** | Tauri | 比 Electron 轻量，Rust 安全 |
| **前端** | React + TypeScript | 生态成熟，开发效率高 |
| **VNC 显示** | noVNC | 纯 Web 实现，易集成 |
| **虚拟化** | QEMU (跨平台) / Virtualization.framework (macOS) | 成熟稳定 |
| **虚拟机系统** | Ubuntu + XFCE | 轻量，镜像小 |
| **Agent** | Python + FastAPI | 基于 Open Interpreter 改造 |
| **云服务** | Python + FastAPI | 快速开发 |
| **数据库** | PostgreSQL | 可靠，功能全 |
| **LLM** | Claude API (主) / GPT API (备) | Claude 编码能力强 |

          │
