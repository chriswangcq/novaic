# 界面瞬时状态管态：Zustand 的收纳艺术

> 路径：`novaic-app/src/stores/` 等全局共享切片

## 1. 被抛弃的 Context 瀑布流之恶
早期复杂的 Web App 喜欢将“谁打开了右边边栏、面板划到了哪个进度”这种属性全挂在极其宏伟的 `React Context API` 提供者上。结果每次鼠标改变选择框都会产生根级别的树崩重洗（Re-Render cascade）。

## 2. Zustand 原子化碎片管控
对于这种仅仅停驻在内存并只负责展示视图结构的瞬态（Ephemeral State）：我们彻底采用了 `Zustand`！
相比起冗长难懂的 `Redux`：
- **脱离树层结构**：你可以在任意毫不相关的深层组件、甚或在纯粹外部普通的 TypeScript 文件里发起 `useControlStore.getState().togglePanel()`，而不需要麻烦的传入动作钩子或是包装一层消费。
- **微订阅重获性能**：例如仅仅在那个右上角小方框按需引入 `const isPanelOpen = useControlStore(s => s.isPanelOpen);`只有真的面板开关动了它才会点亮此节点刷新。其它相邻树状分支兄弟甚至没有察觉这事发生过！

## 3. 为什么不存进数据库？
曾经有人争论将这部分数据统统扔给 Entangled，以求得甚至在不同设备里达成“一比一 UI 界面环境复制”。这在极其庞大（每秒十几帧切换的动画和滑轮改变）的密集触发型前端体系中对数据库造成了堪比压力测试的写入穿透。
现在界线划清：影响业务闭环的进 Rust 持久级数据库，单纯为了取悦眼睛的瞬时变化仅仅存在于这个极速调用的缓存切片堆栈上：关机即走，来去如风生。
