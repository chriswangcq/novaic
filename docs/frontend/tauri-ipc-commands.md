# IPC 的通讯防雷区与序列化节流

> 路径：涉及 `import { invoke } from '@tauri-apps/api'` 与底层通讯的文件夹及封装

## 1. 黑洞级阻塞：巨量序列化天花板
很多把 Tauri 当成银弹的开发者都会遇到一个惊惧事件——“调用原生语言明明非常快！为什么我的拉动滚动条把整个界面拖卡死了卡住了？”
问题归结到底部 IPC（进程间通信）。不论 Rust 有多猛，当你一通发给它一个包含了几千行配置或者带有个整片 `base64_image` 的超标载荷穿透那条管道：
这些庞然大物在底层全被以文本在经历：`JSON.stringify -> (过桥) -> Serde JSON 解析` 走着最消耗算力的反序列化刑场。

## 2. 解套：高频去抖与精简包裹
为了保护脆弱的命令管道：
- **节流阀防爆击**：当我们需要连打指令发给核心层要求移动镜头坐标（模拟滑轮动作）；前端严格地引入了由于 `lodash` 或相似节流方案衍生了如 `Throttle (20ms)`，确保我们不是在一秒内射出了成百上千次的空操作过桥让 Rust 的调度核心耗尽体力。
- **事件信号模式取代全景快照传递**：如果 UI 层想要更新，我们从坚决“把完整的表给 Rust”更改成为告诉 Rust：“嗨我的 ID，某个 ID 修改了开关 `toggle_vm(true)`”。把数据拼接及处理留在此前已经在后台待机的那个完整内存映射池内。

## 3. 防逆流回堵 (Event Listen)
同时，底端 Rust 给前层吐出的时候。对于那种极其巨大的文件流水变更也是采用分发局部变更（`Event Patch`）并在特定的长数据需求上以“触发信令——前端发起拉回提取截断头尾——呈现”的设计架构切开拥堵点。使得这层所谓的异步通信彻底从瓶颈深潭转化为滑入自如的传命飞线。
