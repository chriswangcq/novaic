# P2P 问题修复汇总（2026-03-11）

> 基于 5 名 subagent 调查结果实施的修复

---

## 一、问题与根因

### 1.1 `[CloudBridge] connect_via_relay failed: Invalid relay address: invalid socket address syntax`

**可能原因**：
- relay_url 含前导/尾随空格
- relay_url 无 scheme（如 `relay.gradievo.com`）导致 url 解析失败
- IPv6 host（如 `::1`）拼接成 `::1:443` 无法 parse（需 `[::1]:443`）

**修复**：`p2p/src/relay.rs`
- `parse_relay_url` 对输入做 `trim()`
- 无 scheme 时自动补 `https://`
- IPv6 使用 `[host]:port` 格式
- 错误信息增加 `host`、`port`、`addr_str` 便于排查

### 1.2 `[P2P] Registry register failed: error decoding response body: missing field 'ok'`

**根因**：Gateway 返回 401 时 body 为 `{"detail":"Unauthorized"}`，客户端仍按 `HeartbeatResponse { ok }` 解析失败。

**修复**：`p2p/src/rendezvous.rs`
- 已先判断 `status.is_success()`，失败时 `bail` 不解析
- 若仍进入解析，增加更明确的 parse 错误信息（含 body 前 100 字符）

**说明**：若仍出现 `missing field ok`，多半是 Gateway 返回了 2xx 但 body 格式异常。401 时应直接得到 `heartbeat failed (401): Unauthorized`。

### 1.3 VNC WebSocket 连接被对方重置

**原因链**：
1. PC 端 CloudBridge `connect_via_relay` 失败（relay address 解析等）
2. PC 未完成 `RegisterPc`，relay 端无 session
3. 手机 `connect_via_relay` 收到 `PC offline or session expired`
4. 手机端 `get_or_create_remote_conn` 失败 → `route_vnc` 返回 Err → WebSocket 被 drop。

### 1.4 P2P 心跳 401

**根因**：PC 的 JWT 过期或无效，Gateway 返回 401。

**处理**：重新登录刷新 token；或检查 Gateway 的 JWT 校验逻辑。

## 三、Gateway 部署（git pull）

```bash
cd novaic-gateway
./scripts/deploy-gateway.sh root@api.gradievo.com
```

流程：服务器 `git pull` → 拉取 `restart_gw.sh`、`jwt_secret.env.example`、`deploy-gateway.sh` → 执行 `restart_gw.sh` 重启 Gateway。

## 四、建议排查步骤

1. **确认 relay_url**：在 Gateway 中检查 `RELAY_URL` 或 `jwt_secret.env`，应为 `https://relay.gradievo.com/p2p/relay`
2. **确认 PC 端**：重新登录刷新 token；确认 `connect_via_relay` 错误不再出现
3. **重新构建并安装到手机**：`cd novaic-app && bash scripts/build-and-install-ios.sh`
