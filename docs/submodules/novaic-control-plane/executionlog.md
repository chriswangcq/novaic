已推送。下面讨论 Execution Log 的交互优化。

---

看截图，目前 execution log 是一个顶部浮条，显示"513be29c 5s / 暂无日志"。可以从几个维度讨论：

---

## 1. 信息密度优化

**现状**：一行 ID + 时间 + "暂无日志"，信息量少但占位不小。

**可做**：
- **折叠/展开动画**：默认只显示一行摘要（当前步骤 + 耗时），点击展开后露出完整步骤列表，有 height transition
- **进度感知**：显示 `步骤 3/? · 12s` 而不是纯时间，让用户知道 agent 在第几步

---

## 2. 状态视觉反馈

**现状**：运行中和空闲状态视觉区分不明显。

**可做**：
- **运行中**：左侧加一个绿色呼吸灯或 spinner，标题行有轻微 pulse
- **成功/失败**：完成后变色（绿色 ✓ / 红色 ✗），停留 3s 后自动收起
- **工具调用高亮**：每次调用工具（如 `bash`, `file_write`）出现时，有一个小 tag 出现在摘要行

---

## 3. 交互行为

**现状**：好像只能展开，收起方式不明确。

**可做**：
- **自动收起**：agent 完成后 3～5s 自动收起
- **悬停暂停收起**：鼠标悬浮在 log 上时暂停倒计时，移走后继续
- **ESC 收起**：和 VM 浮窗统一按 ESC 收起

---

## 4. 日志内容本身

**现状**：显示"暂无日志"，内容空。

**可做**：
- **实时流式追加**：每行工具调用 → 结果 按顺序出现，有 fade-in
- **分层折叠**：`bash` 调用默认折叠（显示命令摘要），点击展开看完整 stdout
- **错误凸显**：报错行用红色背景 + 图标，一眼可见

---

## 5. 位置与布局

**现状**：固定在对话区顶部，遮挡内容。

**可做**：
- **吸附到对话底部（输入框上方）**：把 log 条从顶部移到底部，视觉动线更自然（用户看完 log → 直接输入下一条）
- **最小化态**：缩成一个小药丸，只显示 spinner + "Running · 8s"，点击展开详情
