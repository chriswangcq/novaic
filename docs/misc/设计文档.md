明白了，你的意思是：

1. **Tools Server 统一调用风格**：所有工具调用都用 `/internal/agents/{agent_id}/...` 或 `/internal/subagents/...` 风格
2. **Gateway 内部映射**：Gateway 收到请求后，自己决定调 VmControl 还是其他服务

---

## 目标架构

```
Tools Server
    │
    │  统一风格：POST /internal/agents/{agent_id}/tools/{tool_name}
    │  或：POST /internal/agents/{agent_id}/vm/{operation}
    │  或：POST /internal/agents/{agent_id}/mobile/{operation}
    │
    ▼
Gateway
    │
    │  内部映射：
    │  • vm/* → VmControl /api/vmuse/{agent_id}/*
    │  • mobile/* → VmControl /api/android/{device_serial}/*
    │  • 其他 → RO / Queue / ...
    │
    ▼
VmControl (或未来的 PC 端)
```

---

## 对比

| 当前 | 目标 |
|------|------|
| Tools Server 直连 VmControl | Tools Server 只调 Gateway |
| 不同服务用不同 URL | 统一用 Gateway URL + agent 路径 |
| Gateway 做透明代理 | Gateway 做逻辑映射 |

---

## 路径设计

### 方案 A：按工具类型分

```
POST /internal/agents/{agent_id}/vm/screenshot
POST /internal/agents/{agent_id}/vm/browser/navigate
POST /internal/agents/{agent_id}/vm/shell/exec
POST /internal/agents/{agent_id}/vm/file/read

POST /internal/agents/{agent_id}/mobile/screenshot
POST /internal/agents/{agent_id}/mobile/touch
POST /internal/agents/{agent_id}/mobile/app/install
```

### 方案 B：统一工具调用

```
POST /internal/agents/{agent_id}/tools/call
{
  "tool": "browser_navigate",
  "arguments": { "url": "..." }
}

POST /internal/agents/{agent_id}/tools/call
{
  "tool": "mobile_screenshot",
  "arguments": { "device_serial": "..." }
}
```

Gateway 根据 `tool` 名称判断走 VmControl 还是其他。

---

## 推荐：方案 A（按类型分路径）

更清晰，Gateway 路由逻辑简单：

```python
# Gateway 路由

# VM 工具 → VmControl /api/vmuse/
@router.post("/internal/agents/{agent_id}/vm/{path:path}")
async def vm_tool(agent_id: str, path: str, request: Request):
    # 映射到 VmControl
    vmcontrol_url = f"{VMCONTROL_URL}/api/vmuse/{agent_id}/{path}"
    ...

# Mobile 工具 → VmControl /api/android/
@router.post("/internal/agents/{agent_id}/mobile/{path:path}")
async def mobile_tool(agent_id: str, path: str, request: Request):
    # 先查 agent 的 device_serial
    agent = get_agent(agent_id)
    device_serial = agent.android.device_serial
    vmcontrol_url = f"{VMCONTROL_URL}/api/android/{device_serial}/{path}"
    ...
```

## Tools Server 改动

```python
# 当前
url = f"{VMCONTROL_URL}/api/vmuse/{agent_id}/browser/navigate"

# 改为
url = f"{GATEWAY_URL}/internal/agents/{agent_id}/vm/browser/navigate"
```

```python
# 当前
url = f"{VMCONTROL_URL}/api/android/{device_serial}/screenshot"

# 改为（不需要传 device_serial，Gateway 自己查）
url = f"{GATEWAY_URL}/internal/agents/{agent_id}/mobile/screenshot"
```

---

## 好处

1. **Tools Server 简化**：不需要知道 device_serial，只传 agent_id
2. **Gateway 集中控制**：agent → device 映射在 Gateway
3. **云化友好**：Gateway 可以根据 agent_id 找到对应用户的 PC
4. **统一风格**：所有工具调用都是 `/internal/agents/{agent_id}/...`
