# 网页点阵过滤与防沉迷视觉操作 (Vision Automation)

> 路径参考：与 Chromium 操纵及 DOM 打标相关的控制实现域

## 1. “笨死”的 XPath 穷尽法
你可能看到过很多基于 Puppeteer/Selenium 去写的发帖机器人或者是网页监控机器人。这帮代码都充斥着极度令人绝望或者脆弱的死绑指令：如 `button[@id="submit-now"]/div/span`。
只要微博或者是推特工程师把前面的 div 包裹加了个间距或者改了个布局包，整套机器人由于抓不到 DOM 当场宕机摆烂。

## 2. 我们如何教神明认识网页结构
在 VMuse 所使用的这套高阶逻辑里，我们要达成“即使这网页千变万化，大模型依然能通过眼力找到你要它点的提交位置”：
- **页面标记过滤 (Tagging & Striping)**: 
  当需要执行寻找或者是截图给模型看时，我们首先使用底端的协议向这网页注入了一段极度暴力的降维 JS 脚本：不去找这页面的 JS 执行了啥，我们粗暴的遍历那网页并且把屏幕以外或者是对点击毫无用处（纯装饰性的、根本无法触发事件操作的元素如没有下划线的死板 span 等）的冗余结构全部连根切除消隐。
- **建立隐形坐标点阵**：
  并且紧随其上，它给所有可能被认为有操作意义上的“有效标签”强制附加了一层隐形的带数字编号的外边框方块罩，带到了模型截图获取的那一瞬间的原始画幅里；甚至我们干脆将这个抽取了坐标系的极简 DOM Tree 打包成带有关联标记文本，用它自己的理解送回。

## 3. 下针到点
大模型由于极具智慧，它瞬间把图片上被打满格子数字号或者简编里的对象进行了匹配（“哎，我看到 45 号是被盖在了我要他按下的提交按钮位置上的！”）。
从而模型回抛出动作命令：`click(target: 45)`。我们再通过映射器将 `45` 化为了对应它屏幕（X, Y）甚至带个随机抖动偏移去点击那个该死的核心坐标。完美消杀网页频繁热更改版的维护灾难。
