# 夺命主线程：macOS SIGTRAP 与 CGEvents 重构

> 路径参考：`vmcontrol/src/input/handler.rs`

## 1. 那场离奇闪退坠机案 (The SIGTRAP Trace)
这是一场耗费极大精气神并载入 NovAIC 史册的 Bug。当时当外部设备连起本地跑有 `vmcontrol` 驻留在开发用户的机器进行“远控键盘或者拼写”时，会毫无前兆地遭遇：
“应用程序意外停止”！翻看苹果极其隐晦的堆栈调用图发现：报错永远直戳到在系统核心包 `AppKit` 底层某无名偏移调用挂掉的陷阱异常信号：`SIGTRAP`。

## 2. 定位到跨平台键盘录放库 `enigo`
排查后发信：原来在我们接取跨平台键位输入控制并命令我们的 macOS 做点击操作时（模拟用户的手移动或者打字）。
引用了广为认同的一套开源 Rust 底系统外设控制库 `enigo`。
这库在背后偷偷干这极其危险的勾当！由于原版的它试图调用操作系统更贴近 App 环境级别的人机包做键码处理 —— 在苹果苛刻无理的开发条文限制中：
> 所有牵挂到有关前台 UI 以及高频输入拦截相关的核心触发调用，**绝对且只能允许寄存在进程生命初始的那个纯粹的主循环线程队列之上 (The Main Thread)**！
而 `vmcontrol` 底层跑的是极致发疯高速处理各种端口通信网关的 `Tokio` Worker 异步多线程大池子。从某不知名野线程一脚跨进这领地踩碎边界时，内核为了防御非法接管，毫不留情发出了强行中止。
