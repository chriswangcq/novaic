## ⚠️ 重要提醒

### 破坏性变更

1. **API 端点删除**: `/config/ports/{agent_index}` 不再可用
2. **数据格式要求**: 所有 agent 必须有 `ports` 配置
3. **运行时行为**: 不再自动从 `agent_index` 计算端口

### 迁移要求

如果数据库中存在旧格式数据，需要：
1. 手工迁移数据库
2. 为所有 agent 添加 `ports` 配置
3. 删除 `vm_config` 中的 `agent_index` 字段
