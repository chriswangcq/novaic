# Round 008 Gate Criteria

## Gate A - Execution Evidence
- Every DONE item must include:
  - exact command(s)
  - pass/fail summary
  - artifact/doc path

## Gate B - Governance Stability
- Storage/API/Platform governance policies are finalized in stable locations.
- No mandatory governance dependency remains only in round-specific reports.

## Gate C - Operability Finalization
- Runtime and Desktop have replayable, documented final operating checks.

## Gate D - Reliability Finalization
- Agent Runtime and Tools have repeatable replay checks with policy-aligned defaults.

## Fail Conditions
- Any team claims DONE without implementation evidence.
- Any report is analysis-only without executed work.
- Decision-needed issue lacks owner and deadline.
