# Round 006 Gate Criteria

## Gate A - Execution Evidence
- DONE items must include:
  - exact command(s)
  - pass/fail summary
  - produced artifact/doc paths

## Gate B - Governance Closure
- Storage schema governance ownership and approval rule is finalized and documented.
- Evidence path policy (round-specific vs evergreen) is finalized and applied in CI.

## Gate C - Operability
- Runtime probe governance is implemented in code/CI.
- Desktop validation workflow is replayable by others.

## Gate D - Reliability
- Agent Runtime diagnostics and high-load replay are implemented and evidenced.
- Tools reliability replay remains green with prerequisite checks.

## Fail Conditions
- Report claims completion without execution evidence.
- Team submits only analysis/understanding without implementation artifacts.
- Decision-needed issue has no owner/deadline.
