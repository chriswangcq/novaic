# Definition of Done (DoD)

A team task is considered done only when all conditions below are true:

1. Functional change is implemented.
2. Acceptance criteria in dispatch file are met.
3. Evidence is attached (test command output summary, artifact path, docs path).
4. No new critical lints or failing tests introduced by the change.
5. Required docs are updated.

## Evidence Format
- `tests`: list exact commands and pass/fail summary
- `artifacts`: produced files, release tags, or package references
- `docs`: path to updated docs/specs/runbooks
