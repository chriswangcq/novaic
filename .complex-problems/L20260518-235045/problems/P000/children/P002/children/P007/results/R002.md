# 边缘与基础设施服务代码调研结果

## Summary

通过三组并行代码调研，完成了对 novaic-gateway、novaic-device、novaic-llm-factory、novaic-sandbox-service、novaic-logicalfs、novaic-blob-service、novaic-common 以及 novaic-app（含 Entangled 客户端同步引擎）的全面代码级调研。每个服务/库的入口文件、API 路由、核心机制、端口和启动方式均已确认。

## Done

- **novaic-gateway（:19999）**：Python/FastAPI 服务（非 Rust），双 JWT 认证体系（Clerk RS256 外部 + 内部 HS256），WebSocket 信令代理（`/ws/signaling`），Blob 代理路由，OAuth 回调，API 路由按 `routers/` 目录模块化组织
- **novaic-device（:19993）**：Python/FastAPI 服务，类型化命令 WebSocket 协议，支持 3 种设备类型（linux/android/host_desktop），mounted tools 系统实现工具挂载，50+ API 路由，VmControl/CloudBridge 集成
- **novaic-llm-factory（:9100）**：Python/FastAPI 独立服务，Provider 路由机制（OpenAI/Anthropic/Google），RSA-2048 API key 加密存储，SQLite 4 表持久化，指数退避重试策略，外部默认 URL 配置
- **novaic-sandbox-service（:19994）**：Python/FastAPI 服务，AsyncProcessRunner 进程执行引擎，Linux mount namespace 隔离，bind mount 命令构造，支持 shell 命令执行与文件操作
- **novaic-logicalfs**：纯 Python 库（非服务），Snapshot/Authority/Store 三层抽象，BlobObjectStore HTTP 适配器连接 Blob Service，支持目录快照与文件操作
- **novaic-blob-service（:19995）**：Python/FastAPI 服务，双 API 体系（blob:// 原始存储 + object tree 结构化对象树），DiskBackend + OssS3Backend 双后端，multipart 分块上传协议
- **novaic-common**：共享 Python 库，ServiceConfig 严格加载（services.json 配置源），httpx 异步 HTTP 客户端，9 个 contract 模块定义服务间数据契约，5 个 LLM 内建工具定义，wake assembler 消息组装器，EntangledServiceClient WebSocket 客户端
- **novaic-app**：桌面/移动客户端应用（Tauri 2.x + React 18 + TypeScript），内嵌 Entangled 客户端同步引擎（Rust crate），pessimistic-first + optimistic-display 架构，15 个注册实体，WS 同步协议

## Verification

- 每个服务的入口文件已通过代码阅读确认（main.py / app.py / __init__.py）
- API 路由通过 router 注册代码确认
- 端口号通过 ServiceConfig 或启动配置确认
- Gateway 确认为 Python/FastAPI 实现，纠正了此前文档中 Rust 的错误描述
- novaic-app 确认为客户端应用，非 Entangled 后端服务
- novaic-logicalfs 确认为纯库，无独立端口
