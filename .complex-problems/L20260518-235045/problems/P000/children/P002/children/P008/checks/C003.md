# P008 前端与桌面端代码调研成功检查

## Summary

P008 要求调研 novaic-app（Tauri + React）的组件结构/Tauri 命令/状态管理/路由/Entangled 集成、VmControl 的 Rust 模块结构/WebRTC/Scrcpy 集成/API 路由、mcp-vmuse 的 MCP 工具注册和桌面控制能力、以及前后端通信方式。R003 对所有目标均完成了代码级调研，每项 success criteria 均有代码证据支撑。

## Evidence

- app React 组件结构：src/components/ 下 13 个功能子目录，组件树从 App.tsx → AuthPage | LayoutContainer 展开，所有主要页面组件已识别
- Tauri 命令：27 个命令在 src-tauri/src/lib.rs:411-442 通过 generate_handler! 注册，含 gateway 代理、安全存储、文件缓存、WebRTC 信令、Entangled 同步等
- 状态管理：Zustand 主 store（src/application/store.ts）管理启动/身份/Agent/模型/设备/UI/布局等状态域 + TanStack Query 管理 Entangled 实体数据（staleTime=Infinity，事件驱动 invalidation）
- 路由：不使用 react-router，通过 primaryTab + narrowPage 自定义布局状态实现，桌面三栏 + 移动单栏
- Entangled 集成：Rust crate 位于 Entangled/packages/client-rust/，WS 协议（Entangle/Disentangle/Action/Sync/Ack/Schema），SQLite 缓存，悲观写入 + 乐观展示架构，15 个同步实体
- VmControl：独立 Rust workspace crate（src-tauri/vmcontrol/），嵌入式服务架构，统一 WebRTC 入口按 DeviceKind 路由，VideoBroadcaster + Scrcpy 3.3.4 集成，Cloud Bridge WS 30 种命令
- mcp-vmuse：Python aiohttp HTTP JSON 服务器，6 类工具（Desktop/Browser/Shell/File/Window/Context）共 40+ API 端点，AimClick 两阶段鼠标协议
- 前后端通信：三通道架构（GatewayClient HTTP / AppBridge WS / EntangledSyncBridge WS），前端零直连，6 个 Tauri 事件
