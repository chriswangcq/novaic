# 前端与桌面端代码调研方案

## Problem Definition

novaic-app 是基于 Tauri 2.x + React 18 的桌面/移动客户端，内含 Entangled 客户端同步引擎（Rust crate）和 VmControl 模块。novaic-mcp-vmuse 是独立的 MCP 桌面控制模块。需要通过代码阅读确认：React 组件结构、Tauri 命令体系、状态管理、路由、Entangled 集成方式、VmControl 的 Rust 模块结构及 WebRTC/Scrcpy 集成、mcp-vmuse 的 MCP 工具注册和桌面控制能力、前后端通信方式。

## Proposed Solution

分两组并行调研：

1. **novaic-app 前端 + Tauri 层**：阅读 React 组件目录结构（src/ 下的 components/pages/hooks/stores）、Tauri 命令定义（src-tauri/src/ 下的 commands）、状态管理方案（zustand/redux/context）、路由配置、Entangled 客户端集成代码（Rust crate 入口和 React 绑定）
2. **VmControl + mcp-vmuse**：阅读 VmControl Rust 模块（src-tauri/ 内或独立 crate）的结构、WebRTC 连接管理、Scrcpy 集成、API 路由；阅读 novaic-mcp-vmuse 的 MCP 工具注册、桌面控制实现、与 VmControl 的集成方式

## Acceptance Criteria

- React 组件树顶层结构和主要页面组件已识别
- Tauri 命令列表和调用方式已确认
- 状态管理方案和关键 store 已确认
- 路由配置和页面映射已确认
- Entangled 客户端同步引擎的 Rust 入口和 React 绑定已确认
- VmControl 模块结构、WebRTC/Scrcpy 集成方式已确认
- mcp-vmuse MCP 工具注册和桌面控制能力已确认
- 前后端通信方式（HTTP API、WebSocket、Entangled sync）已确认

## Verification Plan

逐条对照 success criteria 验证每项发现是否有代码级证据支撑。

## Risks

- novaic-app 前端代码量较大，可能需要聚焦于架构级结构而非逐文件详读
- VmControl 可能已迁移或重构，实际位置需通过代码确认
- mcp-vmuse 可能依赖外部 MCP SDK，需确认其本地集成方式

## Assumptions

- novaic-app 子模块已 checkout 且包含完整源码
- VmControl 在 novaic-app 的 src-tauri 目录或关联 crate 中
- novaic-mcp-vmuse 子模块已 checkout
