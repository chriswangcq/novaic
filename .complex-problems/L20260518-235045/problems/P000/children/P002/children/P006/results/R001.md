# 后端核心服务代码调研完成

## Summary

完成了 novaic-agent-runtime、novaic-cortex、novaic-business 三个核心后端子模块的代码结构调研。每个服务的入口文件、目录结构、API 路由表、核心模块划分、服务间调用关系、配置和依赖均已详细记录。

## Done

- novaic-agent-runtime 代码结构调研：入口 main_novaic.py (9 种子命令), Queue Service API (45+ 路由), 6 种 Worker 类型, 5 种 Saga 定义, 15+ TaskTopic handler, 5 种内置工具, 完整的服务间调用链 (Queue→Cortex→Business→LLM Factory→Blob)
- novaic-cortex 代码结构调研：入口 main_cortex.py + cli.py, 36 个源文件, 40+ HTTP API 路由 (Scope/Context/Meta/Steps/Payload/Shell), ContextEvent 领域模型 (10 种事件类型), 三级 token 预算压缩, Redis 分布式 scope 锁, SQLite operational store, Sandbox/LogicalFS 集成
- novaic-business 代码结构调研：双进程架构 (API + Subscriber), 9 个 Internal API router, 60+ HTTP 路由, 19 个 Entity Schema, 30+ Entity Action Hook, DispatchSubscriber IM 聚合机制, SubAgent 状态机, 完整服务间调用关系 (Entangled/Device/LLM Factory/Queue/Gateway)

## Verification

- 每个调研由独立的 research agent 执行，通过直接阅读源代码 (入口文件、路由定义、handler 注册表、配置文件、requirements.txt) 获取事实
- 调研覆盖了所有要求的维度：入口文件和启动命令、目录布局、API 路由表、核心模块划分、服务间调用关系、配置和依赖
- 事实基于代码直接读取而非旧文档推断
