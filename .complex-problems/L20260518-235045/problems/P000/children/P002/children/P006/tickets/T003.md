# 后端核心服务代码调研

## Problem Definition

需要深入阅读 novaic-agent-runtime（含 Queue Service）、novaic-cortex、novaic-business 的代码，提取架构事实用于编写新文档。

## Proposed Solution

逐个服务阅读入口文件、目录结构、API 路由定义、核心模块。记录事实而非猜测。

## Acceptance Criteria

- 三个服务各自的入口文件、目录结构、API 路由、核心模块、进程角色记录完整
- 服务间调用关系清晰
- 信息基于代码而非旧文档

## Verification Plan

1. 每条事实可以用 grep/find 在代码中定位
2. API 路由与实际 @app.get/@app.post 装饰器对应

## Risks

- agent-runtime 代码量大，需要聚焦核心路径

## Assumptions

- 入口文件、路由定义、目录结构足以支撑架构文档编写
