# 归档旧文档并清理文档目录

## Problem Definition

docs/ 目录有 651 个文件需要归档到 docs-archive/。CI guard 脚本依赖 docs/ 下特定路径（lint_docs_status_consistency.py 依赖两个文件、lint_current_docs_residue.sh 扫描 docs/architecture 等目录、lint_deploy_fresh_smoke.py 依赖 docs/runbooks/deploy.md、check_start_config_contract.py 检查目录存在性）。

## Proposed Solution

1. `git mv docs docs-archive` 整体归档
2. 创建新 docs/ 目录
3. 在新 docs/ 下创建空的 architecture/、runtime/、runbooks/、reference/ 目录（满足 CI guard 扫描路径）
4. 将 CI guard 硬依赖的文件从归档中复制或创建占位（具体文件：docs/runbooks/deploy.md、docs/roadmap/tickets/PR-338-*.md、docs/architecture/generic-worker-substrate-plan.md）
5. 运行所有 CI guard 确认通过

## Acceptance Criteria

- 旧文件全部在 docs-archive/ 中
- 新 docs/ 目录结构满足 CI guard 路径要求
- 所有 docs 相关 CI guard 通过
- git status 干净

## Verification Plan

1. 确认 docs-archive/ 文件数 == 原 docs/ 文件数
2. 运行 lint_docs_status_consistency.py
3. 运行 lint_current_docs_residue.sh
4. 运行 lint_deploy_fresh_smoke.py
5. 运行 check_start_config_contract.py

## Risks

- git mv 可能影响子模块状态
- CI guard 可能有未发现的路径依赖

## Assumptions

- git mv 保留历史
- CI guard 只依赖已分析的路径
