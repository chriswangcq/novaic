# CI guard 适配与最终验证方案

## Problem Definition

新文档结构下需要确认所有 CI guard 脚本通过，并做最终的文档-代码一致性验证。CI 相关脚本：lint_docs_status_consistency.py、lint_current_docs_residue.sh、lint_deploy_fresh_smoke.py、check_start_config_contract.py。

## Proposed Solution

1. 运行所有 4 个 CI guard 脚本，确认通过
2. 检查 git diff --stat 变更范围
3. 确认 docs-archive/ 完整
4. 交叉验证文档中的关键引用（端口、服务名、路径）

## Acceptance Criteria

- 4 个 CI guard 脚本全部通过（exit 0）
- git diff --stat 变更范围合理（仅 docs/ 和 docs-archive/）
- docs-archive/ 包含所有旧文档
- 新文档不含 CI 禁止的 retired text

## Verification Plan

逐个运行 CI 脚本，检查 exit code 和输出。

## Risks

- check_start_config_contract.py 可能扫描新文档中的内容

## Assumptions

- 前面 P001-P004 的工作已完成
