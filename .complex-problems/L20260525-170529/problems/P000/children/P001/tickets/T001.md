# Add public_title to backend activity projection

## Problem Definition

`agent-activity-records` lacks an explicit public row title field. Runtime emits `title` and `text`, but the title is not consistently the UI title, and the frontend compensated with keyword inference from `text`. The backend contract should provide `public_title` so the UI can render structured labels without reading private/detail content.

## Proposed Solution

Add `public_title` to the shared entity contract and Business schema definition. Extend `_emit_record()` in runtime activity projection to accept and store `public_title`, defaulting to the emitted title when no override is supplied. Update projection tests to assert `public_title` exists for reasoning and tool records while preserving existing `text` behavior.

## Acceptance Criteria

- `public_title` is declared in shared/app entity contracts and Business schema push definitions.
- Runtime records include `public_title` for reasoning, action, observation, and summary records.
- Existing `text` fields are preserved for reasoning detail and shell descriptions.
- Focused runtime projection tests pass.

## Verification Plan

- Inspect relevant schema/contract files before edits.
- Update runtime projection and tests.
- Run `pytest tests/test_pr193_activity_projection.py` in `novaic-agent-runtime`.
- Review focused diff.

## Risks

- Contract definitions may have duplicate copies; missing one would cause app/runtime drift.
- New field must not replace `title` immediately because older consumers may still use it.

## Assumptions

- Adding a nullable/text field to entity contracts is backward-compatible for existing rows.
- `public_title` can initially mirror `title` for records that already have a good public title.
