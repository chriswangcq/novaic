# Frontend public_title consumption implemented

## Summary

Updated the app-side Agent Monitor path to consume `public_title` as the structured title authority, while preserving legacy structured fallbacks and reasoning detail text.

## Done

- Added `public_title` to app Entangled contract, activity entity type, timeline record type, and normalization allowlist.
- Updated `ActivityTimeline` to prefer explicit `public_title` before all fallback title logic.
- Kept legacy shell read/reply fallbacks for records without `public_title`.
- Removed reasoning-text keyword title inference from the rendering path.
- Added tests proving reasoning text containing `回复` does not create `正在组织回复`.
- Added tests proving explicit `public_title` can render as the title authority.
