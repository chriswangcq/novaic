# App monitor same-row streaming rendering

## Problem Definition

The App already subscribes to `agent-activity-records`, but timeline scrolling/rendering was mostly keyed to newly appended rows. Streaming reasoning updates will update the same record's text/status, so the UI must continue to render and follow bottom without a new frontend channel.

## Proposed Solution

Split App work into timeline rendering/autoscroll behavior and data-contract verification. Update the timeline to treat latest same-row text/status changes as follow-bottom triggers and ensure tests cover running reasoning records. Verify existing Entangled store/type contracts remain the only data path.

## Acceptance Criteria

- Same-row reasoning text/status updates can trigger bottom-follow when near bottom.
- Running reasoning row displays existing public title and detail behavior.
- App continues to rely on `agent-activity-records` Entangled stream store.
- Tests cover timeline projection/rendering behavior.

## Verification Plan

- Add/update `ActivityTimeline` tests.
- Run focused App tests for timeline and entity contract.

## Risks

- Browser layout tests in jsdom cannot perfectly prove visual scroll, so dependency/key behavior should be unit-tested and kept simple.

## Assumptions

- Entangled invalidation will refresh React Query data when the backend upserts the same record.
