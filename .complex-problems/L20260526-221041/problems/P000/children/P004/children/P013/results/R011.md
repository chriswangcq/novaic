# Streaming contract and residue review complete

## Summary

Reviewed the touched Factory, Runtime, and App paths for fallback/residue and prompt-history leakage. No runtime cleanup was needed. I tightened the Runtime streaming failure test so future code cannot silently fall back from `stream_chat()` to non-streaming `chat()`.

## Done

- Reviewed Factory streaming route/provider diff: streaming requests use `provider.stream_chat`; unsupported providers fail with `unsupported_stream` and HTTP 400 rather than falling back to non-streaming execution.
- Reviewed Runtime handler path: `handle_llm_call` calls `biz.call(..., stream=True, on_delta=...)`; stream deltas are only accumulated for activity projection, and `prepared.messages` is passed unchanged into the Factory stream request.
- Reviewed Runtime accumulator: final response is assembled from chunks after the stream completes; partial reasoning deltas are not appended to request messages.
- Reviewed App path: Timeline continues to read `agentActivityRecordsStore.useStream(params)` through `useActivityTimeline`; guard tests reject `new EventSource`, `new WebSocket`, and `text/event-stream` in the monitor path.
- Tightened `test_llm_handler_surfaces_stream_failure` so any future fallback to `chat()` raises an assertion.
