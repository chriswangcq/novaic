# Verify reasoning streaming across repos and remove transitional residue

## Problem Definition

The reasoning-streaming change crosses Factory, Runtime, and App. Even with leaf tests passing, final closure needs a repo-wide focused verification pass plus cleanup of any confusing fallback/residue that would make the long-term architecture ambiguous.

## Proposed Solution

Split the verification into focused child problems: one for cross-repo test/build evidence, one for contract/residue review, and one for final documentation/ledger closure. Each child should produce concrete evidence before P004 is summarized.

## Acceptance Criteria

- Focused tests pass in every touched repo: `novaic-llm-factory`, `novaic-agent-runtime`, and `novaic-app`.
- Cross-boundary streaming contracts are either tested directly or guarded clearly enough that future agents understand the path.
- Diff/source review finds no long-term fallback branch, misleading stale path, or accidental raw partial reasoning being fed back into LLM input history.
- Final P004 result cites child result IDs, stress evidence, residual risk, and artifacts.

## Verification Plan

Use child problems to run touched-repo tests, inspect cross-boundary source contracts, inspect diffs for residue/fallbacks, and then summarize only after all children close.

## Risks

- Full suites may be slow or have unrelated failures; child results must distinguish touched-path failures from pre-existing noise.
- Overbroad cleanup could disturb unrelated recent user work, so cleanup must stay scoped to reasoning streaming residue.

## Assumptions

- The intended long-term path is Factory SSE chunks -> Runtime aggregation/activity projection -> Entangled activity records -> App Timeline.
- Non-streaming fallbacks for the new reasoning path are not desired unless they are existing unrelated APIs kept for compatibility outside this construction.
