# Add streaming reasoning activity projection helper

## Problem Definition

Runtime needs to update one stable Agent Monitor reasoning row during streaming and finalize it without duplicate records or per-token writes.

## Proposed Solution

Add `project_reasoning_stream_update()` for deterministic running/final reasoning upserts and a `ReasoningStreamProjector` coalescer that emits only the first, thresholded, and final updates. Reuse the existing `assistant:reasoning` seed so final `project_assistant_message()` remains compatible.

## Acceptance Criteria

- Running update writes the same record id as the final assistant reasoning projection.
- Running update uses `public_title=正在思考` and `status=running`.
- Final update uses `public_title=完成思考` and completed status semantics.
- Coalescer suppresses tiny intermediate updates and always emits final text.

## Verification Plan

- Add tests to `test_pr193_activity_projection.py`.
- Run the focused activity projection tests.

## Risks

- Overwriting sort order on repeated upserts could reorder the running row; acceptable because the row represents current active thought and final append will normalize it.

## Assumptions

- Existing `entity_upsert` is the correct projection write path.
