# Add Runtime Factory SSE aggregator

## Problem Definition

Runtime needs a Factory client method that can request `stream=true`, parse Factory normalized SSE chunks, and return a complete OpenAI-style response compatible with existing callers.

## Proposed Solution

Add stream parsing and aggregation helpers to `task_queue.factory_client`. Implement `FactoryLLMClient.stream_chat()` that sends the same payload as `chat()` plus `stream=true`, parses Factory SSE events, optionally notifies a delta callback, aggregates content/reasoning/tool-call fragments, and returns a final chat completion response.

## Acceptance Criteria

- Parses `data:` SSE chunks for `type=delta`, `type=done`, and `type=error`.
- Aggregates content and reasoning text into final assistant message fields.
- Aggregates function tool-call argument fragments by index.
- Returns a final OpenAI-style response with choices/message/finish_reason/usage.
- Existing blocking `chat()` behavior remains intact.

## Verification Plan

- Add focused unit tests under `novaic-agent-runtime/tests/unit/task_queue`.
- Run the new test file and existing Factory client multimodal test.

## Risks

- Tool call streaming has provider-specific edge cases; cover the current normalized shape and keep the aggregator simple.

## Assumptions

- Factory route emits normalized SSE from P006.
