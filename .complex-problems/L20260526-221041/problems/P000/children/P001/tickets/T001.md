# Define Factory streaming provider and route behavior

## Problem Definition

Factory has a `stream` request field but no implemented stream behavior. Runtime needs a normalized stream it can consume while the existing non-streaming endpoint remains compatible.

## Proposed Solution

Split Factory work into provider-level streaming normalization and route-level response/logging behavior. Implement OpenAI-compatible streaming first, explicitly reject unsupported native provider streaming, and preserve non-streaming tests.

## Acceptance Criteria

- Provider streaming leaf defines and tests normalized chunks for OpenAI-compatible SSE.
- Route streaming leaf exposes an HTTP streaming response and clear unsupported-provider behavior.
- Existing non-streaming chat completion tests keep passing.

## Verification Plan

- Add provider unit tests for SSE parsing and reasoning/content delta extraction.
- Add route tests for `stream=true` and `stream=false` behavior.
- Run the `novaic-llm-factory` focused test suite.

## Risks

- OpenAI-compatible vendors emit variant reasoning fields; parser must cover common keys without overfitting.
- Streaming routes cannot know final token usage until terminal chunk; route must handle logging without corrupting final behavior.

## Assumptions

- First streaming support targets OpenAI-compatible providers only.
- Native Anthropic/Google streaming can be explicit unsupported until separately implemented.
