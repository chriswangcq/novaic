# Wire Factory route streaming response

## Problem Definition

The provider can now produce normalized streaming chunks, but `/v1/chat/completions` still ignores `stream=true`. The route must expose a clean streaming response and preserve logging/error behavior.

## Proposed Solution

When `request.stream` is true, route supported providers to `provider.stream_chat()` and return `StreamingResponse` with `text/event-stream` SSE lines containing normalized JSON chunks. Complete or fail Factory logs inside the generator based on streamed terminal metadata. Reject unsupported native providers clearly. Leave the existing blocking route path unchanged.

## Acceptance Criteria

- `stream=true` returns SSE chunks for OpenAI-compatible providers.
- `stream=false` still returns the existing complete JSON response.
- Unsupported native providers return a clear 400 error for streaming.
- Streaming calls update Factory logs with finish reason, token usage when present, latency, and attempt count.

## Verification Plan

- Add route/provider tests for streaming response behavior and unsupported providers.
- Run focused Factory tests.

## Risks

- Streaming generator errors happen after response headers; test-covered provider setup errors should happen before the stream starts where possible.

## Assumptions

- Runtime will consume Factory SSE lines with normalized chunk JSON.
