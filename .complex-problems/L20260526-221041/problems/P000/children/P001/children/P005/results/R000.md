# OpenAI-compatible streaming parser implemented

## Summary

Implemented the Factory provider streaming primitive for OpenAI-compatible chat completions. The provider now exposes `stream_chat()` and normalizes SSE chunks into reasoning/content/tool-call/final metadata deltas while leaving the blocking `chat()` path unchanged.

## Done

- Added `OpenAIProvider.stream_chat()` using `httpx.Client.stream()` with `stream=true` payloads.
- Added SSE parsing for `data: {...}` lines and `[DONE]` termination.
- Added normalized chunk fields: `content_delta`, `reasoning_delta`, `tool_calls_delta`, `finish_reason`, `usage`, plus raw event preservation.
- Added tests for content, reasoning aliases, tool-call deltas, terminal chunks, and non-streaming regression coverage through the existing suite.

## Verification
