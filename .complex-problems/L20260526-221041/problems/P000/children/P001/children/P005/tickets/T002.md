# Add OpenAI streaming parser and provider iterator

## Problem Definition

`OpenAIProvider` only supports blocking chat completions. Factory needs a bounded streaming primitive that can parse OpenAI-compatible SSE chunks and expose normalized delta dictionaries to the route and Runtime.

## Proposed Solution

Add a `StreamEvent` contract and an `OpenAIProvider.stream_chat()` iterator that posts with `stream=true`, parses SSE `data:` lines, yields normalized chunks, extracts content/reasoning/tool-call deltas, forwards finish reason/usage when present, and raises existing `ProviderError` for transport/API errors.

## Acceptance Criteria

- OpenAI-compatible streaming parser handles content delta chunks.
- Parser handles reasoning delta aliases.
- Parser handles tool call delta chunks without dropping ids/function names/arguments.
- Parser ignores `[DONE]` after yielding prior chunks.
- Non-streaming `chat()` remains unchanged.

## Verification Plan

- Add focused unit tests in `novaic-llm-factory/tests/test_chat_routes.py` or a nearby provider test.
- Run the focused Factory tests after implementation.

## Risks

- Provider-specific variants may add fields later; the parser should be small and extensible.

## Assumptions

- The streaming route can consume normalized chunks and decide wire formatting separately.
