# Promote prod into immutable controller state

## Problem Definition

Prod currently runs Dockerized services, but release-controller state has no prod current or previous pointer. Without those pointers, prod deployment is outside the clean promote/rollback control plane.

## Proposed Solution

First inspect current prod images and state. If current prod images can be represented as immutable digests, bootstrap a prod baseline pointer by promoting the currently running prod image digests. Then promote the staging-verified `4e52e7ea` API and Factory sha-tag images to prod through `/v1/promotions/prod`. Verify prod current and previous pointers, container image refs, health endpoints, and dry-run rollback behavior.

## Acceptance Criteria

- Prod current pointer exists after controller-managed promotion.
- Prod previous pointer exists and points at the bootstrap prod baseline.
- Prod current images are immutable refs for `sha-4e52e7ea8e2f`.
- Prod containers run the current pointer image refs.
- `/v1/rollbacks/prod` with `dry_run=true` succeeds, skips execution, and leaves current pointer unchanged.
- Health checks pass for controller, prod API, prod Factory, and prod ingress behavior is documented.

## Verification Plan

- Query release-controller `/v1/status` before and after promotions.
- Inspect Docker image digests/tags and running prod containers on the API host.
- Call `/v1/promotions/prod` for baseline and current promotion with immutable refs.
- Call `/v1/rollbacks/prod` with `dry_run=true`.
- Curl local prod API/factory and public prod API endpoints.
- Validate/render/status the ledger.

## Risks

- If current prod was deployed from mutable tag without a repo digest, bootstrap baseline may need a temporary immutable tag or direct state seed.
- First prod deploy under controller may restart prod services; health gates must pass before closure.
- Public `api.gradievo.com/api/health` may require authentication and return 401 while local health is healthy.

## Assumptions

- The staging `4e52e7ea` release is acceptable to promote to prod.
- User wants the correct controller-managed path rather than preserving the mutable prod tag runtime.
