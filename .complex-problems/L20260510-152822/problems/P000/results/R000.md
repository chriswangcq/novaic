# Cortex DFS context audit result

## Summary

Audited the active novaic-cortex LLM context assembly path and fixed one confirmed structural bug.

## Active Path

- `POST /v1/context/prepare_for_llm` constructs `ContextEngine` from the agent-root scope path.
- `ContextEngine.prepare_messages_for_llm()` reads root `context.jsonl`, builds a DFS `StepTree` from `steps/_index.jsonl`, merges tool results/scope folds, hoists primary system messages, and applies budget compaction.
- `StepTreeBuilder` recursively reads child scope meta/context/summary according to scope phase.
- Closed non-empty scopes render as one `[Skill ... completed]` system fold.
- Blank closed/structural scopes recurse to child summaries instead of rendering `(no report)`.

## Confirmed Issue Fixed

Before the patch, if a parent scope had multiple unanchored open child scopes, DFS rendering expanded all of them. The authoritative control stack already chose the newest open child as active, but the rendered messages still included stale sibling context. This could leak abandoned old wake context into a new wake after recovery or queue/session confusion.

Fix:

- Added `_render_unanchored_scope_nodes()` in `novaic_cortex/context_stack/engine.py`.
- For unanchored sibling scopes, only the newest open sibling is expanded.
- Older open siblings are logged and suppressed until recovery/finalize archives them.
- Strengthened `test_pr234_control_stack.py` to assert stale open sibling messages are not included in prepared LLM messages.

## Residual Risks

- Code comments still reference deleted design documents (`docs/context-assembly-dfs-step-tree.md`, `docs/context-stack-v2-passive-design.md`). This is documentation residue, not runtime behavior.
- Anchored malformed histories, e.g. a `skill_begin` inside an open scope with no matching `skill_end`, still fall through or skip based on current merge rules. This is acceptable for valid LIFO histories but remains a recovery-hardening area.
- Budget compaction is downstream of DFS assembly and was not redesigned in this ticket.

## Verification

- Reproduced old open-sibling leak with a minimal script: before patch messages were `["stale input", "current input"]`; after patch messages are `["current input"]`.
- Focused tests: `32 passed`.
- Full `novaic-cortex` tests: `355 passed`.
