# Audit active Cortex DFS context assembly path

## Problem Definition

Inspect novaic-cortex LLM context assembly from workspace scopes/steps to final messages, with special focus on DFS traversal correctness, scope folding, child ordering, and whether current behavior matches the intended Cortex skill tree model.

## Proposed Solution

- Read `context_stack` implementation and related `workspace` context APIs.
- Map the active traversal path: entrypoint, tree build, merge, render, budget.
- Check key invariants: open path expansion, archived scope folding, blank structural scope behavior, child ordering, unconsumed child handling, and tool output projection.
- Read focused tests covering these invariants.
- Run focused test set and report failures if any.
- Produce audit findings and recommendations, separated into confirmed bugs and residual risks.

## Acceptance Criteria

- Active code path is identified with file/function pointers.
- DFS/fold/ordering behavior is explained from implementation evidence.
- Test coverage is mapped to the invariants and focused tests are executed.
- Any confirmed issue is listed with severity and pointer; if none, residual risks are explicitly named.

## Verification Plan

- Optionally run full novaic-cortex tests if focused tests expose no failure and runtime is acceptable.
