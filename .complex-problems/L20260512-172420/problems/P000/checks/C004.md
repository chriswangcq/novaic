# P000 Check: Shell Terminal Contract and Display Media Boundary

## Success Criteria Judgment

The root problem is satisfied:

- The shell LLM path is terminal-style receipt text with bounded stdout/stderr previews and diagnostics, not raw base64 or raw shell result JSON.
- Full raw shell output is still available through Cortex step payload storage for RO/payload inspection, keeping observability without stuffing context.
- Display remains the only tool allowed to preserve media `_mcp_content`; this prevents non-display CLI/tool results from smuggling image data into the next LLM context.
- Cortex projection uses only the LLM content side of durable tool-step payloads and does not re-inject raw shell payload.
- Tests explicitly cover the failure mode from the screenshot: display media is preserved structurally until multimodal conversion, and historical/tool text paths do not carry base64 image data as text.
- Deployment and smoke checks passed after the cleaned commit.
