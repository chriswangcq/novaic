# Display Media Check

## Summary

The display media child problem is solved.

## Evidence

- `_ok(..., tool_name="display")` preserves `_mcp_content` image items.
- Cortex projection turns preserved display media into `display_files`.
- Runtime multimodal conversion emits OpenAI-compatible `image_url`.
- Tool message image item is sanitized to placeholder and does not contain raw base64.
- Non-display tools remain text-wrapped.

## Criteria Map

- Preserve display media -> regression test asserts `_mcp_content` survives `_ok()`.
- Cortex parses media -> regression test routes through `parse_tool_result()` and `format_for_display_perception_llm()`.
- Runtime emits `image_url` -> regression test asserts exact `data:image/jpeg;base64,...` in the separate user content part.
- Tool text has no base64 -> regression test asserts base64 is absent from processed tool content.
- History/non-display safe -> existing and new tests cover history and non-display behavior.

## Execution Map

## Stress Test

## Residual Risk
