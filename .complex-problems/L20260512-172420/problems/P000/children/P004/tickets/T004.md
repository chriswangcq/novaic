# Final Checks Commit Deploy Smoke

## Problem Definition

The implementation spans parent ledger files plus changes in runtime and Cortex submodules. It must be tested, committed in the relevant repos, deployed to production via the repository deploy entrypoint, and smoke-verified.

## Proposed Solution

Run the canonical backend test matrix or the strongest feasible subset, review diffs, commit changed submodules and superproject state, deploy backend services, and run post-deploy smoke/status checks.

## Acceptance Criteria

- Relevant changed repos have clean committed states after commit.
- Superproject records updated submodule refs and ledger docs.
- Deployment to production completes without fresh-smoke failures.
- Post-deploy status/smoke confirms runtime/cortex/backend services are alive.
- Any skipped checks are explicitly documented with reason.

## Verification Plan

- Run `scripts/run_all_tests.sh`.
- Run `git diff --stat` and inspect submodule diffs/status.
- Commit changed submodules and the superproject.
- Run `./deploy services` or the appropriate backend target.
- Run `./deploy status` and/or `./deploy fresh-smoke`.

## Risks

- Full test matrix may expose unrelated pre-existing failures.
- Deployment restarts all backend services and must be checked with fresh log boundaries.

## Assumptions

- SSH access and deploy scripts are configured on this machine.
