# Remove Raw Shell Payload From LLM-Facing Envelope

## Problem Definition

The current shell tool result includes `diagnostics.raw_shell_result`, which duplicates full stdout/stderr inside the LLM-facing `tool-output.v1` envelope. This violates the tightened terminal contract: shell should expose bounded terminal text and small metadata only, while complete output stays in Cortex step/payload storage.

## Proposed Solution

Update `_shell_result_text()` / `_shell_result_output()` so diagnostics retain only explicit bounded metadata: exit code, stdout/stderr char counts, truncation flags, and files changed count. Keep terminal text preview behavior and nonzero-exit error behavior.

## Acceptance Criteria

- `raw_shell_result` no longer appears in shell tool output diagnostics.
- Large stdout/stderr remains truncated in `content["text"]`.
- Diagnostics still expose `stdout_chars`, `stderr_chars`, `stdout_truncated`, `stderr_truncated`, `files_changed_count`, and `exit_code`.
- Existing shell tests are updated to assert the new contract.

## Verification Plan

- Run `pytest novaic-agent-runtime/tests/unit/task_queue/test_shell_output_contract.py`.
- Inspect focused diff to confirm only shell diagnostics contract changed.

## Risks

- Internal callers/tests that relied on diagnostics raw output may need updates.

## Assumptions

- Cortex step payload storage remains the authority for complete raw output.
