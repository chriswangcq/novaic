# Manual-trigger cutover success check

## Summary
The original problem is solved: backend/factory CI/CD no longer starts from autonomous Release Controller branch polling. The live service is disabled for background polling, the new code rejects attempts to re-enable it, diagnostic polling cannot perform real releases, and explicit `/v1/triggers` remains the staging entrypoint.

- Live `/v1/polls/once` with `dry_run=true` returns a planned diagnostic result but leaves `branch_heads={}`.
- Live `/v1/triggers` dry-run returns `trigger=manual`, `namespace=staging`, `succeeded=true`.
- The deployed Release Controller container is healthy on image digest `sha256:71638fd91a55890ad708e47cdfb5bdee96da8a110bed2164210d9e077190a604`.
- Local test suites passed: 52 Release Controller tests, 9 parent CI guard tests, and immutable workflow lint.
