# Make current startup Postgres-only

## Problem Definition

The root Linux/cloud startup script still passes server SQLite database files for Entangled and Cortex and does not pass Postgres DSN files for those services. Gateway also starts without Postgres backend flags.

## Proposed Solution

Patch `scripts/start.sh` so migrated server services start with explicit Postgres backend flags and DSN files under `/opt/novaic/secrets`. Add shell variables for the current Postgres DSN file paths, remove server SQLite file arguments, and preserve the existing unrelated script edits. Update any current deploy runbook line that still describes a SQLite gateway.db operation when it is part of current docs.

## Acceptance Criteria

- `scripts/start.sh` no longer contains `entangled.db` or `operational.sqlite3` server launch arguments.
- Entangled launches with `--db-backend postgres --postgres-dsn-file`.
- Gateway launches with `--db-backend postgres --postgres-dsn-file`.
- Cortex launches with `--operational-backend postgres --operational-postgres-dsn-file`.
- A focused scan over current startup/deploy files reports no old server SQLite launch args.

## Verification Plan

- Run `bash -n scripts/start.sh`.
- Run `rg` against `scripts/start.sh` and current deploy docs for `.db`/SQLite launch residue.
- Inspect `git diff -- scripts/start.sh docs/runbooks/deploy.md` to ensure the change is scoped.

## Risks

- The exact production DSN filenames must match deployed secret paths; choose stable service-specific filenames under `/opt/novaic/secrets` and align later deploy if production differs.

## Assumptions

- `/opt/novaic/secrets/*-postgres-dsn` is the intended family of secret files used by the Docker/API machine layout.
