# Gateway and Device SQLite current paths removed

## Summary

Gateway and Device no longer expose current executable server SQLite utilities. Gateway runtime is Postgres-only, old `gateway.db` admin/migration scripts are gone, and Device local DB code has been deleted.

## Done

- Removed Gateway `--db-backend` CLI selection and `gateway.db` file assignment from `main_gateway.py`.
- Simplified `gateway/db/access.py` to initialize `PostgresGatewayDatabase` only via `--postgres-dsn-file`.
- Removed common SQLite DB exports from `gateway/db/__init__.py`.
- Deleted `novaic-gateway/scripts/migrate_gateway_sqlite_to_postgres.py`.
- Removed `aiosqlite` and packaged `sqlite3` hidden imports from Gateway packaging/dependencies.
- Deleted root SQLite utility scripts `scripts/gateway/add_agent_a2.py` and `scripts/gateway/admin_reset_user_password.py`.
- Removed `deploy reset-password`, which still pushed a SQLite `gateway.db` mutation script to the API host.
- Deleted `novaic-device/device/db_access.py` and rewrote Device comments away from local SQLite ownership.

## Verification

- `python -m py_compile main_gateway.py gateway/db/access.py gateway/db/__init__.py gateway/db/schema.py gateway/entity/defs_users.py gateway/infra/auth.py` passed.
- Gateway focused tests passed: `pytest tests/test_gateway_postgres_storage.py tests/test_pr141_no_db_access_alias.py` (`5 passed`).
- `python -m py_compile main_device.py device/config_agents_db.py device/config_agents.py device/ssh_keys.py` passed.
- `bash -n deploy` and `bash -n scripts/start.sh` passed.
- Focused residue scan over current Gateway/Device runtime and deploy script paths found no old SQLite script or DB-file references; remaining matches are Python module namespace imports such as `gateway.db`, not `gateway.db` files.

## Known Gaps

- `novaic-common/config/services.json` still contains the now-unused `database.gateway_db_file` config key. That is outside Gateway/Device submodule runtime code and should be removed in the global residue guard/config cleanup phase.
- Gateway schema still has `init_schema_sync` for old tests/import compatibility inside the module; no current runtime imports it after this patch.
