## Stress Test

- Duplicate task/saga claim: artifact requires `FOR UPDATE SKIP LOCKED` on state authority rows so one worker wins and the other skips or finds another row.
- Recovery racing completion/heartbeat: artifact requires both flows to lock state and lease rows, revalidate state after lock, and make the loser return false/no-match.
- Cancel racing claim: artifact defines row-lock ordering and skip/revalidate behavior so cancel and claim cannot both own the same pending row.
- Lease uniqueness conflict: artifact requires unique `(machine_type, machine_id)` plus transaction retry or false return after re-read.
- Saga step-result racing completion: artifact requires saga state row lock to serialize JSON state updates and final completion.
