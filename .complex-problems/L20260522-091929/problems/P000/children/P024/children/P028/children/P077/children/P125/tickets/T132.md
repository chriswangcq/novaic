# Restart Queue Stack From Clean Postgres Cutover Runtime

## Problem Definition

Production Queue services are frozen after migration. They must restart from the clean cutover runtime in Postgres mode, use the production Queue Postgres credential file, and avoid reopening `/opt/novaic/data/queue.db`.

## Proposed Solution

Start Queue Service from `/opt/novaic/services/novaic-agent-runtime-pg` with `NOVAIC_QUEUE_DB_BACKEND=postgres` and the redacted Queue credential file, then restart Queue-dependent task workers, saga workers, session/saga outbox workers, health, scheduler, business subscriber, and gateway ingress using the same service URLs and production ports as pre-freeze. Save PID/log/startup evidence, run `/health` and `/ready`, and confirm no process holds the old SQLite file.

## Acceptance Criteria

- Queue Service is running on `127.0.0.1:19997` with `database_backend=postgres` in health output.
- Queue Service process command/env evidence shows Postgres backend and credential-file usage without exposing credential values.
- Task workers, saga workers, session outbox worker, saga outbox worker, health, scheduler, business subscriber, and gateway ingress are restarted or explicitly justified if omitted.
- `/health` and `/ready` pass after restart.
- `lsof /opt/novaic/data/queue.db` shows no holders after restart.
- A redacted restart report is saved under ledger artifacts.

## Verification Plan

Launch processes with nohup into existing log directories, wait for Queue Service health and readiness, inventory PIDs and command lines with credential redaction, run lsof on the old SQLite file, and save local/remote reports.

## Risks

- Some service CLIs may still default to SQLite unless environment variables are explicitly set.
- Gateway/business subscriber restart may require existing non-Queue Postgres credential paths and tokens.
- Outbox workers directly access Queue DB and must receive the Queue Postgres environment.

## Assumptions

- Clean runtime `/opt/novaic/services/novaic-agent-runtime-pg` at commit `0517c37ae0c3368ec9c2a87fd6270f3e28603289` is the intended Queue runtime for restart.
- Production service ports and URLs from preflight remain valid.
- Operator approval covers restart actions without another prompt.
