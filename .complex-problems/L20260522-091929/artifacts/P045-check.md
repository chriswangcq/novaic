## Criteria Map

- `entangled_sync_versions` Postgres DDL uses `state_key text primary key` and `version bigint`: satisfied by `sync_versions_create_table_sql`.
- Version persistence uses monotonic upsert so older versions cannot overwrite newer versions: satisfied by Postgres `GREATEST(existing, excluded)` upsert and covered by support-table tests.
- `subagent_state_transitions` Postgres DDL uses a generated identity `bigint` ID and preserves append-only columns: satisfied by backend-aware transition DDL and row-shape tests.
- Transition update plus history insert remains atomic under the existing transaction boundary: satisfied because `transition(...)` still performs status update and `append_subagent_transition(...)` inside one global transaction.
- Migration expectations for resetting transition identity above migrated max ID are documented or implemented in support helpers: satisfied by `reset_subagent_transition_identity_sql(max_id)`.
- Tests cover monotonic version upsert, rollback on failed transition, and row-shape compatibility: satisfied by support-table tests plus existing subagent-state error-path tests that assert failed transitions do not mutate status or emit history rows.
- Existing SQLite support-table behavior remains passing: satisfied by focused SQLite assertions and the full 105-test Entangled suite.

## Stress Test

- Plausible failure mode: an older sync snapshot bump arrives after a newer one and regresses `entangled_sync_versions.version`, causing missed or repeated deltas. The Postgres upsert now uses `GREATEST(...)`, so the database-level write path rejects that regression even if callers race or replay older versions.
- Plausible migration failure mode: imported transition IDs collide with new Postgres identity values. The reset helper emits `ALTER TABLE ... RESTART WITH max_id + 1`, giving migration tooling an explicit post-import sequence step.

- SQLite compatibility remains covered by existing tests; maintaining both backends is temporary until the broader Entangled cutover completes.
