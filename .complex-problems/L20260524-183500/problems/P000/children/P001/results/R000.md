# Controller-only deploy guard implemented

## Summary

Implemented the backend/factory Release Controller invocation contract in code and removed the reachable manual backend deployment paths from `deploy`. The code now treats `deploy services-image` and `deploy factory-image` as guarded internal executor steps, and obsolete backend/factory manual targets fail immediately.

## Done

- Planner now attaches `NOVAIC_DEPLOY_CALLER=release-controller`, run id, namespace, and commit env metadata to backend and Factory deploy steps for branch releases, prod promotion, and rollback.
- Runner now merges step env with the inherited process environment instead of replacing it.
- `deploy` now rejects direct manual `services-image` / `factory-image` calls without controller identity before remote side effects.
- `deploy` disables old backend manual paths: `services`, `api-backend`, `services-legacy`, legacy single-service targets, `factory`, and `all`.
- Removed old GitHub backend release workflows and converted the immutable workflow lint into a guard that prevents those workflows from returning.
- Updated CI guard scripts and docs to describe Release Controller as the backend/factory release interface.
