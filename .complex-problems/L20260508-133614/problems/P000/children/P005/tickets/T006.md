# CI full matrix 与架构守卫门禁接入

## Problem Definition
Standalone FSM/worker/cleanup guards must be part of canonical CI/full-matrix execution so architecture drift cannot pass when only standard checks run.

## Proposed Solution
Wire the retired runtime vocabulary guard into GitHub lint and the canonical local test matrix; keep architecture guard commands explicit and deterministic.

## Acceptance Criteria
- lint.yml runs lint_retired_runtime_vocabulary; run_all_tests runs root architecture guards before package pytest suites; no soft-fail or compatibility bypass is introduced.

## Verification Plan
Run syntax checks for changed CI files, direct guard commands, and the canonical root guard subset.

## Risks
- Full package matrix can be expensive; reserve complete package verification for P006 if needed.

## Assumptions
- Existing lint workflow provides Python and ripgrep through its current setup.
