## Stress Test

- Failure mode: treating Redis as authority. Plan forbids this; Redis is lease/coordination and must be checked by SQLite generation/fencing.
- Failure mode: replacing files with one opaque DB. Plan avoids this; LogicalFS/Workspace remains file authority, SQLite owns operational ledger/projections.
- Failure mode: Blob becomes semantic truth. Plan forbids this; Blob stores bytes, manifests store meaning.
- Failure mode: Cortex restart loses state. Plan forbids process memory authority and requires restart from SQLite + LogicalFS + Blob.
- Failure mode: snapshot/patch shell crash loses writes. Live LogicalFS target requires per-operation durability.

## Residual Risk

This is a design plan, not implementation. The next work should be a phased implementation ledger with code changes, tests, cleanup, deployment, and smoke tests.
