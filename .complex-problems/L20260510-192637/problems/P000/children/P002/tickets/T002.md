# Design Event Backed Active Stack Status

## Problem Definition

Active stack/status still walks projection files. This is a control-plane projection, but the target model should avoid split-brain between event state and file projection state.

## Proposed Solution

Move active stack/status authority to a SQLite-backed operational ledger and projection, while keeping Workspace files as trace/export materialized views.

## Acceptance Criteria

- Define new authoritative tables/events.
- Define read/write path changes for skill_begin, skill_end, finalize, status.
- Define migration and deletion of old projection-walk active path.
- Include tests for unclosed child skill, close-one-skill, close-all/finalize, recovery, and generation mismatch.

## Verification Plan

Check that every previous `_collect_active_stack` responsibility is covered by the new model and that old projection walking becomes non-authoritative or is removed.

## Risks

- Dual-write between SQLite and Workspace can create divergence if not transactionally ordered.
- Cutting too aggressively without replay/export can break debugging.

## Assumptions

- SQLite can be the operational authority for active stack state.
- Workspace projection can remain as human-readable trace, not as runtime authority.
