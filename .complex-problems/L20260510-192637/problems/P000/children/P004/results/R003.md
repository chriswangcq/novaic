# Blob Payload Authority Contract Plan

## Summary

Keep Blob as the raw-byte authority, but make semantic ownership explicit through a payload manifest stored in SQLite and/or Workspace.

## Done

- Blob owns:
  - Raw payload bytes.
  - Binary artifacts.
  - Upload/download serving.
- Blob does not own:
  - Which tool step produced the payload.
  - Whether a payload is part of canonical context.
  - Retention/lifecycle semantics.
  - User/agent/workflow meaning beyond object metadata.
- Proposed manifest:
  - `payload_manifest(payload_ref, root_scope_id, scope_id, step_ref, blob_ref, mime_type, size_bytes, sha256, status, created_at, retention_class, semantic_owner, error_json)`.
  - Stored in SQLite for query/recovery; optionally mirrored in Workspace for human trace.
- Lifecycle:
  - Upload bytes to Blob.
  - Record manifest transactionally with tool step/context event.
  - Fetch verifies size/hash when practical.
  - Missing blob produces explicit `payload_missing` observation, not silent empty content.
  - Retention sweeper reads manifest, then deletes Blob bytes.
- Tests:
  - Externalization threshold stores manifest.
  - Missing Blob returns structured error.
  - Hash mismatch detected.
  - Deleting a scope removes or expires related payload manifests.
  - Display/download paths go through Blob, not LogicalFS shortcuts.

## Verification

- Preserves Blob's role as cheap byte service while preventing semantic drift into Blob.
