# Process Cache Config And Documentation Cleanup Plan

## Summary

Keep process-local state only when it is a rebuildable cache, config value, or client wiring. Clean comments and tests so no future change treats it as authority.

## Done

- Allowed process-local state:
  - `WorkspaceRegistry` cache of adapters/workspace objects.
  - Service clients/factories such as sandboxd client factory.
  - Startup config values already supplied through CLI/env boundary.
  - Redis lock manager object as a client to the coordination service.
- Not allowed:
  - Current active stack only in memory.
  - Pending inbox only in memory.
  - Unflushed semantic events only in memory.
  - Process-local fallback storage if SQLite/LogicalFS/Blob is unavailable.
- Cleanup targets:
  - Update `scope_locks.py` top comment that still says "Today the implementation is asyncio.Lock".
  - Add architecture docs naming Redis as lease plane, SQLite as operational state plane, LogicalFS as file authority, Blob as byte authority.
  - Remove any fallback language implying local `/tmp/novaic-cortex-sandbox-*` is stable.
  - Review registry docs to state caches are not authority and restart is safe.
- Guardrails:
  - Unit tests instantiate repositories with explicit clock/id/config.
  - Architecture grep/lint for `os.getenv` below boundary modules where practical.
  - Runtime readiness endpoint reports storage backends and whether authority stores are reachable.
  - Crash/restart integration test verifies no process cache is needed for recovery.
