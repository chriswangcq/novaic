# Design Process Cache Config And Documentation Cleanup

## Problem Definition

Process-local caches and stale comments are not necessarily wrong, but they can mislead future agents into thinking Cortex owns durable state or still uses old in-process lock assumptions.

## Proposed Solution

Define allowed process-local state and cleanup residues that confuse the authority model.

## Acceptance Criteria

- Classify allowed caches/config/globals.
- Define explicit dependency boundary requirements.
- List cleanup targets.
- Define tests/lints that prevent new hidden state.

## Verification Plan

Check that every process-local object is restart-rebuildable and every hidden input is either injected or documented at service boundary.

## Risks

- Removing harmless caches can hurt performance with no correctness gain.
- Leaving stale comments can waste future engineering time.

## Assumptions

- Process memory is allowed for cache/config/client objects only.
