# CLI blob contract inventory result

## Summary

The active shell CLI surface is generated by `novaic-cortex/novaic_cortex/shell_capabilities.py` and consists of `agentctl`, `cortex`, and `devicectl`. The confirmed live blob-contract violation is `devicectl hd screenshot`, with `devicectl hd file-pull` also risky because the upstream HD tool proxy returns base64 file data. `agentctl` and `cortex payload` mostly operate on text/control-plane or bounded payload inspection and do not directly emit binary artifacts.

## Done

- Inspected capability script generation in `novaic-cortex/novaic_cortex/shell_capabilities.py`.
- Inventoried command surfaces:
  - `agentctl im read/reply/send/history/search/context`
  - `agentctl subagent spawn`
  - `agentctl media audio-qa`
  - `devicectl hd screenshot/mouse/keyboard/shell-exec/clipboard-get/clipboard-set/file-pull/file-push`
  - `cortex payload read/search/summarize/qa`
- Inspected upstream HD proxy implementation in `novaic-app/src-tauri/vmcontrol/src/api/routes/hd_tools.rs`.
- Identified confirmed/likely risk surfaces:
  - `devicectl hd screenshot`: upstream returns `{success, screenshot: base64, width, height, format}`.
  - `devicectl hd file-pull`: upstream returns `{success, data: base64, is_base64: true, size}`.
  - `devicectl hd shell-exec`: returns text stdout/stderr and already truncates long stdout at the HD service boundary; it is a text command, not an artifact command.
  - `agentctl media audio-qa`: consumes `blob://` audio and sends base64 only inside the Factory request, then returns text answer; no user-visible artifact violation.
  - `cortex payload read/search/summarize/qa`: bounded payload inspection; does not itself unwrap arbitrary blobs into shell stdout beyond explicit limits.

## Verification

- `rg` and `sed` inspection of `shell_capabilities.py`, related tests, and `hd_tools.rs`.
- Existing tests already assert `tool-output.v1` artifact projection expectations in `novaic-cortex/tests/test_tool_output_projection.py` and `novaic-agent-runtime/tests/unit/task_queue/test_tool_output_contract.py`, but the live `devicectl` CLI does not yet produce that contract.

## Known Gaps

- `devicectl hd screenshot` still needs repair to upload decoded screenshot bytes to Blob Service and print a bounded `tool-output.v1` artifact manifest.
- `devicectl hd file-pull` should be repaired similarly for pulled file bytes.
- Tests need to cover the shell CLI wrapper behavior, not only downstream projection helpers.
