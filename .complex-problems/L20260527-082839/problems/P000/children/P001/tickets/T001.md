# Discover controller contract through docs/config and live status

## Problem Definition

Release must be driven through the centralized Release Controller. We need exact local configuration and live controller status before changing remote state.

## Proposed Solution

Split discovery into local contract inspection and live controller probing. Local inspection should read docs/config/scripts to identify endpoint, trigger API, branch rules, namespaces, and health checks. Live probing should call the controller read APIs to confirm reachability and current runs.

## Acceptance Criteria

- Local docs/config identify the intended trigger endpoint, branch/namespace policy, deploy steps, and health URLs.
- Live controller status/runs are reachable or the blocker is explicit.
- The final discovery result gives exact trigger parameters for later deployment.

## Verification Plan

Use `sed`, `rg`, and HTTP calls against configured controller endpoints. Record file paths and API responses in child results.

## Risks

- Config may use host-only URLs or remote paths; avoid guessing and use the checked-in controller config or service unit where possible.

## Assumptions

- Release Controller code/config under `novaic-release-controller`, `docker/release-controller`, and `docs/*` is authoritative unless live status contradicts it.
