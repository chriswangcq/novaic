# Read local Release Controller docs and config

## Problem Definition

Deployment trigger parameters need to be derived from checked-in docs/config/source.

## Proposed Solution

Read Release Controller docs, compose/config files, service source, planner, and deployment runbooks to extract the trigger API, namespace policy, deploy commands, health URLs, dry-run behavior, and prod promotion policy.

## Acceptance Criteria

- Local controller contract is summarized with file evidence.
- Trigger and promotion API shapes are identified.
- Namespace/branch/health/deploy-script values are identified or marked absent.

## Verification Plan

Use source reads and `rg` against the release-controller and deploy docs.

## Risks

- Some secrets/host URLs may be configured only on the server, so local discovery may need the live probe child for final endpoint confirmation.

## Assumptions

- Checked-in docs/source are trustworthy for controller behavior.
