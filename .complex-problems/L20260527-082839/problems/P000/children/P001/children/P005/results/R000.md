# Local Release Controller contract identified

## Summary

Read local release-controller architecture, deploy runbook, source, and sample config. The deployment contract is explicit: branch releases use `/v1/triggers` with `branch` and `commit`, `main` maps to `staging`, prod is promotion-only, and `deploy services-image` / `deploy factory-image` are controller internal executor steps.

## Done

- Identified controller local API: `http://127.0.0.1:19880` on the API host.
- Identified public read-only dashboard: `https://api.gradievo.com/cicd/dashboard`; write APIs are not public through Nginx.
- Identified staging trigger: `POST /v1/triggers` body `{"branch":"main","commit":"<sha>"}`; omit `dry_run` for real release.
- Identified prod promotion: `POST /v1/promotions/prod` with immutable `api_image`, `factory_image`, `commit`, and optional `promoted_from`.
- Identified branch rules in `novaic-release-controller/config.sample.json`: `main -> auto_deploy staging`, `preview/* -> auto_deploy preview-{slug}`, `release/* -> candidate_only`.
- Identified deploy commands planned by controller: build/push API and Factory images, then `deploy services-image <namespace> <api-image>` and `deploy factory-image <namespace> <factory-image>` with Release Controller env identity.
- Identified health URLs in sample config: staging `https://staging-api.gradievo.com/api/health`, prod `https://api.gradievo.com/health`.
