# Probe live controller read APIs

## Problem Definition

The deployment needs the live controller base URL/status/runs before triggering release state changes.

## Proposed Solution

Try public read-only `/cicd` APIs first, then SSH/API-host local curl if needed. Capture status, rules, recent runs, current releases, and controller health.

## Acceptance Criteria

- Live status, rules, and recent runs are captured from the actual controller.
- Current staging/prod pointers are known before triggering.
- Any reachability issue is explicit with command output.

## Verification Plan

Use `curl -fsS` and, if needed, `ssh root@api.gradievo.com 'curl -fsS http://127.0.0.1:19880/...'`.

## Risks

- Public read-only route may not expose `/health`; use the host-local API through SSH for write-trigger confirmation.

## Assumptions

- `root@api.gradievo.com` SSH is configured if public read-only routes are insufficient.
