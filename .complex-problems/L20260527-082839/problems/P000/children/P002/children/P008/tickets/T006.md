# Commit and push touched submodule repositories

## Problem Definition

Subrepo commits must exist on remotes before root records submodule pointers for CI/CD.

## Proposed Solution

Explicitly stage construction-owned files in `novaic-llm-factory`, `novaic-agent-runtime`, and `novaic-app`; commit with concise messages; push each `main` branch; record SHAs.

## Acceptance Criteria

- Factory commit pushed to `origin/main`.
- Runtime commit pushed to `origin/main`.
- App commit pushed to `origin/main`.
- No unrelated files are staged.

## Verification Plan

Use `git status`, `git diff --cached`, `git commit`, `git push`, and `git rev-parse HEAD` in each subrepo.

## Risks

- Remote push can fail if branch moved; use non-destructive pull/rebase only if needed and inspect first.

## Assumptions

- All touched subrepos are on `main` tracking `origin/main`.
