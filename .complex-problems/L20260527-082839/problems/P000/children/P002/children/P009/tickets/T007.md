# Commit root release pointer state

## Problem Definition

The root repository must point to the pushed subrepo commits before Release Controller can build/deploy them from a root commit.

## Proposed Solution

Stage only updated submodule pointers and `.complex-problems` ledger/index artifacts, exclude unrelated `CLAUDE.md`, commit to `main`, push to `origin/main`, and record the root commit SHA for the trigger.

## Acceptance Criteria

- Root commit points `novaic-llm-factory` to `d846b1d9bba721aa126af456cb9f7518398a8779`.
- Root commit points `novaic-agent-runtime` to `d9a7b3d8dc85f72d25aae0d176e20035f51315c6`.
- Root commit records the App pointer and complex-problems ledgers/index.
- `CLAUDE.md` is not staged.
- Root commit is pushed to `origin/main`, and SHA is recorded.

## Verification Plan

Use explicit `git add`, `git diff --cached --stat`, `git diff --cached --submodule=log`, `git status`, `git commit`, `git push`, and `git rev-parse HEAD`.

## Risks

- Additional ledger changes after this commit may remain local; the release trigger should use the pushed root SHA from this step.

## Assumptions

- Committing ledger state before final deployment closure is acceptable because the release images do not copy root `.complex-problems` into the API backend or Factory images.
