# Run focused pre-push checks

## Problem Definition

The commit/push stage needs current test evidence for the touched paths.

## Proposed Solution

Run focused tests in Factory, Runtime, and App, plus capture git status summaries before commits.

## Acceptance Criteria

- Factory focused tests pass.
- Runtime focused tests pass.
- App focused tests pass.
- Dirty file list is known for explicit staging.

## Verification Plan

Run the exact focused test commands and `git status --short` in touched repos and root.

## Risks

- A failure here blocks committing until fixed.

## Assumptions

- These focused tests are sufficient for local pre-push; Release Controller quality gates are authoritative CI after push.
