# File/display architecture design check

## Summary

The requested systemic thinking is satisfied. The result moves beyond the narrow current-round bug fix and defines a cleaner architecture where large artifacts live in file/blob/Cortex storage, LLM context receives bounded manifests, and `display` is the explicit perception tool for intentional visual dereference.

## Evidence

- Code inspection confirmed current user attachments already use a reference-first model and instruct the agent to call `display(file_url=...)` when needed.
- Code inspection confirmed `display` can serve as a budgeted dereference/perception tool for Blob refs.
- Code inspection confirmed current step-result projection can still turn stored `display_files`/data URLs into model-visible image/base64 content.
- Previous backend evidence showed real slowness came from prompt growth to about 189k tokens after screenshot-heavy multi-round execution.
- Result `R000` defines planes, contracts, invariants, migration phases, and risks.

## Criteria Map

- Explain why the previous current-round fix is necessary but insufficient -> `R000` explains it is a guardrail, not the target abstraction.
- Define elegant resource model -> `R000` defines storage plane, artifact manifest plane, LLM context plane, perception/tool plane, and monitor/UI plane.
- Define tool/Cortex/LLM/display interaction -> `R000` defines artifact manifests, current-turn display grants, historical placeholder projection, and explicit `display` dereference.
- Identify invariants, risks, migration phases -> `R000` lists required invariants, known risks, and phases 0-5.
- Avoid immediate code changes -> no implementation files were changed beyond ledger/design artifacts.

## Execution Map

- `T000` classified as `one_go` because the task was a bounded architecture analysis.
- `R000` records the actual design and verification evidence.

## Stress Test

- Failure mode: model never sees screenshots if everything becomes a file ref. Mitigation: current-turn display grants and explicit `display(file_url=...)`.
- Failure mode: monitor UI loses rich visuals. Mitigation: UI projection consumes artifact manifests independently of LLM prompt projection.
- Failure mode: old history still bloats context. Mitigation: historical artifacts render as bounded manifests, never raw base64.
- Failure mode: compatibility paths keep leaking `display_files`. Mitigation: phase 5 removes raw compatibility after migration.

## Residual Risk

- This is a design result, not an implementation.
- Implementation should be tracked as separate tickets before modifying Runtime/Cortex behavior.
