# Add full Runtime display image preparation regression

## Problem Definition

Runtime has helper-level image conversion tests, but not a full `prepare_llm_call` test covering Cortex step-ref expansion plus `system` ordering.

## Proposed Solution

Add a focused unit test that monkeypatches the Cortex bridge to return image `_mcp_content`, calls `prepare_llm_call` with a current-round display tool step, and asserts the provider-ready messages include an inserted `user(image_url)` before the following `system` message.

## Acceptance Criteria

- The test uses `prepare_llm_call`.
- The test uses real `expand_messages_for_llm`, `sanitize_context`, and `process_multimodal_messages`.
- The test proves raw image base64 is removed from the tool text receipt.
- The test proves the inserted user image message is present even with a following system message.

## Verification Plan

- Run the touched Runtime test file.
- Run the broader Runtime image/step-ref test set.

## Risks

- Minimal production risk because this is a regression test first.

## Assumptions

- A fake bridge is sufficient to represent Cortex `read_step_formatted`.
