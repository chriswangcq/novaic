# Redact provider-native image data in Factory log snapshots

## Problem Definition

Factory stores request bodies for debugging. Once Runtime correctly injects provider-native image messages, those logs can contain raw base64 image data, making the UI noisy and hiding whether image delivery actually occurred.

## Proposed Solution

Add a log-only redaction step inside `build_chat_log_snapshot` that preserves message structure and image metadata while replacing base64 payloads with explicit placeholders.

## Acceptance Criteria

- OpenAI `image_url` data URLs are redacted in logs.
- Anthropic `image.source.data` base64 is redacted in logs.
- Redaction does not alter the provider transport request, only the logged snapshot.
- Tests prove raw payload strings are absent and image metadata/markers remain.

## Verification Plan

- Run Factory chat route/unit tests.

## Risks

- Redaction should not remove ordinary text or tool schema data.

## Assumptions

- Factory request logging is built exclusively through `build_chat_log_snapshot`.
