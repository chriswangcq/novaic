# Make empty display perception diagnostic

## Problem Definition

`format_for_display_perception_llm` uses a generic empty fallback that emits `OK`. For a display projection, this can hide that no media was actually available for injection.

## Proposed Solution

Parameterize the MCP content formatter's empty fallback and make `format_for_display_perception_llm` emit a diagnostic display-specific text marker when parsed content has no text and no display media.

## Acceptance Criteria

- Empty display perception output is not `OK`.
- Existing image display projection remains unchanged.
- History/current generic projections are not unnecessarily changed.
- Tests cover empty display projection.

## Verification Plan

- Run Cortex projection tests.

## Risks

- Diagnostic text appears to the model when display had no readable content, but that is preferable to silent success.

## Assumptions

- `OK` can remain for non-display empty projections.
