# Deploy quality-gate controller and verify staging gate execution

## Problem Definition

The quality-gate commit is on `main`, but the API-host Release Controller still runs the previous image and config. The runtime must be upgraded and a staging release must prove that quality gates execute in the intended order before build/deploy.

## Proposed Solution

Build a new Release Controller image from commit `aaf2c823e24e970498479acda9c95ad56f03835a`, push it to the internal registry as an immutable sha tag/digest, update `/opt/novaic/release-controller/config.json` with the default quality gates while preserving existing settings, deploy the new controller image, trigger a staging release for the commit, inspect the run command plan and execution results for quality gate ordering, verify staging health, confirm prod pointer did not change, re-enable polling, and verify manual deploy guards still reject direct calls.

## Acceptance Criteria

- New controller image deployed on API host is built from commit `aaf2c823e24e970498479acda9c95ad56f03835a`.
- Remote config contains `release-controller-ci` and `release-path-lints` quality gates.
- Staging Release Controller run succeeds and includes `quality-release-controller-ci` and `quality-release-path-lints` before build/deploy.
- Quality gate execution results are present and successful in the run output.
- Staging public health is healthy after deployment.
- Prod current pointer remains on commit `e4fb5740cc8481617bcd3d9f2928d7f4ebf586df` unless explicitly promoted.
- Polling is re-enabled with `last_error=null`.
- Direct manual `services-image` and `factory-image` calls still fail locally before remote side effects.

## Verification Plan

Use Docker image inspect, remote config JSON inspection, Release Controller `/v1/triggers`, `/v1/runs/{run_id}`, `/v1/status`, public health curls, prod pointer comparison, and direct guarded deploy commands.

## Risks

- Remote quality gates may fail because the controller image or worktree lacks pytest or repo dependencies.
- If remote config JSON is edited incorrectly, branch rules or smoke URLs could be damaged.
- Re-enabling polling before verifying the staged run could make debugging harder.

## Assumptions

- The API host can build and push the Release Controller image from the pushed commit.
- Staging deployment of the new commit is expected; prod promotion is intentionally not part of this ticket.
