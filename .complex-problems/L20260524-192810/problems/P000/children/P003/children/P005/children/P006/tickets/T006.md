# Add test tooling to Release Controller image

## Problem Definition

The Release Controller image is now expected to run default pytest-based quality gates, but the image does not install pytest. The controller must carry the minimal tooling needed for its configured gate contract.

## Proposed Solution

Update `docker/release-controller/Dockerfile` to install pytest alongside the release-controller package, add a Dockerfile invariant guard requiring pytest installation, run local release-controller tests and CI guard tests, commit and push this small fix while polling remains paused.

## Acceptance Criteria

- Dockerfile installs pytest or an equivalent package set sufficient for `python3 -m pytest`.
- CI guard test asserts the Dockerfile keeps pytest available.
- Local tests pass after the Dockerfile/test update.
- The fix is committed and pushed without unrelated dirty files.

## Verification Plan

Run release-controller tests, `scripts/ci/test_release_controller_ci.py`, immutable workflow lint, root pytest, `bash -n deploy`, and inspect staged files before commit.

## Risks

- Installing test tooling into the controller runtime image makes it slightly larger.
- If future gates require additional dependencies, this fix only covers the current default gate set.

## Assumptions

- A slightly larger controller image is acceptable because the controller is the CI/CD authority and must run its own gates.
