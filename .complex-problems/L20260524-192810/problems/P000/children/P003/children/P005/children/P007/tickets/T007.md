# Finish remote quality-gate rollout through polling path

## Problem Definition

The latest `main` commit contains the quality-gate model and controller image test tooling, but the API-host Release Controller still needs to run that code and execute the gates in a real staging admission flow.

## Proposed Solution

Build and push an immutable Release Controller image from `6a74b2ac08b199a4c2c0db54c3fc31708bd59261`, update API-host runtime config with the default quality gates while keeping polling paused, deploy the new image, execute `/v1/polls/once` so the branch polling path records the new main head and deploys staging, inspect the run plan/results for successful `quality-*` steps before build/deploy, verify staging health and prod pointer stability, then re-enable polling and verify status is clean.

## Acceptance Criteria

- Running API-host controller image is the new digest built from commit `6a74b2ac08b199a4c2c0db54c3fc31708bd59261`.
- Remote config contains the two default quality gates.
- `/v1/polls/once` creates a successful staging run for commit `6a74b2ac08b199a4c2c0db54c3fc31708bd59261`.
- Run command plan and execution results show both quality gates before build and deploy, with exit code 0.
- Staging health passes and prod pointer remains on the previous prod commit.
- Polling is re-enabled with `last_error=null` and unchanged main head skip.
- Direct manual backend/factory deploy commands still fail.

## Verification Plan

Inspect remote worktree, build/push image, deploy image, read remote config, call `/v1/polls/once`, fetch run details, parse command order/results, curl staging/prod health/status, and run local direct deploy guard commands.

## Risks

- The quality gates may fail remotely if the controller image still lacks a tool.
- The staging run may take several minutes because it builds API and Factory images.
- Re-enabling polling before branch head is recorded could duplicate the staging deployment.

## Assumptions

- Only `main` exists as a remote branch, so `/v1/polls/once` will not unexpectedly process preview or release branches.
