# Install httpx for controller CI gate

## Problem Definition

The remote `release-controller-ci` quality gate fails because FastAPI/Starlette `TestClient` requires `httpx`, and the Release Controller image does not install it.

## Proposed Solution

Install `httpx` in the Release Controller Docker image together with pytest, update the Dockerfile invariant guard, run local tests, commit and push the fix, rebuild/redeploy the controller image, and verify the default quality gate command succeeds inside the container before retrying staging rollout.

## Acceptance Criteria

- Dockerfile installs `pytest` and `httpx` for controller CI gate execution.
- CI guard test asserts both dependencies remain in the Dockerfile.
- Local tests pass.
- Commit is pushed to `main` while API-host polling remains disabled.
- The quality gate command succeeds manually inside the upgraded controller container.

## Verification Plan

Run local release-controller tests, CI guard tests, immutable workflow lint, root pytest, remote container manual quality gate command, and remote status polling disabled check.

## Risks

- More test-only dependencies increase controller image size.
- Future tests may introduce additional dependencies if the gate remains whole-suite based.

## Assumptions

- Installing `httpx` is the minimal correct fix because the default gate intentionally runs the controller test suite.
