# First-class quality gate model implemented

## Summary

Implemented the code-level Release Controller quality gate model. Controller config now has typed `quality_gates`; branch release plans render them as named `quality-*` steps after checkout/submodule update and before image builds; promotion and rollback remain immutable image pointer deploy flows; runner failure behavior already blocks subsequent steps and is now covered specifically for failing quality gates.

## Done

- Added `QualityGate` model with stable lowercase slug names and argv validation.
- Added top-level `ControllerConfig.quality_gates` parsing and serialization, defaulting to an empty tuple for backward-compatible existing remote config.
- Added duplicate gate-name rejection and malformed gate tests.
- Updated planner to insert `quality-<name>` command steps in branch release plans before `build-api-backend` and `build-llm-factory`.
- Added tests proving prod promotion and rollback plans do not include quality or build steps.
- Added runner test proving a nonzero `quality-*` step stops later build/deploy steps.
- Exported `QualityGate` from the release-controller package.
