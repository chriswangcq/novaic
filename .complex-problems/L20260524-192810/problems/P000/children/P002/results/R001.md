# CI/CD quality gate flow configured and documented

## Summary

Configured the default Release Controller quality gate contract and documented the canonical CI/CD flow. Sample config now declares named quality gates, preflight remains lightweight, docs distinguish local tests, controller CI gates, staging smoke/integration, and prod promotion, and repository guard tests now pin the contract.

## Done

- Added top-level `quality_gates` to `novaic-release-controller/config.sample.json`.
- Default gates are `release-controller-ci` and `release-path-lints`, both executable in the controller worktree.
- Kept `deploy.verify_commands` as lightweight preflight: `bash -n deploy` and `py_compile docker/api-backend/write_env.py`.
- Updated Release Controller architecture docs with the canonical CI/CD flow and quality gate semantics.
- Updated deploy runbook with the developer flow: local focused unit tests, Release Controller authoritative staging admission, staging smoke/integration, prod promotion from accepted immutable images.
- Added guard assertions in `scripts/ci/test_release_controller_ci.py` and `scripts/ci/lint_immutable_image_workflows.py` to prevent sample/docs drift.
