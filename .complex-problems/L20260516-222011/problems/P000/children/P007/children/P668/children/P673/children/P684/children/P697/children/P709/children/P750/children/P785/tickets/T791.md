# App resource/generated and backend startup remediation ticket

## Problem Definition
The app contains committed resource/generated VMuse copies and backend startup scripts that may diverge from source package cleanup and current service topology, creating stale packaged behavior even after source packages are cleaned.

## Proposed Solution
Split the work into bounded child problems: sync/remove app VMuse resource/generated FastMCP residues, audit and fix backend startup scripts/service port/resource expectations, and update stale HD screenshot-to-LLM comments in VmControl. Each child should verify with targeted scans/tests and avoid creating source/generated divergence.

## Acceptance Criteria
- App VMuse committed copies reflect the source VMuse HTTP contract or are removed if they should not be committed.
- Backend startup scripts and service resource expectations are consistent with current topology and port assignments.
- VmControl HD tools comments match the blob/display contract.
- Focused scans/checks cover source resources, generated assets, scripts, and relevant Rust code.

## Verification Plan
Use `rg` and `diff` against source VMuse/resources/generated copies, inspect backend scripts/configs, patch only active committed files, and run focused shell/Rust/script checks available locally.
