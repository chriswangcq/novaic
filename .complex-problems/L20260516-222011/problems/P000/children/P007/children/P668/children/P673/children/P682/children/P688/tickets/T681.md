# Scan Config and Deployment Launch References

## Problem Definition

Backend topology may be encoded in deployment manifests, service config files, app service manifests, docker/systemd files, environment templates, and worker launch configuration rather than in scripts or Python modules. We need reproducible scan artifacts for those config/deploy launch references.

## Proposed Solution

Generate `.complex-problems/L20260516-222011/tmp/p688/` artifacts from config/deploy file candidates and targeted service/worker launch keyword scans. Keep this read-only and defer role classification to later problems.

## Acceptance Criteria

- Config/deploy candidate files are scanned and saved.
- Launch-related service/worker names are searched with evidence.
- Expected topology domains such as queue, saga, outbox, scheduler, health, Blob, LogicalFS, Sandbox, Cortex, Gateway, Business, and Device are checked.
- No production code is changed.

## Verification Plan

Check artifact line counts, inspect representative head samples, and verify expected topology terms appear in the scan summary or are explicitly missing.

## Risks

- Config scans can include historical docs or examples; classification is deferred.
- Some deployment references may be outside this repository.

## Assumptions

- This is a fifth-level, read-only, artifact-producing scan suitable for one-go after classification.
