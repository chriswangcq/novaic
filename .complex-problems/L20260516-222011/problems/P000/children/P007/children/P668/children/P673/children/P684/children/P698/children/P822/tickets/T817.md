# Classify blob/sandboxd dual entrypoints as intentional wrapper pattern

## Problem Definition

Both `novaic-blob-service` and `novaic-sandbox-service` have two entrypoint files each. Need to determine if this is intentional wrapper pattern or stale duplicate.

## Proposed Solution

Classify as **intentional wrapper pattern**. The `main_*.py` files are thin launchers that handle PyInstaller bundling (`sys._MEIPASS`) and sys.path setup, then delegate to the internal `*/main.py` which defines the actual app. This is a standard pattern for services that need to run both as standalone dev processes and as frozen PyInstaller bundles.

## Acceptance Criteria

- Each dual pair is classified with rationale.
- Wrapper imports internal main (no divergent logic).
- launch scripts reference the correct entrypoint.

## Verification Plan

- Read both wrapper files and confirm they import from internal main.
- Check start.sh for which entrypoint it uses.

## Risks

- none — classification only.

## Assumptions

- The PyInstaller path setup is needed for production packaging.
