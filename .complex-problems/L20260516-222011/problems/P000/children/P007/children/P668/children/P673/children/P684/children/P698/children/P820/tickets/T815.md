# Classify queue_service/main.py as active app module

## Problem Definition

`novaic-agent-runtime/queue_service/main.py` appears residual because launch scripts use `main_novaic.py queue-service`. However, `main_novaic.py:118` does `from queue_service.main import app` — the file is the actual FastAPI application module. The `if __name__` block is for standalone development use.

## Proposed Solution

Classify `queue_service/main.py` as **active** — it is the app module imported by the CLI launcher. No deletion needed. Record the classification as evidence.

## Acceptance Criteria

- queue_service/main.py is classified as active with rationale (imported by main_novaic.py).
- The import chain is verified: main_novaic.py -> queue_service.main.app -> FastAPI app.

## Verification Plan

- Verify `main_novaic.py` line 118 imports from `queue_service.main`.
- Confirm no other standalone entrypoint duplicates the app creation.

## Risks

- none — this is a classification, not a code change.

## Assumptions

- main_novaic.py is the authoritative CLI launcher used by start.sh.
