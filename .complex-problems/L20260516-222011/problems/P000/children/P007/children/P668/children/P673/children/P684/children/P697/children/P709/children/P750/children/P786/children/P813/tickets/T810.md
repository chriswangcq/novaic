# Delete unused SmartValue component

## Problem Definition

`novaic-app/src/components/Visual/SmartValue.tsx` exports `SmartValue`, `ImagePreview`, `CollapsibleText`, and `JsonTree`. None of these are imported by any other file in the codebase. The component is dead code — a dormant raw JSON renderer with `JSON.stringify` fallback behavior that creates misleading residue and copy/paste risk.

## Proposed Solution

Physically delete `SmartValue.tsx`. Verify no barrel re-export or dynamic import references exist. Confirm TypeScript compilation passes after deletion.

## Acceptance Criteria

- `SmartValue.tsx` is deleted from disk.
- No remaining import, re-export, or reference to `SmartValue`, `ImagePreview` (from SmartValue), `CollapsibleText`, or `JsonTree` (from SmartValue) exists anywhere in `novaic-app/src`.
- `tsc --noEmit` or the project build succeeds without errors related to the deletion.

## Verification Plan

- `grep -r` for `SmartValue`, `ImagePreview` (excluding `ImagePreviewOverlay`), `CollapsibleText`, `JsonTree` across `novaic-app/src` after deletion confirms zero hits from the deleted module.
- TypeScript check passes.

## Risks

- A lazy or dynamic import could reference SmartValue at runtime without appearing in static grep. Mitigated by also checking barrel/index files.

## Assumptions

- The grep audit showing zero external imports is accurate and complete.
- No code generation or build step synthesizes SmartValue imports.
