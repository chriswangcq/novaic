# Shell artifact manifest output contract discovery ticket

## Problem Definition

We need to verify the active shell contract for screenshot/artifact-producing CLI commands, especially `devicectl hd screenshot`, and identify whether any active shell path still emits raw screenshot/base64 bytes instead of terminal text plus artifact manifests.

## Proposed Solution

Inspect Cortex shell capability wrappers, shell output projection helpers, runtime shell-output tests, and docs. Record exact file pointers for the active command wrapper, output schema, truncation behavior, and tests that reject raw media fields in stdout. List any active violations for remediation.

## Acceptance Criteria

- Active `devicectl hd screenshot` shell wrapper is identified with evidence.
- Shell stdout artifact manifest structure is identified with evidence.
- Tests/docs that enforce no raw screenshot/base64 stdout are identified.
- Any active shell output violation is listed, or absence of active violations is justified with evidence.

## Verification Plan

Run targeted inspections for `_wrap_hd_screenshot`, `_HD_PROXY_PATHS`, `tool-output.v1`, `screenshot`, `base64`, `runtime-artifact`, and shell output contract tests in `novaic-cortex`, `novaic-agent-runtime`, and docs.

## Risks

- Command wrapping may be generated dynamically, so grep must include registration tables and tests.
- Old archived outputs can look like current behavior; the result must distinguish active code from historical context.

## Assumptions

- This ticket is discovery-only and does not patch code.
