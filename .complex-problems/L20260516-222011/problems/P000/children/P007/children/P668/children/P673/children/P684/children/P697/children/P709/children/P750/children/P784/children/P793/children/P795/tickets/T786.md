# VMuse source FastMCP removal ticket

## Problem Definition
Source VMuse still contains a stale FastMCP server entry point and CLI wiring that can directly return MCP image content, bypassing the current HTTP/blob/tool-output contract.

## Proposed Solution
Delete the stale source `main.py` FastMCP server module. Rewrite the source CLI so `serve` delegates to the current `http_server.main`, remove SSE/stdio/FastMCP transport options, remove `.main` imports from info/skills paths, and update package comments/metadata wording to reflect the HTTP server contract. Preserve unrelated existing changes in VMuse tools.

## Acceptance Criteria
- `novaic-mcp-vmuse/src/novaic_mcp_vmuse/main.py` is removed or no longer part of source package behavior.
- Source CLI no longer imports `.main`, `mcp`, or advertises FastMCP/SSE/stdio transports.
- Source package comments no longer instruct importing FastMCP `mcp`.
- Current HTTP server entry remains runnable/importable.

## Verification Plan
Run targeted `rg` for FastMCP and `.main` imports in source VMuse. Compile/import the package and run focused tests after the follow-on contract-test child updates markers.
