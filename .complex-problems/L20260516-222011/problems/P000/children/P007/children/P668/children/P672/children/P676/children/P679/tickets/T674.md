# Remediate Local Dev Backend Launcher

## Problem Definition

`novaic-app/scripts/start-backends.sh` is an active local development launcher. P678 found that its wording claims to start all Python backends, but the inspected body appears to start only Gateway, Queue Service, Blob Service, and runtime workers. This can mislead diagnosis when Business, Cortex, Sandboxd, Device, or Entangled are required.

## Proposed Solution

Inspect the full script and current guards. If the script intentionally starts only a subset, make the script wording/status explicit and point to the cloud/full topology path. If it is supposed to start all services, patch it to align with current topology only if that can be done safely and verified locally. Prefer minimal clarity fixes over unverified process expansion.

## Acceptance Criteria

- Full local launcher is inspected.
- Misleading "all backend" wording or status is corrected, or full launch behavior is safely aligned.
- Changed script passes `bash -n`.
- Relevant guard tests/lints are run or a residual risk is recorded.

## Verification Plan

Read the script fully, compare with `scripts/start.sh` topology and P678 classification, patch with minimal diff, then run `bash -n novaic-app/scripts/start-backends.sh` and relevant static guard(s) if applicable.

## Risks

- Expanding local launcher to full topology could require local config/secrets and break developer workflows.
- Wording-only fixes improve diagnosis but do not make local launcher a full runtime replacement.

## Assumptions

- Live process startup is not required for this leaf; syntax/static verification is sufficient unless a safe smoke command exists.
