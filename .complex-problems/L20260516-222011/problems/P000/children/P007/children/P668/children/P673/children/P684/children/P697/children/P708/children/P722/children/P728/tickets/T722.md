# Legacy and standalone media-byte surface classification ticket

## Problem Definition

There may be remaining image/base64-emitting surfaces outside the active shell/runtime path, especially standalone MCP/plugin code and tests. We need to classify them so active violations become remediation candidates and legitimate non-shell surfaces are not mistakenly removed.

## Proposed Solution

Scan for media-byte/base64/image content emitters across active packages, docs, plugin/MCP code, and tests. Classify each relevant hit as active shell/runtime, standalone MCP/plugin, test fixture, doc, or obsolete residue. Record exact remediation candidates for active violations or misleading stale docs/code.

## Acceptance Criteria

- Remaining media-byte/image-content emitters are listed with file pointers.
- Each relevant surface is classified by role.
- Active shell/runtime violations are listed if found.
- Obsolete or misleading residue candidates are listed for remediation.

## Verification Plan

Run broad `rg` searches for `base64`, `data:image`, `_mcp_content`, `type": "image`, `image_ref`, `screenshot`, and `take_screenshot` across runtime, cortex, vmuse, device, docs, and tests. Inspect key non-test code paths and classify.

## Risks

- Broad searches will include many legitimate test fixtures; classification must not inflate noise into false bugs.

## Assumptions

- Discovery/classification only; remediation is deferred to `P723`.
