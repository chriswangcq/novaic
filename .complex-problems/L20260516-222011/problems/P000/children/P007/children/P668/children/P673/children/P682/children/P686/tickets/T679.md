# Scan Script and Package Entrypoints

## Problem Definition

Shell scripts, npm/package scripts, Tauri resources, and app backend resources can define real backend launch topology. We need a reproducible, bounded scan of these files before classifying worker/service roles.

## Proposed Solution

Use `rg --files`, targeted path filters, package metadata reads, and keyword scans to produce script/package entrypoint artifacts under `.complex-problems/L20260516-222011/tmp/p686/`. Include command transcript and compact summaries for known launch surfaces.

## Acceptance Criteria

- Script/package/resource candidate file list is produced.
- Keyword scan captures start/stop/deploy/smoke/health/worker/service references in those candidates.
- Known local and packaged backend launchers are present in the artifact or absence is explained.
- No production code is changed.

## Verification Plan

Check artifact existence, line counts, and targeted presence of `start-backends.sh`, package scripts, and resource backend launchers. Inspect representative head/tail slices rather than loading large raw output.

## Risks

- The scan may be noisy; classification is intentionally deferred to later children.
- Package metadata may be spread across nested projects.

## Assumptions

- This is a bounded read-only scan and is suitable for one-go execution after classification.
