# Cross-verify launch scripts against service topology

## Problem Definition

Three launch scripts must match the actual service topology: 8 services on specific ports.
- start.sh: production full set
- start-backends.sh: desktop dev subset (Gateway, Queue, Blob + workers)
- launch_split_only.sh: minimal split mode (Gateway + app binary)

## Proposed Solution

Map each script's service list against the known topology. Verify ports, entrypoints, and completeness. Classify as correct or stale.

## Acceptance Criteria

- Each script's service list maps to actual service entrypoints.
- No stale port/service references.
- Scripts are consistent with their documented purpose.

## Verification Plan

- Read each script's service launch commands.
- Cross-reference ports and entrypoint paths against actual files.

## Risks

- none — verification only.

## Assumptions

- The Explore agent's service topology summary is accurate.
