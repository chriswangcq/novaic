# Documentation media-byte residue classification ticket

Docs may still describe raw base64/image payload contracts after the shell/display Blob artifact changes. We need to classify and list stale docs.

## Proposed Solution

Run targeted documentation searches for base64, data:image, screenshot payloads, display image contracts, Blob artifacts, and VMuse media APIs. Inspect high-signal hits and classify them.

## Acceptance Criteria

- High-signal docs hits are classified with file pointers.
- Stale/misleading docs are listed as remediation candidates.
- Current contract docs are separated from historical or standalone protocol docs.

Use `rg` over `docs` and inspect matching docs with `sed`.

## Risks
