# Deployment script guard verification and patch result

## Summary

The deployment-script guard now covers the packaged backend launchers that previously escaped the runtime-topology cleanup. This closes the specific risk that the checked-in packaged launcher scripts could regress back to claiming a full backend topology or a fake Agent Runtime port while local/resource hygiene tests still passed.

## Done

- Added the checked-in generated Apple packaged backend launcher to the active-path guard set in `scripts/ci/test_no_legacy_file_hot_paths.py`.
- Added explicit packaged launcher guard targets for both `novaic-app/src-tauri/resources/backends/start-backends.sh` and `novaic-app/src-tauri/gen/apple/assets/backends/start-backends.sh`.
- Added `RETIRED_PACKAGED_LAUNCHER_TEXT` assertions that fail if packaged launchers reintroduce stale full-topology wording such as `Start all backend services`, `Starting NovAIC backends`, `All backends started`, `AGENT_RUNTIME_PORT`, or `Started Agent Runtime on`.
- Added `test_packaged_backend_launchers_do_not_claim_full_topology` so future edits to packaged launchers are caught by CI instead of relying on manual review.

## Verification

- `python3 -m pytest -q scripts/ci/test_no_legacy_file_hot_paths.py` passed with `2 passed`.
- `python3 -m pytest -q scripts/ci/test_app_resource_hygiene.py` passed with `2 passed`.
- `python3 scripts/ci/check_start_config_contract.py` passed with `start_config_contract OK`.
- Targeted stale-token scan over the two packaged launcher paths found no retired full-topology or fake-port phrases.
- Verification transcript was saved at `.complex-problems/L20260516-222011/tmp/P681-deployment-guard-verification.txt`.

## Known Gaps

- The guard intentionally covers checked-in source/resource launcher files, not transient build output under `target/`.
- Cloud/full-topology scripts may still legitimately use full-backend wording; this guard is scoped to packaged desktop launchers so it does not block valid cloud deployment terminology.
