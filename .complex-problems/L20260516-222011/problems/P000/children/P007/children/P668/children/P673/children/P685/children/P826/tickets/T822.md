# Verify CI guards match current service topology

## Problem Definition

30+ CI lint scripts may reference service boundaries. Topology-relevant guards must pass with current code.

## Proposed Solution

Identify and run topology-relevant CI guards. Verify they pass without false positives.

## Acceptance Criteria

- Topology-relevant CI guards identified and run.
- All pass without stale failures.

## Verification Plan

- grep CI scripts for port/topology references.
- Run identified guards.

## Risks

- none

## Assumptions

- CI guards that reference ports or service names are topology-relevant.
