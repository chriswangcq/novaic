# Historical image replay guardrail discovery result

## Summary

Historical image replay guardrails are in place. Runtime resolves step refs with projection modes: historical/non-current tool messages use `history`, current non-display tool messages use `current_tool_result`, and only current display tool messages use `display_perception`. Blob image refs are resolved to base64 only in `display_perception`, then converted into provider-native image user messages by `process_multimodal_messages`; historical replay remains text-only.

- Identified `_resolve_display_image_refs` as Blob image-ref resolution, gated to `projection == "display_perception"`.

## Known Gaps

- Current-round display perception intentionally converts Blob image bytes to provider-native base64 image content. This is expected for LLM vision, but it should never be stored as public history. Existing tests cover that boundary.
- The projection gate uses current round/latest tool-call heuristics; this is appropriate for current architecture but should stay covered by regression tests when harness sequencing changes.
