# Aggregate frontend payload residue verification

## Problem Definition

After P813 (SmartValue deleted) and P814 (AssistantMessage JSON.stringify removed), P786-scoped surfaces need a cross-cutting scan to confirm no raw payload rendering residue remains in factory logs, AssistantMessage, Visual components, or nearby code.

## Proposed Solution

Run targeted grep scans across `novaic-app/src/components/` and `novaic-llm-factory/src/` for `JSON.stringify`, `base64`, and raw rendering vocabulary. Classify each hit as safe (error formatting, data serialization, test-only) or unsafe (UI rendering of unbounded payloads). Run all Visual component tests and tsc. Report findings.

## Acceptance Criteria

- Every `JSON.stringify` in component production code is classified as safe or removed.
- base64-related code is classified as safe (data handling) or test-only.
- Visual component tests (4 files, 18 tests) pass.
- tsc passes.
- No unreviewed raw payload rendering path remains.

## Verification Plan

- `grep -rn JSON.stringify novaic-app/src/components/` on production code (excluding tests).
- `grep -rn base64|btoa|atob novaic-app/src/` to classify base64 usage.
- `npx vitest run src/components/Visual/` for test pass.
- `npx tsc --noEmit` for type safety.

## Risks

- A dynamically constructed rendering path could bypass static grep. Mitigated by also reviewing the event switch in AssistantMessage and the Visual component surface area.

## Assumptions

- The P786 scope is limited to `novaic-app/src/components/` and `novaic-llm-factory/src/` frontend/log surfaces.
