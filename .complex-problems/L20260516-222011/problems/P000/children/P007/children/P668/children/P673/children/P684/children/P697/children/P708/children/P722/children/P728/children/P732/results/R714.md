# Active production media-byte surface classification result

## Summary

No active shell/devicectl history path still prints screenshot base64 into normal shell text. The remaining base64 surfaces fall into four buckets: provider-native current-round display transport, auth/audio provider transport, legacy/standalone MCP image APIs, and a Device VmControl screenshot endpoint that still exposes MCP image bytes directly and should be treated as a remediation candidate if it is still product-facing.
