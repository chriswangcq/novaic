# App backend startup graph cleanup ticket

## Problem Definition
App backend startup scripts, packaged/generated backend scripts, service ports, and resource expectations may be stale or inconsistent with current backend topology.

## Proposed Solution
Split into a read-only topology audit and a bounded remediation child. The audit should identify current scripts/config/service ports/resources. The remediation should patch only confirmed stale script/config/comment residue and verify with shell syntax/config scans.

## Acceptance Criteria
- Backend script/service graph is explicitly mapped before edits.
- Current scripts agree on service list, ports, and expected packaged resources.
- Port naming conflict around Cortex/vmcontrol is resolved or clarified.
- Focused script/config checks pass.

## Verification Plan
Inspect `start-backends.sh` variants, service configs, resource files, vmcontrol setup templates, and port constants. Run `bash -n`, targeted `rg`, and any available focused tests/checks after patching.
