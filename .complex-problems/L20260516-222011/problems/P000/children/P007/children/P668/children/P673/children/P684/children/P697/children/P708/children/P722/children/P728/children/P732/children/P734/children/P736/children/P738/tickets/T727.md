# VMuse desktop screenshot MCP surface classification ticket

## Problem Definition

The VMuse desktop screenshot path encodes screenshots as base64 internally and wraps them as FastMCP image content. We need to classify this path precisely.

## Proposed Solution

Inspect `novaic-mcp-vmuse/src/novaic_mcp_vmuse/main.py` desktop screenshot and mouse aim wrappers plus `tools/desktop.py` screenshot/aim result construction. Record whether the path is standalone MCP image transport or current runtime shell behavior.

## Acceptance Criteria

- `main.py` screenshot and mouse aim wrapper functions are classified with file pointers.
- `tools/desktop.py` screenshot result and aim result construction are classified with file pointers.
- Cleanup/remediation candidates are explicit.

## Verification Plan

Use `rg` for `screenshot`, `base64`, `to_image_content`, and inspect relevant code with `sed`.

## Risks

- FastMCP image content is expected to carry bytes; do not flag it as a shell text leak without evidence it flows into current shell.

## Assumptions

- This ticket does classification only.
