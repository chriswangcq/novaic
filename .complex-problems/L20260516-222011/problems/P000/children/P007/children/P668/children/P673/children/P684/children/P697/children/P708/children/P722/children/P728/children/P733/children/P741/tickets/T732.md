# Test and generated-resource media-byte residue classification ticket

## Problem Definition

Tests and copied/generated resources may still contain base64/media contract references. We need to classify them as legitimate fixtures, source-of-truth generated copies, stale tests, or remediation candidates.

## Proposed Solution

Search tests and app packaged VMuse resource copies for media/base64 hits. Inspect representative fixtures and compare generated resource hits to source VMuse files.

## Acceptance Criteria

- Test hits are classified as legitimate fixture or stale contract expectation.
- Generated/copied resource hits are classified with source-of-truth handling.
- Exact tests/resources needing remediation are listed.

## Verification Plan

Use `rg` over test directories and app packaged VMuse resource directories; inspect relevant code with `sed`.

## Risks

- Generated resource copies should generally be regenerated from source rather than hand-edited.

## Assumptions

- This ticket is classification-only and does not change tests/resources.
