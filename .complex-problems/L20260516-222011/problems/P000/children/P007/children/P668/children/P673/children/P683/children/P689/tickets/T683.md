# Map Queue Service Worker and FSM Roles

## Problem Definition

Queue service currently hosts session coordination, FSM decisions, inbox/outbox handling, saga/task workers, scheduler and health endpoints. We need a compact source-evidence map of these entrypoints and roles.

## Proposed Solution

Inspect queue-service source files, launchers, and scan artifacts. Save focused role-map evidence under `.complex-problems/L20260516-222011/tmp/p689/`, summarize entrypoint files and role boundaries, and patch only if a tiny stale/misleading issue is clearly safe.

## Acceptance Criteria

- Queue-service entrypoint files and launch commands are listed with evidence.
- Worker/FSM roles are mapped to source files and responsibilities.
- Generic FSM infrastructure is distinguished from queue/session business coordination.
- Any code change is verified; if no code change is made, that is explicitly recorded.

## Verification Plan

Use `rg`, `sed`, and focused file lists to identify queue-service files. Check artifact existence and inspect representative source slices. If changed, run focused tests/import checks.

## Risks

- Queue service has both generic FSM substrate and business-specific session logic; conflating them would mislead future architecture work.
- Some workers may be launched through scripts rather than direct package metadata.

## Assumptions

- This is a bounded evidence-map leaf and can be one-go if no code remediation is discovered.
