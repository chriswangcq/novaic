# Scan Python Module and CLI Entrypoints

## Problem Definition

Backend service and worker entrypoints may be Python modules, package console scripts, `python -m` targets, uvicorn/gunicorn calls, or direct `if __name__ == "__main__"` modules. We need reproducible evidence for these Python entrypoint surfaces.

## Proposed Solution

Generate `.complex-problems/L20260516-222011/tmp/p687/` artifacts using `rg --files`, targeted pattern scans for Python entrypoint idioms, package metadata reads (`pyproject.toml`, setup metadata), and known worker/service keywords. Keep the scan read-only and pointer-oriented.

## Acceptance Criteria

- Candidate Python files and Python package metadata are scanned and saved.
- Entrypoint idioms such as `__main__`, `uvicorn`, `argparse`, Typer/Click, `python -m`, and console scripts are captured where present.
- Known backend names such as queue, runtime, cortex, sandbox, logicalfs, and blob are searched.
- No production code is changed.

## Verification Plan

Check artifact line counts and presence of expected backend modules/metadata. Inspect bounded head samples and confirm artifacts exist.

## Risks

- Python entrypoints may be invoked by shell scripts rather than package metadata.
- Pattern scans may include tests or examples; classification is deferred.

## Assumptions

- This is a fifth-level, read-only, artifact-producing scan suitable for one-go after classification.
