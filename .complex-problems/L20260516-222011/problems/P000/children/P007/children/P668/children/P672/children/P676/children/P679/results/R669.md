# Local dev backend launcher remediation result

## Summary

Fixed the active local dev launcher and runbook so they no longer claim `novaic-app/scripts/start-backends.sh` starts every Python backend. The script now clearly identifies itself as a local desktop backend subset launcher and warns that Entangled, Business, Device, Cortex, and Sandboxd are not started by it.

## Done

- Updated `novaic-app/scripts/start-backends.sh` header, usage text, stop/status labels, startup banner, and final banner from "all backends" to "local backend subset".
- Added explicit runtime warning in `novaic-app/scripts/start-backends.sh`: starts Gateway, Queue Service, Blob Service, runtime workers; does not start Entangled, Business, Device, Cortex, or Sandboxd.
- Updated `docs/runbooks/local-backends.md` to match the script's actual behavior and warn that services outside the subset must be started separately for agent/runtime flows.
