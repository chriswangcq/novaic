# Cortex tool output and context projection discovery result

## Summary

Cortex projection is clean for the active contracts inspected. `tool-output.v1` renders artifact manifests as text with display access hints and does not directly project artifact images. `tool-step-payload.v1` uses `llm_content` instead of raw shell payload. Display/image content is only visual in explicit display perception projection, while history/current tool result projections remain text-only. Context event projection omits unknown large tool payloads and points to `payload_ref` instead of inlining raw media.

## Done

- Identified `novaic-cortex/novaic_cortex/step_result_projection.py` as the tool result parsing/projection module.
- Identified `_parse_tool_output_v1`, `parse_tool_result`, `format_for_history_llm`, `format_for_current_tool_result_llm`, and `format_for_display_perception_llm` as the relevant projection path.
- Identified `novaic-cortex/novaic_cortex/context_event_projection.py` as the event-to-context projection layer, with `_tool_result_content` omitting unknown payload details by `payload_ref`.
- Verified step write tests reject inline raw tool result requests and externalize large payloads to payload/blob refs.
- Verified display/image fixture behavior is intentionally projection-mode-specific: history/current text-only; display perception may carry image/image_ref.
