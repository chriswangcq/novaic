# Remediate Packaged Backend Launcher

## Problem Definition

P678 found packaged backend launchers under `novaic-app/src-tauri/resources/backends/start-backends.sh` and generated apple assets that appear stale (`AGENT_RUNTIME_PORT=19991`, only Blob/Gateway/Agent Runtime). We need determine whether these are active packaging surfaces and remediate without leaving source/generated drift.

## Proposed Solution

Inspect Tauri/resource configuration and the packaged launcher source. If the resource launcher is active, update the source resource script to match current explicit subset semantics or remove/guard stale behavior if unused. Handle generated copy consistently and verify resource hygiene/static tests.

## Acceptance Criteria

- Source resource launcher and generated copy are inspected with evidence.
- Active/unused/generated status is determined.
- Stale packaged launcher wording or behavior is fixed at the source-truth level; generated copy is either updated consistently or classified as generated residue.
- Relevant shell syntax/resource hygiene checks pass.

## Verification Plan

Use targeted `rg`/`sed` inspection of `src-tauri` resource references, patch with `apply_patch`, run `bash -n` on changed shell scripts, and run `python3 -m pytest -q scripts/ci/test_app_resource_hygiene.py` plus relevant legacy hot-path guards.

## Risks

- Generated apple assets may be overwritten by Tauri tooling; source resource should remain authoritative.
- Packaged backend launch may be deprecated but still present as a resource; deleting without a guard could break packaging expectations.

## Assumptions

- Live packaged app build is out of scope; local static/resource checks are acceptable for this remediation leaf.
