# Doc CI guards pass after topology changes

## Problem Definition

Doc-focused CI guard scripts (lint_docs_status_consistency.py, lint_current_docs_residue.sh) validate documentation against current code state. After topology audits and any doc edits, these guards must pass cleanly.

## Proposed Solution

Run both doc CI guard scripts and verify they pass. If any failures, diagnose whether they are pre-existing or caused by recent changes, and document the outcome.

## Acceptance Criteria

- lint_docs_status_consistency.py runs without failures (or pre-existing failures documented).
- lint_current_docs_residue.sh runs without failures (or pre-existing failures documented).
- No stale doc claims flagged by guards.

## Verification Plan

1. Locate both guard scripts.
2. Run each guard script.
3. Analyze output for pass/fail.
4. Document results.

## Risks

- Guards may have pre-existing failures unrelated to this audit.

## Assumptions

- Guard scripts are runnable from the repo root.
