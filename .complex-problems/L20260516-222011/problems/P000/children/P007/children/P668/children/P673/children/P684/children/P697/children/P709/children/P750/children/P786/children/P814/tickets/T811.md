# Guard extractContent JSON.stringify fallback in AssistantMessage

## Problem Definition

`AssistantMessage.tsx` event rendering is live and handles `text`, `final`, `warning`, `error`, `image` types. The `extractContent` helper (line 96-113) contains a `JSON.stringify(content)` fallback on line 105 that fires when a nested property (`content`, `data`, `text`, `error`, `message`) exists but is not a string. This can render large raw JSON, base64 blobs, or payload-like content directly into the chat surface.

The events switch itself is clean — unknown types return `null`, known types delegate to safe renderers (Markdown, bounded text). The risk is isolated to `extractContent`.

## Proposed Solution

Replace the `JSON.stringify(content)` fallback in `extractContent` with an empty string return. Non-string nested content should not be rendered raw in a chat bubble. If the extracted value is not a string, treat it as absent content — the UI already handles empty strings gracefully (returns `null` for empty text events, shows fallback text for warnings/errors).

Specifically change line 105 from:
```ts
return typeof content === 'string' ? content : JSON.stringify(content);
```
to:
```ts
return typeof content === 'string' ? content : '';
```

## Acceptance Criteria

- `extractContent` no longer calls `JSON.stringify` on non-string content.
- String event data still renders normally.
- Non-string nested content results in empty string (safe fallback).
- `tsc --noEmit` passes.

## Verification Plan

- Read the changed `extractContent` function and confirm no `JSON.stringify` path remains.
- TypeScript check passes.
- Manual review of all `extractContent` call sites confirms they handle empty string correctly (`text`/`final` return null for empty content; `warning`/`error` show fallback labels).

## Risks

- A legitimate non-string event payload that was previously rendered (as JSON text) would now show empty. This is the intended behavior — chat surfaces should not display raw JSON.

## Assumptions

- No downstream code depends on `extractContent` returning stringified JSON objects.
- `extractContent` is only used within this file.
