# Deploy and build docs topology audit

## Problem Definition

Deploy/build docs (runbooks, deployment-and-startup, CI docs) may reference stale service names, ports, or paths that no longer match the current 8-service topology.

## Proposed Solution

Scan all deploy/build/release/CI documentation for service topology claims (service names, ports, paths, process references). Cross-check each claim against the current service topology and repo structure. Report discrepancies or confirm accuracy.

## Acceptance Criteria

- Deploy docs reference correct services and paths.
- Build/release docs are current.
- Historical docs clearly marked.

## Verification Plan

1. Find all deploy/build/release docs via file search.
2. Grep for service names (Entangled, Gateway, Business, Device, Queue, Blob, Sandboxd, Cortex) and port numbers (19900, 19999, 19998, 19993, 19997, 19995, 19994, 19996).
3. Cross-check each reference against actual service entrypoints and repo layout.
4. Confirm any historical references are clearly labeled as such.

## Risks

- A doc may be intentionally historical but not clearly marked, requiring judgment.

## Assumptions

- The 8-service topology documented in service-topology.md is the ground truth.
