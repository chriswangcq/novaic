# Inventory Deployment and Start Scripts

## Problem Definition

Deployment/start/supervision scripts may contain stale runtime process names or ambiguous roles after recent worker/FSM/service changes. We need locate these scripts, classify active versus historical/test-only surfaces, and patch concrete low-risk stale claims if found.

## Proposed Solution

Use repository-wide file and text search to identify scripts/configs that start, deploy, supervise, health-check, or smoke backend services. Inspect high-signal files, create an evidence-backed script topology inventory, and patch only active misleading script comments, names, or diagnostics that are safe to change locally.

## Acceptance Criteria

- Script/config candidates are located with reproducible search commands.
- Active scripts are distinguished from test-only/historical artifacts.
- Current process role expectations in scripts are summarized with file evidence.
- Any low-risk stale active script issue found is fixed and verified.

## Verification Plan

Run `rg --files` and targeted `rg` searches for deploy/start/worker/service/smoke/health terms. Read high-signal files. For script changes, run shell syntax checks or existing CI guard scripts where applicable. Record scan artifacts in the ledger tmp directory.

## Risks

- Some active deploy behavior may be outside the repository.
- Scripts may depend on environment variables unavailable locally; avoid executing live deploy commands.
- Over-broad search terms may include unrelated docs or generated files; classify rather than bulk edit.

## Assumptions

- This child should not perform production deployment or restart.
- Low-risk changes are limited to local scripts/docs/tests/diagnostics with local verification.
