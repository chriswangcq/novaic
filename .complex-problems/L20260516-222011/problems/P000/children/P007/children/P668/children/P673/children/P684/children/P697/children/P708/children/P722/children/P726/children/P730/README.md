# Cortex tool output and context projection discovery

## Problem

Discover how Cortex parses step result payloads, artifact manifests, display files, and context event projections so artifact references remain structured and history-visible text stays bounded.

## Success Criteria

- Cortex parsing/projection code for `tool-output.v1` and display files is identified with file pointers.
- Step/event context projection behavior is identified with tests or code evidence.
- Any Cortex active path that converts Blob/display artifacts into raw base64 text for history is listed.
