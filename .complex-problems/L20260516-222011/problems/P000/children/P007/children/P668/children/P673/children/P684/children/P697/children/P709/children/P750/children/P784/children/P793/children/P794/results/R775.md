# VMuse active entry map result

## Summary
Mapped VMuse entry points. The active installed service path is HTTP server based (`novaic_mcp_vmuse.http_server`), while `main.py` and parts of `cli.py` still expose stale FastMCP/direct-media code that can be deleted in the source cleanup child problem.

## Evidence
- Active service entry evidence:
  - `novaic-mcp-vmuse/run_server.sh:10` runs `python3 -m novaic_mcp_vmuse.http_server`.
  - `novaic-app/src-tauri/vmcontrol/src/api/routes/setup.rs` and `vmuse.rs` systemd templates use `ExecStart=/opt/novaic/venv/bin/python3 -m novaic_mcp_vmuse.http_server`.
  - `novaic-mcp-vmuse/src/novaic_mcp_vmuse/http_server.py` defines aiohttp routes for desktop/browser/shell/file/window/context tools and health.
- Stale source evidence:
  - `novaic-mcp-vmuse/src/novaic_mcp_vmuse/main.py` imports `FastMCP` and `Image`, registers many `@mcp.tool` handlers, decodes screenshot base64, and returns MCP `ImageContent` directly.
  - `novaic-mcp-vmuse/src/novaic_mcp_vmuse/cli.py` labels the server as FastMCP, imports `.main import mcp`, and starts `mcp.run(...)` for `serve`.
  - `novaic-mcp-vmuse/src/novaic_mcp_vmuse/__init__.py` says HTTP server has no FastMCP dependency but still comments that `main.py (FastMCP)` is a separate entry point.
  - `novaic-mcp-vmuse/pyproject.toml` script `novaic = novaic_mcp_vmuse.cli:main` routes users into the stale CLI.
- App resource/generated copies contain the same stale VMuse files; those are package-copy cleanup concerns for the later app resource remediation problem, not this source-only map.

## Deletable In Cleanup Child
- Delete or retire source `novaic-mcp-vmuse/src/novaic_mcp_vmuse/main.py`.
- Rewrite `novaic-mcp-vmuse/src/novaic_mcp_vmuse/cli.py` so `serve` delegates to `http_server.main` and no longer imports `.main`/`mcp` or advertises FastMCP/SSE/stdio transports.
- Update `novaic-mcp-vmuse/src/novaic_mcp_vmuse/__init__.py` comments to remove `main.py`/FastMCP entry references.
- Update source contract tests to reject reintroduced FastMCP markers/direct media entry points.

## Residual Risk
This map did not change code. It intentionally leaves app-bundled copies for a separate remediation because they live under `novaic-app` packaging resources and generated assets.
