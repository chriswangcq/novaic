# Map Agent Runtime Loop and Worker Roles

## Problem Definition

Agent Runtime contains the agent loop, shell/tool execution boundaries, and worker mode wrappers that interact with Queue Service, Cortex, Business, Blob, Sandbox, and LLM Factory. We need a source-evidence map that explains runtime process roles without conflating them with queue-service durable state.

## Proposed Solution

Inspect `novaic-agent-runtime` entrypoints, worker registry, runtime loop modules, shell/tool execution modules, and launch scripts. Produce focused role-map artifacts under `.complex-problems/L20260516-222011/tmp/p690/` and patch only clearly low-risk stale labels if discovered.

## Acceptance Criteria

- Agent-runtime entrypoint files and launch commands are identified with evidence.
- Runtime loop/tool execution/worker mode roles are summarized in a compact map.
- Queue-service durable state responsibilities are distinguished from runtime process execution.
- Any code change is verified; if no code change occurs, artifact verification is recorded.

## Verification Plan

Use targeted `rg` and file inventory reads, then inspect representative source samples. If production files change, run focused compile/import/tests. Otherwise verify artifact counts and expected file presence.

## Risks

- Runtime worker modes can look like queue-service workers because their CLI names overlap.
- Tool execution and shell sandbox boundaries may be spread across multiple modules.

## Assumptions

- This is a bounded mapping leaf and can be one-go if no remediation is discovered.
