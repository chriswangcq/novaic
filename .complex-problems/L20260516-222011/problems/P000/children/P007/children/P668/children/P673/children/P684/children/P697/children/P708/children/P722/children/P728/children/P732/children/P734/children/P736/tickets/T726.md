# VMuse MCP tool media-byte classification ticket

## Problem Definition

Standalone VMuse MCP tool modules return or convert screenshots and other media bytes. We need a module/function-level classification so legitimate MCP image transport is not confused with current shell/runtime history leakage.

## Proposed Solution

Inspect VMuse MCP entrypoints and tool modules with base64/image hits. Split into focused child problems for desktop/FastMCP screenshot wrappers and browser/window/misc media surfaces, then summarize their classifications.

## Acceptance Criteria

- VMuse MCP screenshot and media-byte tool surfaces are classified by module/function.
- Legitimate standalone MCP image transport is separated from current runtime shell behavior.
- Dead or misleading imports are listed as cleanup candidates.
- Exact remediation candidates, if any, are identified.

## Verification Plan

Use `rg` and `sed` over `novaic-mcp-vmuse/src/novaic_mcp_vmuse/main.py`, `tools/desktop.py`, `tools/browser.py`, and `tools/windows.py`.

## Risks

- MCP image content intentionally uses media blocks; do not flag it as a shell contract violation unless it feeds the active shell/runtime path.

## Assumptions

- `novaic-mcp-vmuse` is standalone/plugin-adjacent unless evidence shows it is mounted into the current agent shell path.
