# Frontend runtime log and observability hooks audit

## Problem Definition

The frontend (novaic-app) may contain observability hooks, log viewers, or diagnostic components that reference backend log/health APIs. Need to verify endpoints are valid and no raw payloads are exposed.

## Proposed Solution

Search novaic-app for observability/log-related hooks, components, and API references. Verify backend endpoint references match actual routes. Check for raw payload exposure in observability UI.

## Acceptance Criteria

- Frontend observability/log-related hooks and components located.
- Backend endpoint references verified against actual routes.
- No raw payload exposure or stale API references in observability UI.
- Relevant tests/lints pass after any fixes.

## Verification Plan

1. Grep novaic-app for health, log, observability, monitor-related components and hooks.
2. Extract any backend API URL references.
3. Verify references against backend routes.
4. Check for raw JSON/payload rendering in observability components.
5. Run tsc if any fixes made.

## Risks

- Frontend may have many components, need focused search.

## Assumptions

- novaic-app/src is the search scope.
