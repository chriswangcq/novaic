# Runtime health and log access observability audit

## Problem Definition

Health endpoints, runtime log access paths, LLM Factory log APIs/UI, and observability code may have stale contracts or poor diagnostics. Need to locate, inspect, and fix low-risk issues.

## Proposed Solution

1. Locate health endpoints across all services (Gateway, Runtime, Cortex, Blob, etc.).
2. Inspect LLM Factory log API/UI paths.
3. Check runtime log access/observability code in frontend and backend.
4. Verify contracts against actual code and tests.
5. Fix any concrete low-risk diagnostics or contract gaps found.

## Acceptance Criteria

- Health/log/LLM-factory observability routes, clients, and UI components located and inspected.
- Request/response contracts checked against code evidence.
- Concrete low-risk diagnostics or contract gaps fixed.
- Relevant tests or local checks run with evidence recorded.

## Verification Plan

1. Grep for health endpoints, log access routes across services.
2. Inspect LLM Factory log UI (factory-logs.html) and API paths.
3. Check frontend observability hooks and components.
4. Run relevant tests/lints after any fixes.

## Risks

- Some health endpoints may only be testable against a running server.
- LLM Factory UI may need running backend for full verification.

## Assumptions

- Static code analysis and existing tests are sufficient for contract verification.
- LLM Factory log UI is at novaic-llm-factory/static/factory-logs.html.
