# LLM Factory log API and UI audit

## Problem Definition

LLM Factory provides a log viewing UI (factory-logs.html) and associated API endpoints. Need to verify UI file exists, API paths referenced are valid, and the fetch/display contract is consistent.

## Proposed Solution

Locate factory-logs.html, inspect its fetch calls and API path references, then verify those paths exist in the LLM Factory backend code.

## Acceptance Criteria

- factory-logs.html located and inspected.
- Fetch paths in UI code match backend route definitions.
- No stale API references or broken fetch paths.

## Verification Plan

1. Find factory-logs.html.
2. Extract all API endpoint references from its JavaScript.
3. Grep LLM Factory backend for matching route definitions.
4. Report any mismatches.

## Risks

- LLM Factory may use dynamic route generation making static analysis harder.

## Assumptions

- factory-logs.html is at novaic-llm-factory/static/factory-logs.html.
