# Smoke and deploy guard script inventory and contract check

## Problem Definition

Smoke-test scripts and deployment freshness guards across the repo may reference stale services, endpoints, or file paths. Need a complete inventory and contract check.

## Proposed Solution

1. Find all scripts in scripts/ci/ and scripts/deploy/ related to smoke testing or deploy freshness.
2. Inspect each for service references, file paths, and endpoint contracts.
3. Cross-check against current topology and codebase.

## Acceptance Criteria

- All smoke/deploy guard scripts inventoried.
- Service references and file paths verified against current topology.
- Stale or broken references identified.

## Verification Plan

1. List all scripts in scripts/ci/ and scripts/deploy/.
2. Grep for service names, ports, file paths.
3. Verify each reference against actual code.

## Risks

- Some scripts may be deployment-only and hard to verify statically.

## Assumptions

- Static analysis of script content is sufficient.
