# Smoke-test ergonomics and deployment freshness guard audit

## Problem Definition

Smoke-test scripts and deployment freshness guards may have stale contracts, missing checks, or poor diagnostics. Need to audit them for correctness and ergonomics.

## Proposed Solution

1. Locate all smoke-test scripts and deployment freshness guards.
2. Inspect their contracts and expected behavior.
3. Verify they reference correct services, endpoints, and tool contracts.
4. Identify and fix any concrete low-risk gaps.

## Acceptance Criteria

- Smoke scripts, deploy freshness guards, and related docs/tests located and inspected.
- Expected smoke behavior documented or encoded where missing.
- Low-risk improvements applied for concrete gaps.
- Smoke-related checks run locally where possible.

## Verification Plan

1. Find all smoke/deploy-guard scripts.
2. Inspect their logic and service references.
3. Run relevant scripts/tests locally.
4. Document findings.

## Risks

- Some smoke scripts may require production access to fully test.

## Assumptions

- Local CI-style checks are sufficient for static validation.
