# Audit Deployment Runtime Observability and Smoke Paths

## Problem Definition

Recent runtime/FSM/shell/display/logging changes make deployment and smoke diagnostics important. We need inspect deployment scripts, runtime process topology expectations, health/log surfaces, LLM factory logs visibility, and smoke-test ergonomics without destructive production changes. The goal is to find and apply low-risk improvements that make failures easier to diagnose.

## Proposed Solution

Audit the deployment and runtime observability surfaces in focused slices: process/service scripts, health/log endpoints and dashboards, smoke scripts or runbooks, and current tests/guards for deployment freshness. Then apply only low-risk code/docs/test optimizations that improve diagnosis or prevent stale assumptions. If the surface is broad, split into child problems before implementation.

## Acceptance Criteria

- Inspect deployment/runtime process layout scripts, configs, and documentation without destructive actions.
- Inspect current smoke-test and observability paths for logs, health, and LLM factory visibility.
- Identify concrete low-risk gaps, stale docs, or guard/test holes.
- Apply targeted fixes where safe, or record residual risk when a fix would require production access or broader design work.
- Run relevant local checks/tests/linters for any changed files.

## Verification Plan

Use `rg`, `find`, targeted file reads, and existing local scripts/tests. Prefer read-only inspection first. For changes, run focused tests or lint scripts that cover deployment freshness, runtime observability, health endpoints, and frontend/API log visibility. Avoid live destructive deployment actions unless explicitly requested later.

## Risks

- Runtime/deployment behavior may depend on external infrastructure that local checks cannot fully reproduce.
- Smoke scripts can be flaky if they hit live endpoints; local/static checks should be separated from production assertions.
- Broad cleanup of deployment scripts can break active ops paths if not verified carefully.

## Assumptions

- This ticket can split if the deployment/runtime observability surface is larger than a safe one-go pass.
- Low-risk improvements include docs, guard scripts, static API/frontend fixes, and focused tests.
- Production deployment or live restart is out of scope for this ticket unless the ledger later creates an explicit deployment execution problem.
