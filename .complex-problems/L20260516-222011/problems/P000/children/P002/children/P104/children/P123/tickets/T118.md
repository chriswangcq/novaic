# Display Tool History Sanitization Audit Ticket

## Problem Definition

Display tool results may include image data for current-turn perception, but historical/public tool output must be sanitized so raw base64 or data URLs do not re-enter ordinary LLM context.

## Proposed Solution

Inspect runtime display executor wrapping and Cortex step-result projection, run focused display/history tests, and add or adjust guards if any history path includes raw image data.

## Acceptance Criteria

- Public/historical display tool output is text or placeholder-only.
- Historical display output does not contain raw `data`, `data:image`, or base64 image payloads.
- Current perception path is not evaluated here except to ensure it is separated from history.
- Focused tests cover no historical image injection.

## Verification Plan

Read display executor/projection code, run `test_tool_handlers_display_chat_history.py`, `test_no_historical_tool_image_injection.py`, and Cortex projection tests.

## Risks

- A test can assert current-turn image injection and accidentally miss that the same payload leaks into history.

## Assumptions

- Current display perception is handled by sibling P124.
