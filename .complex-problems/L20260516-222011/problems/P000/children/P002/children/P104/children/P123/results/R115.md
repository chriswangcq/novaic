# Display Tool History Sanitization Audit Result

## Summary

Audited display history sanitization. Runtime wrapping stores full current-turn display perception data in durable payload `llm_content`, but public/history tool content uses image placeholders without raw `data`; Cortex projection tests also confirm historical/current non-display projections remain text-only.

## Done

- Inspected runtime display executor and wrapping tests in `test_tool_handlers_display_chat_history.py`.
- Inspected no-historical-image-injection tests proving ordinary/history display tool messages do not create user image messages and placeholders strip image data.
- Inspected Cortex projection tests around tool-output/display formatting.
- Confirmed current display perception is intentionally carried only through `_projection="display_perception"`, not generic history.
