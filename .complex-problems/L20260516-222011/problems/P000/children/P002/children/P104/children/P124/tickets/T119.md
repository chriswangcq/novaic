# Provider-Native Image Projection Audit Ticket

## Problem Definition

Display history sanitization must not break current-turn image perception. Current display output should become provider-native image content through the dedicated multimodal path, not raw base64 text in tool content.

## Proposed Solution

Inspect multimodal conversion and provider adapter tests, run focused current-display image projection tests, and add or adjust regression guards if image handoff relies on text leakage.

## Acceptance Criteria

- Current display perception is converted into provider-native image content for OpenAI-compatible messages.
- Tool message content remains sanitized while a separate user image message or provider image part is produced.
- Legacy/nested result shapes do not trigger image injection.
- Focused tests cover display projection ordering relative to following system messages.

## Verification Plan

Run runtime multimodal/no-historical-image tests and inspect provider conversion code for the display perception path.

## Risks

- A provider adapter could receive a system message after display before the image part, causing image context to be lost or misplaced.

## Assumptions

- Provider adapter may encode bytes as data URLs internally; this is acceptable only at provider-native image content boundary, not as tool text/history.
