# Artifact-Producing CLI Blob Manifest Contract Audit Ticket

## Problem Definition

Artifact-producing shell CLIs must not print raw media bytes or base64. They should return `tool-output.v1` JSON with concise text, artifact metadata, and `blob://runtime-artifact/...` references.

## Proposed Solution

Inspect the artifact manifest builder and devicectl media/file commands, run focused blob contract tests, and add missing coverage if any artifact path lacks base64-leak assertions.

## Acceptance Criteria

- `devicectl hd screenshot` output is `tool-output.v1` with a blob artifact ref.
- `devicectl hd file-pull` output is `tool-output.v1` with a blob artifact ref.
- Raw base64 source payloads are absent from stdout.
- The manifest advertises `display` access for image artifacts where appropriate.

## Verification Plan

Read `shell_capabilities.py` manifest helper paths and run `novaic-cortex/tests/test_shell_capabilities_blob_contract.py`.

## Risks

- Screenshot path may be covered while file-pull path leaks bytes.

## Assumptions

- Device API may still return base64 internally; the shell CLI boundary must convert it to Blob artifact output.
