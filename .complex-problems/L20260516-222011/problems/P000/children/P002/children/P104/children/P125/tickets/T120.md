# Blob Artifact and Display Base64 Regression Sweep Ticket

## Problem Definition

The final blob/display sweep must prove shell artifact manifests, display history sanitization, and current display perception remain coherent together, without raw base64 or data URLs leaking into ordinary context.

## Proposed Solution

Run focused runtime/Cortex tests covering artifact manifests, tool-output projection, display history, no historical image injection, and shell-output contracts; perform targeted source search for active leakage risks; add minimal guards if a gap remains.

## Acceptance Criteria

- Artifact CLI, display history, current display perception, and shell-output contract tests all pass together.
- Active source search finds only protocol-local base64 use, Blob upload conversions, or provider-native multimodal conversion, not raw tool text/history leakage.
- No additional leak path remains untested.

## Verification Plan

Run focused pytest bundles in Cortex and Runtime and a targeted `rg` classification for `data:image`, `_mcp_content`, `tool-output.v1`, and `base64` in active code/tests.

## Risks

- Source search will include allowed protocol-local base64 uses; classification must be explicit rather than blind deletion.

## Assumptions

- Provider-native image payload conversion may use data URLs/base64 internally and is allowed only outside ordinary tool text/history.
