# Cortex CLI Coverage and Workspace Access Audit Ticket

## Problem Definition

Cortex shell access must expose stable payload inspection commands and stable `/cortex/ro`/`/cortex/rw` workspace paths. Agents should not depend on ephemeral backing paths or removed direct payload tools.

## Proposed Solution

Split into payload CLI command coverage and stable workspace path guidance/abuse guards. Verify implementations/tests and patch bounded discoverability gaps.

## Acceptance Criteria

- Cortex payload read/search/summarize/qa CLI coverage is audited.
- Stable workspace path guidance and ephemeral-path guards are audited.
- Focused tests/help checks verify both surfaces.
- Missing/misleading coverage is fixed or routed.

## Verification Plan

- Inspect `shell_capabilities.py`, Cortex tests, runtime shell schema, and runtime path abuse tests.
- Run Cortex payload/help/workspace focused tests.

## Risks

- summarize/qa require model config for live calls; use tests/help for contract shape unless a local fake is already available.

## Assumptions

- Payload and workspace path concerns are independent enough to split.
