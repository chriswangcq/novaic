# Test and Schema Stable Path Residue Classification Ticket

## Problem Definition

Tests and schema descriptions can mention `novaic-cortex-sandbox-*` only as a negative warning, runtime guard implementation, or regression fixture. They must not drift into positive guidance for copied backing paths.

## Proposed Solution

Search common/Cortex/runtime tests and schema files for stale backing path mentions, classify each hit, add or adjust guards if any public schema warning or runtime test coverage is missing, and record allowed mentions.

## Acceptance Criteria

- Every test/schema `novaic-cortex-sandbox` mention is classified as negative warning, guard implementation, or regression fixture.
- Public shell schema has tests proving stable path guidance and negative backing-path warning.
- Runtime stale-path guard tests cover representative bare and quoted stale path commands.
- No test fixture normalizes stale backing paths as a supported path.

## Verification Plan

Run `rg` over test/schema/code surfaces, inspect hits, run common/Cortex focused tests, and add minimal regression tests if coverage is incomplete.

## Risks

- A fixture can accidentally make old paths look supported if assertions only check success.
- Schema descriptions can warn against stale paths but lack test coverage.

## Assumptions

- Implementation constants in guard code are allowed mentions when tests prove rejection.
