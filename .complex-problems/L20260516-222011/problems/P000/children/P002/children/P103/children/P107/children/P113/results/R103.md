# Cortex Payload CLI Coverage Audit Result

## Summary

Audited the Cortex payload shell surface for `cortex payload read/search/summarize/qa`. The implementation, generated help, LLM-facing shell schema, runtime payload interpretation wiring, and focused tests all point to the stable shell CLI path rather than direct payload LLM tools.

## Done

- Located implementation in `novaic-cortex/novaic_cortex/shell_capabilities.py` via `PAYLOAD_HELP` and `_cortex_payload`, covering `read`, `search`, `summarize`, and `qa` subcommands backed by `/v1/payload/read`, `/v1/payload/search`, `/v1/payload/summarize`, and `/v1/payload/qa`.
- Verified shell schema in `novaic-common/common/tools/llm_builtin.py` advertises `cortex payload read/search/summarize/qa`.
- Added a common schema regression guard in `novaic-common/tests/test_tool_definitions_contract.py` so the public LLM shell tool description cannot silently drop the payload command family.
- Confirmed generated `cortex --help` and `cortex payload --help` scripts exit 0 and show payload CLI help.
