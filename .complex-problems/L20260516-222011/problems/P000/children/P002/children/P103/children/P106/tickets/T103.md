# Devicectl CLI Coverage and Artifact Contract Audit Ticket

## Problem Definition

`devicectl` is the shell-first device command surface. It includes media/file-producing commands that must emit blob artifact manifests instead of base64, and non-media device commands that must remain reachable/discoverable through shell.

## Proposed Solution

Split the audit into artifact-producing command contract and non-media command coverage. Verify implementation, help/schema guidance, and focused tests; fix bounded gaps.

## Acceptance Criteria

- Devicectl media/file-producing commands are audited for blob artifact output.
- Devicectl non-media command coverage/help/schema is audited.
- Focused tests or safe help checks verify each surface.
- Missing or misleading coverage is fixed or routed.

## Verification Plan

- Inspect `novaic-cortex/novaic_cortex/shell_capabilities.py` devicectl implementation.
- Run Cortex shell capability blob contract tests and generated help checks.
- Add guard tests if command/schema drift is found.

## Risks

- Live device interactions can be side-effectful; use existing fake-server tests and help checks instead of touching a real device unless needed.

## Assumptions

- Screenshot and file pull are the primary artifact-producing device commands to guard in this pass.
