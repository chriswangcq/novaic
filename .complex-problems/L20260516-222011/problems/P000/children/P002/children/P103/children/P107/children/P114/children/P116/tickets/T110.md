# Runtime Stale Backing Path Guard Audit Ticket

## Problem Definition

The runtime shell handler must prevent agents from reusing copied `/tmp/novaic-cortex-sandbox-*` paths from prior outputs. Such paths are ephemeral and should fail early with a clear message instead of timing out or producing confusing filesystem errors.

## Proposed Solution

Locate the stale backing-path guard in runtime shell handling, inspect its matching behavior and error contract, run focused tests, and add the smallest missing guard/test if stale paths can slip through.

## Acceptance Criteria

- Runtime shell handler detects `novaic-cortex-sandbox-*` backing paths in commands.
- The rejection message points agents to `/cortex/ro`, `/cortex/rw`, `$RO`, or `$RW` instead.
- Focused tests cover stale-path rejection for representative copied paths.
- No fallback behavior attempts to resolve or translate stale backing paths.

## Verification Plan

Search runtime shell handling for backing path detection, read the implementation, run focused runtime shell guard tests, and add regression coverage if the current test set is too narrow.

## Risks

- The guard may only catch one string shape and miss quoted or nested command occurrences.
- The guard may rely on a brittle message that does not teach the stable replacement path.

## Assumptions

- Rejection is preferable to compatibility translation because copied backing paths are outside the contract.
