# Stable Path Schema and Help Guidance Audit Ticket

## Problem Definition

The public shell surface must tell agents to use stable Cortex workspace paths (`/cortex/ro`, `/cortex/rw`, `$RO`, `$RW`) and avoid copied `novaic-cortex-sandbox-*` backing paths.

## Proposed Solution

Inspect the canonical shell tool schema and generated Cortex help text, add the smallest missing schema/help regression guard if needed, and verify generated help output.

## Acceptance Criteria

- Canonical shell schema contains stable path guidance.
- Generated `cortex --help` contains stable path guidance.
- No schema/help text encourages direct use of copied `novaic-cortex-sandbox-*` paths.
- Focused tests or smoke checks prove the guidance remains present.

## Verification Plan

Use `rg`/file reads for schema/help source, run generated help smoke for `cortex --help`, and run the relevant common/Cortex tool schema tests.

## Risks

- Guidance may exist in schema but not in generated help.
- A negative mention of backing paths may be mistaken as positive guidance.

## Assumptions

- `$RO` and `$RW` are acceptable aliases, but `/cortex/ro` and `/cortex/rw` are the preferred stable paths.
