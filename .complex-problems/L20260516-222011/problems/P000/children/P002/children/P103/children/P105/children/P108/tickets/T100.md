# Agentctl IM CLI Contract Audit Ticket

## Problem Definition

Agents need `agentctl im` commands in shell for reading notifications, replying to users, and inspecting/searching IM context. The command surface must be implemented, discoverable, and referenced by the shell tool schema.

## Proposed Solution

Inspect the `agentctl im` implementation, command registration/help shape, shell tool schema mentions, and related tests. Run focused tests/help checks and patch any bounded documentation or registration gap found.

## Acceptance Criteria

- `agentctl im` implementation and subcommands are identified.
- read/reply/history/search commands or intended equivalents are present.
- Shell schema points agents to the correct IM commands.
- Focused tests/help checks pass.

## Verification Plan

- Use `rg` and targeted reads for `agentctl im` code/tests.
- Run relevant runtime/common tests and safe CLI help checks if available.

## Risks

- Live reply/read behavior may require services and credentials; this ticket verifies command contract, not full live IM delivery.

## Assumptions

- `agentctl im reply --message-file <path>` is the preferred reply path for long text.
