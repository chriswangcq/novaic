# Runtime Stale Backing Path Guard Check

## Summary

Success. R105 proves the runtime rejects copied `novaic-cortex-sandbox-*` backing paths before execution, gives stable replacement guidance, and now has a regression guard for quoted stale-path commands.

## Evidence

- `novaic-cortex/novaic_cortex/sandbox.py` defines `_reject_ephemeral_sandbox_paths` and calls it at the start of `ShellExecutionOrchestrator.exec`.
- Rejection returns `exit_code=-2`, empty stdout, no file changes, and stderr containing `/cortex/ro`, `/cortex/rw`, `$RO`, and `$RW`.
- Existing and newly added tests in `novaic-cortex/tests/test_sandbox_requires_mount_namespace.py` cover bare and quoted copied backing paths.
- Focused sandbox tests passed: 8 passed.

## Criteria Map

- Detect backing paths in commands: satisfied by full-command marker check and tests.
- Teach stable replacements: satisfied by stderr assertions for `/cortex/ro` and `/cortex/rw`, plus implementation message includes `$RO` and `$RW`.
- Focused tests for representative copied paths: satisfied by bare path and quoted path tests.
- No fallback translation: satisfied because the guard returns a rejection before sandbox executor configuration or execution.

## Execution Map

- R105 inspected runtime implementation and tests.
- R105 added the missing quoted-path regression test and reran focused sandbox tests.

## Stress Test

A plausible failure mode is an agent pasting a stale path inside quotes or inside a larger `printf`/script command. The new quoted-path test covers that shape and proves the guard is command-string based, not only path-token based.

## Residual Risk

Low for the known stale backing path marker. The guard intentionally does not attempt to recognize every possible temp directory pattern, because the contract-specific leak marker is `novaic-cortex-sandbox-*`.
