# Agentctl Subagent CLI Contract Audit Ticket

## Problem Definition

Subagent operations should be reachable through `agentctl subagent ...` from shell, especially `spawn`, so long-running or delegated work does not require stale direct harness tools.

## Proposed Solution

Inspect `agentctl subagent` implementation, help/schema mentions, and tests. Run focused tests/help checks, and patch any bounded discoverability or registration gap.

## Acceptance Criteria

- `agentctl subagent` implementation and `spawn` registration are located.
- Shell schema points agents to `agentctl subagent spawn`.
- Safe help checks or focused tests verify the command surface.
- Any missing/misleading command coverage is fixed or routed.

## Verification Plan

- Targeted `rg`/file reads for subagent command implementation and schema mentions.
- Focused common/Cortex tests plus generated capability-script help checks.

## Risks

- Live spawn requires service state; avoid creating actual subagents during this contract audit.

## Assumptions

- `spawn` is the required subagent CLI surface for this ticket; richer coordination can be added by separate tickets if specified.
