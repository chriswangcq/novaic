# Agentctl IM CLI Contract Audit Result

## Summary

Audited the `agentctl im` shell CLI contract. The implementation already supports `read`, `reply`, `send`, `history`, `search`, and `context`, but the LLM-facing shell schema only advertised `read`, `reply`, `history`, and `search`. Fixed the schema to include `agentctl im send` and `agentctl im context`, and added a guard test.

## Done

- Inspected `novaic-cortex/novaic_cortex/shell_capabilities.py` and confirmed `agentctl im` supports:
  - `read`
  - `reply`
  - `send`
  - `history`
  - `search`
  - `context`
- Inspected `novaic-common/common/tools/llm_builtin.py` and found shell description omitted `agentctl im send` and `agentctl im context`.
- Updated `novaic-common/common/tools/llm_builtin.py` to list the full intended IM command family.
- Added `test_shell_schema_lists_agentctl_im_command_family` in `novaic-common/tests/test_tool_definitions_contract.py` so this drift is caught.
- Ran safe generated-script help checks for `agentctl --help` and `agentctl im --help`.

## Verification

- `PYTHONPATH=novaic-common:novaic-cortex:novaic-logicalfs:novaic-sandbox-sdk pytest -q novaic-common/tests/test_tool_definitions_contract.py novaic-cortex/tests/test_shell_capabilities_internal_auth.py`
  - Result: `13 passed in 0.62s`.
- Generated capability scripts via `_write_capability_scripts(...)` and ran:
  - `agentctl --help` => exit 0, prints `agentctl - agent/session command surface inside the Cortex shell`.
  - `agentctl im --help` => exit 0, prints `Usage: agentctl im <read|reply|send|history|search|context> [options]`.

## Known Gaps

- This ticket did not exercise live IM delivery because that requires live service state and credentials; it verifies contract shape, command registration/help, and schema discoverability.
- Subagent and media `agentctl` command families are intentionally handled by sibling problems P109 and P110.
