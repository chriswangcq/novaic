# Agentctl Devicectl Cortex CLI Coverage Audit Ticket

## Problem Definition

Shell-first usability requires the shell-visible CLI layer to expose the intended agent, device, and Cortex actions without requiring direct LLM tools. The audit must verify `agentctl`, `devicectl`, and Cortex-related CLI commands are present, discoverable, and aligned with the terminal/artifact output contract.

## Proposed Solution

Inspect CLI entrypoints, command registrations, help output tests, and runtime shell capability documentation for `agentctl`, `devicectl`, and Cortex payload/workspace commands. Run focused tests that exercise the available CLI contract. If a missing or misleading path is found, either fix it directly if bounded or spawn a smaller follow-up problem.

## Acceptance Criteria

- Identify the code paths that define/register `agentctl`, `devicectl`, and Cortex shell commands.
- Verify intended capabilities are reachable through shell-visible CLI commands.
- Verify the commands are described in the shell tool surface or help surface.
- Verify no obvious stale/direct-tool-only capability remains for the intended shell-first surface.
- Record focused tests or exact gaps.

## Verification Plan

- Use `rg` and targeted file reads for CLI registrations, command help, and shell tool schema mentions.
- Run focused pytest suites around shell capabilities, tool surface boundary, CLI/device output contracts, and Cortex payload commands.
- If executable CLI smoke tests are cheap and local, run help/version/list commands without external side effects.

## Risks

- Some commands may depend on deployed service URLs; avoid side-effectful production calls during this audit.
- CLI coverage may span several repos/packages; if the surface is too broad, split into child problems instead of pretending one pass is exhaustive.

## Assumptions

- “Cortex-related commands” covers payload read/search/summarize/qa and workspace/RO/RW access documented for shell use.
- Device commands should use blob/artifact output for media-producing operations and terminal text for normal command output.
