# Active Docs Stable Path Residue Audit Ticket

## Problem Definition

Active docs must not instruct users or agents to copy `novaic-cortex-sandbox-*` backing paths as current Cortex workspace paths.

## Proposed Solution

Search `docs/` for stale Cortex backing path mentions, inspect surrounding context, rewrite active guidance if found, and record justified historical or negative mentions.

## Acceptance Criteria

- `docs/` contains no active operational guidance that tells agents to use copied `novaic-cortex-sandbox-*` backing paths.
- Any remaining doc mention is historical, negative, or roadmap context rather than current instructions.
- Stable path guidance points to `/cortex/ro`, `/cortex/rw`, `$RO`, and `$RW` where current guidance is needed.

## Verification Plan

Run `rg -n "novaic-cortex-sandbox|/tmp/novaic-cortex-sandbox|/cortex/ro|/cortex/rw" docs`, inspect hits, edit active stale guidance if needed, and rerun the search.

## Risks

- Roadmap docs may intentionally discuss prior bugs or rejected designs; deleting those would damage historical context.

## Assumptions

- Current user-facing operational docs live under `docs/`; ticket/roadmap docs may retain clearly historical context.
