# Devicectl Non-Media Command Coverage Audit Ticket

## Problem Definition

Non-media `devicectl hd` commands should remain reachable/discoverable through shell and should not require direct device tools. The command map, generated help, and shell schema should agree.

## Proposed Solution

Inspect `_HD_PROXY_PATHS`, generated help, shell schema description, and tests. Run safe help checks and focused schema tests. Add a guard test if help/schema drift is not already covered.

## Acceptance Criteria

- Non-media command map is inspected.
- Help/schema mention intended commands: shell-exec, mouse, keyboard, clipboard-get, clipboard-set, file-push, and related JSON forms.
- Focused tests/help checks pass.
- Any bounded drift is fixed.

## Verification Plan

- Read `shell_capabilities.py` and `llm_builtin.py` devicectl mentions.
- Run common/Cortex schema tests and generated `devicectl --help` / `devicectl hd --help` checks.

## Risks

- Actual mouse/keyboard/clipboard calls are side-effectful; use help/schema checks only.

## Assumptions

- Artifact-producing screenshot/file-pull behavior is already covered by P111.
