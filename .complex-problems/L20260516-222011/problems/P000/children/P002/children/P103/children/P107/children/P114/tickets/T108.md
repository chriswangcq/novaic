# Cortex Stable Workspace Path Contract Audit Ticket

## Problem Definition

The shell and Cortex workspace contract must consistently point agents at stable paths (`/cortex/ro`, `/cortex/rw`, `$RO`, `$RW`) and prevent agents from reusing ephemeral `novaic-cortex-sandbox-*` backing paths observed in old outputs. Prior incidents showed copied backing paths can break later turns and create confusing stale-path behavior.

## Proposed Solution

Audit the stable path contract across the LLM-facing shell schema, generated Cortex shell help, runtime shell command handling, tests, and active docs. Verify that stale backing paths are rejected or clearly discouraged, then add focused guards for any uncovered public boundary.

## Acceptance Criteria

- Shell schema and generated help explicitly recommend stable `/cortex/ro`, `/cortex/rw`, `$RO`, and `$RW` paths.
- Runtime shell command handling rejects or blocks direct use of stale `novaic-cortex-sandbox-*` backing paths where applicable.
- Focused tests cover stable-path guidance and stale-path rejection behavior.
- Any discovered gap is fixed without reintroducing fallback or compatibility branches.

## Verification Plan

Run targeted `rg` inspections, read the relevant shell schema/help/runtime code, run focused Cortex/runtime tests for stable path and stale backing-path behavior, and add regression tests if a guidance or guard gap is found.

## Risks

- A guard might be present in runtime but not visible in the schema/help, causing agent misuse despite correct runtime behavior.
- A doc could still teach the old ephemeral backing path pattern.
- Tests may only cover positive stable paths and miss stale-path abuse.

## Assumptions

- The desired contract is no backward compatibility for copied `/tmp/novaic-cortex-sandbox-*` backing paths.
- `$RO` and `$RW` may resolve to internal mounts, but user/agent-visible guidance should remain stable path first.
