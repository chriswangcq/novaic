# Agentctl CLI Coverage Audit Ticket

## Problem Definition

`agentctl` is now the primary shell-first interface for agent IM, reply, history/search, subagent spawning, and selected media/payload helper actions. It must be implemented, discoverable, documented in the shell tool schema, and covered by tests so agents do not fall back to removed direct harness tools.

## Proposed Solution

Inspect the `agentctl` CLI implementation and command registration, compare it with the shell tool description, scan tests for coverage, run focused unit/contract tests, and fix any bounded gap in command exposure or docs.

## Acceptance Criteria

- `agentctl` command implementation and registrations are located.
- Shell tool schema lists the intended `agentctl` commands.
- Focused tests or help checks prove the command surface is present.
- Any stale direct-tool-only or misleading `agentctl` surface found is fixed or explicitly routed to a follow-up.

## Verification Plan

- Use `rg`/targeted reads for `agentctl` code paths and shell schema mentions.
- Run focused runtime/common tests that guard tool surface and shell capabilities.
- Run safe local help/import checks if available without requiring live services.

## Risks

- Some `agentctl` actions need live queue/business services; avoid side-effectful calls and prefer help/unit coverage for this audit.

## Assumptions

- The audit is about CLI availability and contract shape, not live IM delivery behavior.
