# Agentctl Media Audio-QA CLI Contract Audit Result

## Summary

Audited `agentctl media audio-qa`. The implementation is shell-only, fetches audio from Blob Service, sends base64 only inside the Factory request payload, and prints a text result rather than binary media. Attachment guidance already points agents to shell `agentctl media audio-qa`. Added a common shell-schema guard test for the command.

## Done

- Inspected `novaic-cortex/novaic_cortex/shell_capabilities.py::_agentctl_media`.
- Confirmed `agentctl media audio-qa` accepts `--file-url` plus `--prompt`/`--prompt-file` and requires `blob://` input.
- Confirmed audio bytes are fetched from Blob Service and only encoded into the Factory `input_audio` request, not normal stdout.
- Inspected `novaic-agent-runtime/task_queue/utils/user_content.py` and tests; audio attachments prompt agents to call `agentctl media audio-qa` via shell.
- Added `test_shell_schema_lists_agentctl_media_audio_qa` in `novaic-common/tests/test_tool_definitions_contract.py`.
