# Agentctl Media Audio-QA CLI Contract Audit Ticket

## Problem Definition

Audio QA should be available through `agentctl media audio-qa` in shell, not through a normal direct LLM tool. The command must be discoverable, schema-advertised, and keep output within the shell/text artifact contract.

## Proposed Solution

Inspect `agentctl media audio-qa` implementation, shell schema mentions, user-content prompts for audio attachments, and tests. Run focused tests/help checks and add/fix guard coverage if needed.

## Acceptance Criteria

- `agentctl media audio-qa` implementation and help surface are located.
- Shell schema and attachment guidance mention the command agents should use.
- Direct `audio_qa` is not exposed as an active builtin tool.
- Focused tests/help checks pass; bounded drift is fixed.

## Verification Plan

- Read shell capability implementation and common tool schema tests.
- Run common/runtime tests for tool surface and user-content audio guidance.
- Run generated `agentctl media --help` check.

## Risks

- Live audio QA calls require blob/model services; this ticket avoids side-effectful model calls and verifies command contract shape.

## Assumptions

- Audio attachment guidance should continue to instruct agents to call `agentctl media audio-qa` via shell.
