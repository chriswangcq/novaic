# Cortex Payload CLI Coverage Audit Ticket

## Problem Definition

`cortex payload read/search/summarize/qa` must be available through shell, documented in help/schema, and backed by tests so large payload inspection stays out of direct LLM tools.

## Proposed Solution

Inspect payload CLI implementation/help/schema, run focused payload tests and generated help checks, and add/fix guard coverage if drift is found.

## Acceptance Criteria

- Payload CLI implementation and commands are located.
- Help/schema advertise read/search/summarize/qa.
- Focused tests/help checks pass.
- Bounded gaps are fixed.

## Verification Plan

- Read `shell_capabilities.py` payload code.
- Run Cortex payload inspection/shell schema tests.
- Run generated `cortex payload --help` check.

## Risks

- summarize/qa live calls require model config; command shape can be verified without live model calls.

## Assumptions

- The CLI commands delegate to Cortex payload APIs for actual read/search/summarize/qa behavior.
