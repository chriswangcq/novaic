## Problem Definition

Scripts, examples, and support files outside active docs/tests must not contain copyable `novaic-cortex-sandbox-*` commands or stale backing-path guidance.

## Proposed Solution

Search non-doc support surfaces for `novaic-cortex-sandbox` and stale backing path patterns, classify hits, and remove or rewrite active stale examples.

## Acceptance Criteria

- No repository script/example teaches copied `novaic-cortex-sandbox-*` paths as current workflow.
- Remaining mentions outside docs/tests are guard implementation or ledger/history artifacts, not active support scripts.
- Focused search evidence confirms allowed categories only.

## Verification Plan

Run repository-wide searches excluding `.complex-problems` when checking source, then inspect any support-file hits. Run a final `rg` summary to distinguish source hits from ledger history.

## Risks

- Ledger files can contain old evidence; these are not active support scripts and should not be edited as source.
- Some package lock/cache files may be irrelevant and should be excluded from source classification.

## Assumptions

- `.complex-problems` ledger history is an audit artifact, not active application code or user-facing guidance.
