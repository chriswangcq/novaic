# Shell Output and Desc Contract Audit Result

## Summary

Audited the runtime shell output contract and Agent Monitor `desc` projection path. The active implementation already keeps shell as terminal-style text only for LLM-visible output, bounds stdout/stderr previews, stores raw stdout/stderr only in durable payload, and projects `desc` only into monitor action/observation titles.

## Done

- Inspected `novaic-agent-runtime/task_queue/handlers/tool_handlers.py` around `_preview_shell_stream`, `_shell_result_text`, `_shell_result_output`, and `_exec_shell`.
- Confirmed shell stdout/stderr are previewed at `_SHELL_STREAM_PREVIEW_CHARS = 1600` each and final public text is bounded by `_SHELL_TEXT_LIMIT_CHARS = 4000` through `tool_text(...)`.
- Confirmed raw shell stdout/stderr/files_changed are kept under `durable_payload.raw`, not in the public `content` that should be carried as normal LLM-visible tool text.
- Inspected `novaic-common/common/tools/llm_builtin.py`; shell schema includes optional `desc` for Agent Monitor display.
- Inspected `novaic-agent-runtime/task_queue/utils/activity_projection.py`; shell `desc` is used for action/observation titles and does not alter command execution semantics.
- Inspected multimodal/display guard tests and runtime context utilities to verify shell and historical tool messages are not treated as image-bearing display perception.
