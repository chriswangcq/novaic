# Result: T018 ActivityTimeline behavior inventory

## Findings
- `ActivityTimeline.tsx` treats old direct IM names as normal product logic:
  - `BOOKKEEPING_TOOLS = {'im_read', 'im_reply', 'chat_reply', 'skill_end'}`
  - title mapping directly checks `im_read`, `im_reply`, `chat_reply`
  - reasoning mapping checks text containing `im_read` / `im_reply`
- It already has a current shell test (`shows shell monitor descriptions instead of low-level shell labels`), but message read/reply tests still use direct IM tool records.
- Focused tests using old direct IM records:
  - `novaic-app/src/components/Visual/ActivityTimeline.test.tsx`
  - `novaic-app/src/components/Visual/ActivityTimeline.acceptance.test.tsx`
  - `novaic-app/src/components/hooks/useActivityTimeline.test.ts`

## Implementation Targets
- Make shell/agentctl-shaped records the normal test path for message read and reply.
- Rename any necessary old-name handling to explicit legacy naming, e.g. `LEGACY_DIRECT_IM_TOOLS`.
- Keep public output stable: “读取了你的消息”, “已回复你”, “已保存本轮上下文”; hide raw low-level labels.
