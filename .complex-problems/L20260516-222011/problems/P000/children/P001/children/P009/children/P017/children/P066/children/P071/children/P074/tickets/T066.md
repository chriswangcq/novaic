# Audit business dispatch adapter compatibility residue

## Problem Definition

`novaic-business` dispatch/subscriber code may still contain migration-era, fallback, compatibility, TODO/FIXME, or direct-tool wording around IM aggregation, ownership, and queue handoff. These residues can hide active branching or leave misleading maintenance cues.

## Proposed Solution

Scan active `novaic-business` implementation and focused tests for residue terms and old direct-tool/dispatch vocabulary. Classify each hit as active-risk, safe comment cleanup, test-only guard, or unrelated domain term. Apply safe tiny cleanup directly where it removes misleading residue without changing runtime behavior, then run focused business tests for touched modules.

## Acceptance Criteria

- Active `novaic-business` code is scanned for legacy/fallback/compat/migration/TODO/FIXME and direct tool terms such as `im_read`, `im_reply`, `payload_read`, `audio_qa`, and `subagent_spawn`.
- Each hit is classified with a reason and active-path risk.
- Safe misleading comments or dead code are removed or rewritten to current-state wording.
- No broad behavior change is made without spawning a follow-up subproblem.
- Focused tests for touched business code pass, or any missing test path is explicitly recorded.

## Verification Plan

Run targeted `rg` scans over `novaic-business` implementation and tests, inspect all non-obvious hits, apply minimal cleanup, then run the relevant `pytest` subset for dispatch/subscriber code. Finish with a second residue scan over touched files.

## Risks

- Some terms may be legitimate product/provider compatibility concepts rather than old runtime compatibility paths.
- Business dispatch tests may depend on external service fixtures; if so, classify the missing verification and spawn a follow-up rather than guessing.

## Assumptions

- The intended current boundary is queue/FSM/shell-first dispatch, not direct agent harness tool dispatch.
- Comment-only cleanup is acceptable when it reduces misleading residue and does not change behavior.
