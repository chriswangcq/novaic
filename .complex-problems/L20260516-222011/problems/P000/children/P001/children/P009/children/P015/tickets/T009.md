# Split direct tool residue by capability family

## Problem Definition

Direct tool residue covers multiple old surfaces. IM tools, payload tools, and subagent/audio tools have different intended shell CLI replacements and different active-code risk.

## Proposed Solution

Split into child scans by capability family. Each child searches active code/tests/docs with ledger exclusions, triages hits, and applies tiny cleanup only when safe.

## Acceptance Criteria

- IM direct tool residue is checked separately.
- Payload direct tool residue is checked separately.
- Subagent/audio direct tool residue is checked separately.
- Current tool schema guard tests remain aligned with intended shell-only exposure.

## Verification Plan

- Use focused `rg` searches with exclusions for `.complex-problems`.
- Inspect current schema tests and tool definitions.
- Run targeted tests if any code changes are made.

## Risks

- Some direct names are valid inside CLI implementation or tests that assert absence; those must not be deleted blindly.

## Assumptions

- Current intended direct LLM tool set is `shell`, `display`, `skill_begin`, `skill_end`, and `sleep`.
