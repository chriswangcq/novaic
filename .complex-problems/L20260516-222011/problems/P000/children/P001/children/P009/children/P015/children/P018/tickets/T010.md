# Scan IM direct tool residue

## Problem Definition

IM capabilities should be shell CLI capabilities through `agentctl im ...`, not direct LLM-visible tools. Search for residue in active code/tests/docs and fix stale current-contract references if found.

## Proposed Solution

Run focused `rg` searches for IM tool names with `.complex-problems` excluded. Triage hits into current CLI implementation, guard tests, historical docs, active stale docs, or active code issue.

## Acceptance Criteria

- IM tool name references are searched with bounded output.
- Current intended references are identified.
- Any stale current-contract docs or active direct exposure are fixed.
- Targeted schema tests are identified or run if code changes occur.

## Verification Plan

- Search active tree excluding `.complex-problems`.
- Inspect relevant hits around `TOOL_SCHEMAS` and `shell_capabilities`.
- Check tool schema guard tests if needed.

## Risks

- Historical docs may intentionally mention retired tools; avoid deleting useful migration history unless misleading current instructions.

## Assumptions

- Direct LLM tool schema should not expose IM tools.
