# App test residue scan

## Problem Definition

Scan App tests for stale skip/TODO/FIXME/compat/fallback/legacy/base64/direct-tool fixtures that could preserve removed frontend behavior.

## Proposed Solution

Run bounded residue scans over `novaic-app/src` test/spec files, inspect hits, classify guard fixtures vs stale residue, clean small misleading fixture text where safe, and run focused App tests.

## Acceptance Criteria

- App tests scanned for residue markers.
- Hits classified as guard coverage, harmless fixture text, current product vocabulary, or stale residue.
- Tiny stale residue cleaned if safe.
- Focused App tests run.

## Verification Plan

- `rg` scans over `*.test.*`/`*.spec.*`.
- Focused `npm run test:unit` for touched tests.
