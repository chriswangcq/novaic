# Delete generated dashboard and ignore future snapshots

## Problem Definition

`dashboards/complex-problems-dashboard.html` is a tracked generated HTML snapshot of old `.complex-problems` data. It is large, stale, and pollutes residue searches with obsolete architecture terms.

## Proposed Solution

Delete the tracked generated dashboard file and update `.gitignore` to ignore generated dashboard HTML snapshots under `dashboards/`.

## Acceptance Criteria

- The tracked dashboard HTML file is deleted.
- `.gitignore` prevents future `dashboards/*.html` generated snapshots from being committed.
- `.complex-problems` ledger source remains untouched as the source of truth.
- Residue search for payload direct-tool names no longer hits `dashboards/complex-problems-dashboard.html`.

## Verification Plan

- Use `git ls-files`/`git status` to verify deletion.
- Use `git check-ignore` to verify future dashboard HTML ignore behavior.
- Rerun focused residue search excluding `.complex-problems` and confirm no dashboard hit.

## Risks

- If someone relied on the checked-in dashboard artifact, they must regenerate it with the ledger dashboard renderer instead. That is preferable to stale generated source.

## Assumptions

- Generated dashboard HTML is not a source artifact and can be regenerated.
