# Document final shell display blob output contract

## Problem Definition

`P065` needs the active runtime-facing docs to state the final output contract plainly. Developers and future agents should not infer from old examples that shell, display, or media tools may pass raw base64 or ephemeral sandbox paths through chat context.

## Proposed Solution

Update the active contract documentation, centered on `docs/runtime/tool-chain-dispatch.md`, to describe:

- shell output as bounded terminal text with full data available through Cortex step payload/RO inspection;
- media CLIs as `tool-output.v1` artifact/blob manifest producers, never raw base64 stdout producers;
- `display(blob://...)` as the explicit image perception path whose public tool text remains a placeholder/summary;
- stable `/cortex/ro`, `/cortex/rw`, `$RO`, and `$RW` paths as the only documented Cortex file paths.

Keep historical roadmap/ticket material untouched unless the focused scan finds it is being used as active guidance.

## Acceptance Criteria

- Active docs contain explicit terminal text plus blob/artifact guidance.
- Active docs warn against raw base64 in shell/tool text.
- Active docs warn against copying `novaic-cortex-sandbox-*` backing paths.
- Focused scans over active docs show the final wording and no active contradictory guidance.

## Verification Plan

- Inspect the edited active docs around the new contract text.
- Run focused `rg` scans for contract terms such as `bounded terminal text`, `tool-output.v1`, `blob://runtime-artifact`, `display`, `base64`, and `novaic-cortex-sandbox`.
- Run the ledger validation path after recording the result.

## Risks

- Over-editing historical docs could destroy useful design history.
- Under-specifying the display boundary could leave the same base64-context bug ambiguous for future agents.

## Assumptions

- `docs/runtime/tool-chain-dispatch.md` is the primary active doc for this contract.
- Historical roadmap/ticket docs are not the authoritative current contract unless a focused scan proves otherwise.
