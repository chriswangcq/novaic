# T065 Result — App monitor UI residue cleanup

## Summary

Cleaned the app monitor's remaining direct-tool compatibility residue. The ActivityTimeline now recognizes IM read/reply activity only through shell `agentctl` records, and the generic visual JSON renderer no longer upgrades inline base64/data-image strings into first-class image previews.

## Done

- Removed ActivityTimeline's legacy direct IM compatibility path; public monitor titles now recognize IM read/reply only through shell `agentctl im read` / `agentctl im reply` records.
- Removed the legacy direct `im_reply` compatibility test and added guard assertions so the ActivityTimeline source cannot reintroduce `LEGACY_DIRECT`, `legacyDirectTool`, or `isLegacyDirect` vocabulary.
- Removed raw base64/data-image rendering from `SmartValue`; visual JSON inspection now treats images as `blob://` or URL resources instead of accepting inline base64 media.
- Removed explicit `screenshot?: string // base64` fields from the frontend `ToolCallEvent` type, leaving media transfer to the blob/artifact contract.
- Reworded app-layer comments that described old migrations/fallbacks into current-state boundary wording.
