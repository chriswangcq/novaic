# Split app business MCP residue audit by adapter boundary

## Problem Definition

`P071` must scan App, Business, MCP, and scripts for stale fallback/compatibility/direct-tool residue without mixing UI compatibility, dispatch behavior, and CLI adapter behavior into one opaque scan result.

## Proposed Solution

Split by adapter boundary: App monitor/UI text, Business dispatch/subscriber logic, and MCP/scripts shell adapter surface. Each child should run focused scans, classify hits, and apply safe cleanup or route larger issues explicitly.

## Acceptance Criteria

- App, Business, MCP, and scripts are each scanned with bounded patterns.
- Hits are classified by active path status and risk.
- Safe tiny cleanup is applied directly.
- Touched areas receive focused tests/lint or explicit no-code-change verification.

## Verification Plan

- Run targeted `rg` scans per child boundary.
- Inspect active hits before editing.
- Run focused tests/lint for touched frontend/business/MCP areas.

## Risks

- UI test fixtures and localization strings can look like stale tool names but may be product wording or historical visual-regression coverage.
