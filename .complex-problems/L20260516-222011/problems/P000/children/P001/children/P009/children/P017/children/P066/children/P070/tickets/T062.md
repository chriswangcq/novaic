# Audit Cortex common residue and clean safe hits

## Problem Definition

`P070` must verify whether active Cortex/common code still contains stale fallback, compatibility, legacy, migration, TODO/FIXME, pass/skip, base64, or ephemeral-path residue around context projection, tool schemas, payload handling, and sandbox exposure.

## Proposed Solution

Run focused scans in `novaic-cortex` and `novaic-common`, inspect representative hits, classify them, and apply only tiny safe cleanup. Spawn child work if a larger active compatibility branch is found.

## Acceptance Criteria

- Focused scans cover the requested terms in Cortex/common active code and tests.
- Hits are classified by active path status and risk.
- Tiny high-confidence cleanup is applied directly if safe.
- Touched Cortex/common areas receive focused tests or explicit no-code-change verification.

## Verification Plan

- Run `rg` scans for fallback, compat, legacy, migration, TODO/FIXME, pass, skip, base64/data image, and ephemeral path terms.
- Inspect active hits before editing.
- Run focused Cortex/common tests for touched areas.

## Risks

- Some base64 fields are valid provider-native or display-perception payloads and must not be removed blindly.

## Assumptions

- Raw media/base64 is allowed only in explicitly structured image/perception payloads, not public shell/tool text.
