# Ticket: Clean ActivityTimeline legacy direct-tool assumptions

## Problem Definition
The shell-first contract moved IM/reply behavior behind `shell` + `agentctl im ...`, but ActivityTimeline still treats retired direct tool names (`im_read`, `im_reply`, `chat_reply`) as the normal product path. Tests also use those retired names as canonical examples.

## Proposed Solution
- Update ActivityTimeline projection logic so shell/agentctl-shaped records are the primary path for message read/reply public labels.
- If historical trace support is still needed, isolate old names behind explicitly named legacy helpers/constants.
- Update focused tests to use shell/agentctl examples for current behavior and keep only explicit legacy compatibility tests if necessary.

## Acceptance Criteria
- Public labels remain correct for shell/agentctl message read and reply records.
- Raw low-level names are still hidden from user-visible timeline.
- Old direct tool names do not appear as normal-path examples in ActivityTimeline tests.
- Any remaining old names in component code are behind explicit `LEGACY_*` naming.
- Focused ActivityTimeline tests pass.

## Verification Plan
- Inspect ActivityTimeline component and focused tests.
- Patch narrowly.
- Run ActivityTimeline vitest tests.
- Run focused grep to classify remaining old names.

## Risks
Removing all legacy handling could make old timeline records noisier. Keep compatibility only if it is explicit and isolated.
