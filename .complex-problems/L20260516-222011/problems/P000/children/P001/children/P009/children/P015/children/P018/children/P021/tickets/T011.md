# Update current IM docs and placeholder strings

## Problem Definition

Some current-facing docs and UI placeholders still present IM capabilities as direct LLM tools. This contradicts the current shell-first contract and can mislead agents or maintainers.

## Proposed Solution

Edit only current-facing docs/placeholders discovered by P018: runtime tool-chain/architecture docs, architecture pipeline docs, and UI placeholder strings. Leave historical roadmap tickets intact unless they claim current active state.

## Acceptance Criteria

- Edited current docs describe IM through shell `agentctl im ...`.
- UI placeholders use `shell, display` or `shell, display, skill_end` instead of `shell, display, im_read`.
- Focused `rg` searches confirm edited current surfaces no longer advertise direct IM tools.
- Changes are documentation/locale-only unless a test guard needs adjustment.
