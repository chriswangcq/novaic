# MCP/scripts shell adapter residue scan

## Problem Definition
After moving agent-facing interfaces into shell CLI paths, MCP helper scripts and adapters may still contain stale direct-tool, compatibility, or old base64/payload behavior. These residues can keep old paths alive or mislead future changes.

## Proposed Solution
Scan MCP/script/shell-adapter code for direct-tool and legacy-residue markers. Remove stale code where safe, rewrite misleading comments to current shell/CLI contracts, and run focused tests for touched adapters.

## Acceptance Criteria
- MCP/script/shell-adapter active code has no unclassified direct-tool or legacy-residue markers.
- Any remaining marker is classified as current behavior or guard-test vocabulary.
- Focused tests for touched adapters pass.

## Verification Plan
- Scan likely MCP/script/shell adapter locations for `im_read`, `im_reply`, `subagent_spawn`, `audio_qa`, `payload_read`, `direct tool`, `compat`, `fallback`, `TODO`, `base64`, and blob contract terms.
- Run focused tests for modified packages.
