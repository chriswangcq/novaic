# Classify App settings/model config residue

## Problem Definition

The settings/model config slice has non-monitor App hits around provider compatibility naming, legacy config comments, and settings labels. These must be classified and stale wording cleaned safely.

## Proposed Solution

Inspect `novaic-app/src/components/Settings`, `novaic-app/src/data/entities`, and related settings hooks/types for fallback/compat/legacy/provider naming residue. Rewrite stale comments where safe; leave current product/provider names intact with classification.

## Acceptance Criteria

- Relevant settings/model files are scanned and inspected.
- Each active hit is classified.
- Stale comments are cleaned if found.
- Focused App tests or explicit no-code-change verification is recorded.

## Verification Plan

- Run targeted `rg` before/after cleanup.
- Run focused App unit tests if code/test files change.
