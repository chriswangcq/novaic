# Ticket: Update Cortex notification hint to shell-first IM read

## Problem Definition
Active Cortex context projection still tells the LLM to call direct `im_read(...)` for Environment notifications. That violates the shell-first contract and can make the model call a tool that is no longer meant to be directly exposed.

## Proposed Solution
Patch `context_event_projection.py` notification hint text to instruct shell execution of `agentctl im read --notification-id ...`. Update focused tests or add coverage for the new hint.

## Acceptance Criteria
- Active notification hint text contains shell/agentctl guidance.
- Active notification hint text no longer contains direct `Use im_read(...)` guidance.
- Focused tests pass.

## Verification Plan
- Locate existing context projection tests.
- Patch code/tests.
- Run focused pytest and grep.
