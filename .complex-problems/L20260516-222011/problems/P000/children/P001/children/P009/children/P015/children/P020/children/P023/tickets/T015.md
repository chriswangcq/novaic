# Update vmuse long-command guidance

## Problem Definition

Vmuse shell tool descriptions and bundled app resource copies still tell agents to use direct `subagent_spawn(task=...)` for long-running commands. This is stale under the current shell-first NovaIC contract.

## Proposed Solution

Edit the source vmuse files and the two bundled app resource copies to describe background delegation generically and, for NovaIC shell environments, use `agentctl subagent spawn --task ...`.

## Acceptance Criteria

- No vmuse source/resource text recommends `subagent_spawn(task=...)` direct syntax.
- Long-command guidance remains useful and points to `agentctl subagent spawn --task ...` in NovaIC shell environments.
- Timeout warning avoids direct-tool syntax.
- Focused search verifies the edited surfaces.

## Verification Plan

- Patch source and bundled copies consistently.
- Run focused `rg` for `subagent_spawn` in vmuse source/resource paths.
- Review diff.

## Risks

- Vmuse may be used outside NovaIC; wording should not assume `agentctl` exists everywhere. Mention host-provided background executor generically, with NovaIC shell as the concrete example.

## Assumptions

- Bundled app resource copies are intentionally checked in and should be kept consistent for now.
