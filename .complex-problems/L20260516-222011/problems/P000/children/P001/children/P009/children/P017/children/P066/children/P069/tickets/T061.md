## Problem Definition

`P069` must verify whether active runtime/queue code still contains stale fallback, compatibility, legacy, migration, TODO/FIXME, pass/skip, or old direct-tool residue around session dispatch and tool output handling.

## Proposed Solution

Run focused scans in `novaic-agent-runtime` excluding caches. Inspect representative hits, classify them, and apply only tiny safe cleanup. If a hit reveals a larger active compatibility branch, spawn a follow-up child instead of hiding it.

## Acceptance Criteria

- Focused scans cover the requested residue terms in runtime active code and tests.
- Hits are classified by active path status and risk.
- Any tiny high-confidence cleanup is applied directly.
- If code changes happen, focused runtime tests pass; otherwise no-code-change verification is explicit.

## Verification Plan

- Run `rg` scans in `novaic-agent-runtime` for fallback, compat, legacy, migration, TODO/FIXME, pass, skip, direct tools, base64, and sandbox path terms.
- Inspect active hits before editing.

## Risks
