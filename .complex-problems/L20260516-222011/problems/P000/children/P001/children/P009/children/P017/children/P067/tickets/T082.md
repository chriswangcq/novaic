# Split test skip/TODO/fixture residue audit by test boundary

## Problem Definition

`P067` must scan active test directories for stale skip/TODO/FIXME/compat/fallback/legacy fixtures without allowing old behavior to remain acceptable through tests.

## Proposed Solution

Split by test boundary: Runtime/queue tests, App tests, and service/library tests (Business/Cortex/Common/MCP/scripts). Each child should run bounded scans, classify hits, and clean tiny stale residue while preserving intentional guard fixtures.

## Acceptance Criteria

- Active test directories are scanned for skip, xfail, TODO, FIXME, compat, fallback, legacy, and stale direct-tool/base64 fixture markers.
- Hits are classified as intentional guard coverage, stale acceptance, or harmless fixture text.
- Tiny stale test residue is cleaned directly when safe.
- Broad/risky rewrites are routed to child/follow-up work.

## Verification Plan

- Run targeted `rg` scans per test boundary.
- Inspect representative hits before editing.
- Run focused test commands for touched tests.
