# Capture repo layout and command entrypoints

## Problem Definition

The audit needs a bounded map of top-level repository layout and likely test/deploy entrypoints before deeper optimization work.

## Proposed Solution

Run bounded read-only commands to list top-level directories, project metadata files, deploy scripts, test config files, and service entrypoints.

## Acceptance Criteria

- Top-level directories/submodules are summarized.
- Likely test commands and project files are identified.
- Deploy/runtime scripts and service entrypoints are identified where visible.
- No implementation edits are made.

## Verification Plan

- Use `find`, `rg --files`, and selected file reads with bounded output.
- Record evidence pointers rather than dumping full files.
- Confirm no product file edits occur.

## Risks

- Entry point discovery may miss runtime commands encoded in external deployment tooling; record as residual risk.

## Assumptions

- Local files are sufficient for an initial entrypoint map.
