# Rerun robust per-submodule dirty-state capture

## Problem Definition

The previous per-submodule status command failed because `printf` interpreted a leading dash in the format string as an option. The dirty-state evidence needs to be recaptured with a robust command.

## Proposed Solution

Run `git submodule foreach` with a safe `printf '%s\n' "--- $name ---"` pattern, capture branch and `git status --short`, and record whether any dirty entries appear.

## Acceptance Criteria

- The command has clear submodule labels.
- The command exits without `printf` option errors.
- Branch and dirty-state output are captured for each submodule.
- The result supersedes the weak claim in R001.

## Verification Plan

- Inspect command stdout/stderr.
- Confirm labels are present and no `printf` errors occur.
- Record any dirty entries if present.

## Risks

- Output can be long; bound it to the current submodule set.

## Assumptions

- Read-only git status commands are safe.
