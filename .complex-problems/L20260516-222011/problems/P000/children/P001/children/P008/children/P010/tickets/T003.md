# Split git state capture by superproject and submodules

## Problem Definition

The git state fact capture should separate the superproject from submodule repositories because they have different dirty-state semantics and different commit responsibilities.

## Proposed Solution

Split into two child problems: superproject branch/dirty-state capture, and submodule pointer/dirty-state capture. Both are read-only and bounded.

## Acceptance Criteria

- Superproject state is captured independently.
- Submodule state is captured independently.
- Any dirty state is attributed to the correct repository boundary.
- No implementation edits are made.

## Verification Plan

- Run read-only git commands only.
- Confirm outputs are bounded.
- Record evidence in child results.

## Risks

- Some submodules may not have local branches; use `git submodule status` and per-submodule status without assuming branch policy.

## Assumptions

- Current workspace is available locally with submodules checked out.
