# Capture submodule git state

## Problem Definition

Submodule state must be captured separately from the superproject so later optimization edits and dirty-state markers can be attributed to the correct repository boundary.

## Proposed Solution

Run bounded read-only submodule commands: `git submodule status`, list submodule paths, and sample per-submodule branch/status for all submodules with compact output.

## Acceptance Criteria

- Submodule pointers are recorded.
- Per-submodule dirty state is checked with bounded output.
- Dirty or detached states are interpreted conservatively.
- No implementation edits are made.

## Verification Plan

- Use read-only git commands.
- Keep output bounded with `sed`/compact formatting.
- Record evidence and residual risks in the result.

## Risks

- Submodule branch names may not reflect deployment semantics; pointers are the primary evidence.

## Assumptions

- Submodule checkout paths are available locally.
