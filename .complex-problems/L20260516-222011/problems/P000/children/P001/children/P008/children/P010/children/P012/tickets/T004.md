# Capture superproject git state

## Problem Definition

The superproject git state must be captured before implementation edits so ledger side effects and code changes can be separated.

## Proposed Solution

Run bounded read-only git commands in the superproject: branch, top commit, status short, and relevant ledger directory status.

## Acceptance Criteria

- Current branch and HEAD are recorded.
- Superproject `git status --short` is recorded.
- Ledger-created files are distinguished from implementation edits.
- No code edits are made by this ticket.

## Verification Plan

- Run read-only git commands.
- Confirm no implementation file modifications are introduced by this ticket.
- Record commands and observations in the result.

## Risks

- Ledger creation itself changes `.complex-problems`; this must not be mistaken for product code dirtiness.

## Assumptions

- The current working directory is the superproject root.
