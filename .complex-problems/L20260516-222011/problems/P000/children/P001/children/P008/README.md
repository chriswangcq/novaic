# Workspace branch submodule and dirty-state inventory

## Problem

Collect a bounded, factual inventory of the workspace branch, submodule pointers, dirty state, top-level repos, and likely test/deploy entry points. This is evidence gathering only.

## Success Criteria

- Branch and dirty state are captured for the superproject and key submodules.
- Top-level repo/module layout is summarized with stable evidence.
- Test/deploy entry points are identified without running expensive commands.
- Result records no implementation edits.
