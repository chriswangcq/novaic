# Result: Blob Workspace Boundary Guardrail Wired into CI

## Summary

The Blob/Workspace boundary audit found that runtime boundary tests existed, but the repository-level lint workflow did not have a lightweight, explicit guard for the Blob-as-byte-store vs LogicalFS-as-workspace-authority contract. I added a root CI lint script that reuses the existing `novaic-cortex/cortex_tests/blob_boundary_policy.py` policy and wired it into `.github/workflows/lint.yml`.

## Done

- Added `scripts/ci/lint_blob_workspace_boundary.py` as a pure-stdlib repository-level guard.
- The new script scans live runtime Python sources under Cortex, LogicalFS, and Sandbox Service.
- The guard rejects direct Blob object authority (`BlobCortexStore`, `/v1/objects`) outside the explicit LogicalFS Blob adapter.
- The guard rejects BlobRef/Blob API references (`blob://`, `/v1/blobs`, `/v1/blobs/uploads`) outside the explicit allowed Cortex projection/display/payload flows.
- Wired the guard into `.github/workflows/lint.yml` immediately after the existing Cortex boundary lint.
- Confirmed the existing pytest guardrail remains green.
