# Tests docs and architecture guard audit result

## Summary

Closed the audit branch by splitting it into focused documentation, test-contract, and architecture-guard work. The work found and fixed concrete stale residue instead of only cataloging it: docs now describe the current Blob/LogicalFS/Cortex boundary more precisely, generated Python/cache artifacts were physically cleaned, and a stale runtime main-path guard was retargeted from a deleted handler to the current context handler path.

- Completed P659 / R656: audited docs for stale architectural claims, patched `docs/blob-service-architecture.md` to clarify `/v1/objects/*` as an internal object-store primitive rather than Cortex/shell live `/ro` `/rw` semantics, and patched `docs/cortex/object-keys.md` to use `/rw/subagents/{subagent_id}/scratch/...` instead of historical root `/rw/scratch/...` as the default Cortex/shell scratch contract.
- Completed P660 / R657: audited tests for stale contracts around Blob artifacts, display projection, shell terminal-text output, no historical image injection, and lower-layer LogicalFS arbitrary path behavior. High-risk hits were classified as current guards rather than stale test residue.
- Completed P661 / R664: inventoried architecture guards, built a guard gap matrix, cleaned generated artifact residue, reviewed guard false positives, and fixed the stale `scripts/ci/lint_agent_main_path_acceptance.sh` anchor that still referenced deleted `environment_tool_handlers.py`.
- Added and exercised `scripts/ci/lint_blob_workspace_boundary.py`, and wired the Blob workspace boundary lint into `.github/workflows/lint.yml`.
