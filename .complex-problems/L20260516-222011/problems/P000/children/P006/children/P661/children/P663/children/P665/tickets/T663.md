# Build Guard Gap Matrix

## Problem Definition

We need an explicit matrix that maps current architecture contracts to concrete guards before patching anything. This prevents both under-guarding and noisy guard proliferation.

## Proposed Solution

Use the P662 inventory and current contract list to build a guard matrix artifact. For each contract, record root CI guard coverage, module test coverage, gap status, and next action.

## Acceptance Criteria

- Matrix covers Blob/LogicalFS/Sandbox/Cortex/Runtime shell-display/FSM/deploy/generated artifact contracts.
- Each row cites guard files or says coverage is missing.
- Each row marks status as covered, covered-by-tests, missing, or stale/needs-fix.
- Artifact is saved under ledger tmp.

## Verification Plan

Inspect P662 inventory summary plus high-signal guard files and write a concise matrix. No tests required for matrix creation.

## Risks

- Matrix can be too optimistic if it only cites names; cite specific guard responsibilities.

## Assumptions

- Patch implementation is handled by sibling `P666`; this ticket only decides gaps.
