# Clean Generated Artifacts and Re-run Guard

## Problem Definition

`P665` found a concrete guard gap: generated Python/test artifacts are physically present in active trees. This violates the AI-era cleanup principle and causes `lint_generated_artifacts.sh` to fail even though the guard is correctly wired.

## Proposed Solution

Physically remove generated artifacts (`__pycache__`, `.pytest_cache`, `.pyc`, egg-info if present) from the active source/resource tree using a targeted cleanup command, then rerun `lint_generated_artifacts.sh` with `pipefail`. Do not modify source behavior.

## Acceptance Criteria

- Removes generated artifacts reported by `lint_generated_artifacts.sh`.
- Does not delete source files or user-authored artifacts.
- Reruns `lint_generated_artifacts.sh` successfully.
- Records cleanup command and guard output.

## Verification Plan

Run a pre-clean count, perform targeted deletion of generated artifacts, run `lint_generated_artifacts.sh` with `set -o pipefail`, and inspect `git status --short` to confirm only generated/untracked cache residue changed.

## Risks

- Deleting tracked generated files would be destructive if any are intentionally committed; verify via `git status` after cleanup.

## Assumptions

- Python cache directories and `.pytest_cache` are generated residues, not source of truth.
