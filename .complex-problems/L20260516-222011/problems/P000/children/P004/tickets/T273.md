# Ticket: Queue FSM session and worker boundary audit

## Problem Definition

Audit and optimize queue service/session runtime boundaries after the FSM migration. The core risk is that new FSM infrastructure exists but old imperative dispatch/finalize/worker branches may still be active or confusing, leading to hidden state bugs and AI-era residue.

## Proposed Solution

Split the audit into bounded branches:

- map current queue/FSM entry points and worker roles;
- inspect session state SSOT, active session view/cache, generation checks, and durable outbox behavior;
- identify and remove/tighten active direct-path branches or compatibility shims;
- verify dispatch/session/finalize behavior with focused tests.

If residue is discovered, fix it rather than leaving dormant compatibility code.

## Acceptance Criteria

- Queue/FSM entry points and worker roles are mapped with file references.
- Residual direct-path branches or compatibility shims are classified.
- High-confidence stale/unsafe residue is removed or tightened.
- Focused tests cover any modified dispatch/session/finalize behavior.
- Any large or ambiguous issue is split into child problems instead of one-go.

## Verification Plan

Use `rg`, targeted file reads, focused pytest selections for queue/session/FSM/outbox/finalize tests, and final static scans for direct legacy paths.
