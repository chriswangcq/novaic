# Finalize generation aggregate regression verification

## Problem Definition

The targeted finalize/session-ended fixes must be proven as a closed boundary across repository, outbox, runtime handlers, recovery/watchdog, restart, and Cortex archive diagnostics. This ticket does not assume the previous child tickets are sufficient; it performs an aggregate regression audit and creates follow-up child problems for any uncovered unsafe path or stale compatibility residue.

## Proposed Solution

Build and execute a skeptical verification matrix:

1. Map runtime surfaces that can finalize, archive, clear active state, restart pending input, or recover a session:
   - `queue_service/session_repo.py`
   - `queue_service/session_fsm.py`
   - `queue_service/session_recovery.py`
   - `queue_service/session_outbox.py`
   - `task_queue/handlers/session_handlers.py`
   - `task_queue/handlers/cortex_handlers.py`
   - `task_queue/sagas/wake_finalize.py`
   - `task_queue/utils/cortex_bridge.py`
2. Run focused pytest suites for finalize ownership, session FSM, outbox cutover, attach/finalize generation checks, watchdog/recovery, restart/pending input, and legacy cleanup.
3. Run Cortex focused tests for scope archive diagnostics, context event lifecycle, projection, and write authority.
4. Run source guards for:
   - optional or missing generation in finalize/session-ended mutation paths
   - `int(generation)` / `int(session_generation)` coercion
   - direct active-session clearing outside the repository/FSM boundary
   - fallback generation defaults or current-active lookup in archive/finalize diagnostics
   - remaining-stack/archive fields being synthesized from implicit active state
5. Inspect any guard hit manually. If a hit is unsafe or ambiguous, create a follow-up child problem instead of claiming success.
6. Close only if stale or missing finalize/session-ended payloads cannot clear, restart, or archive a newer active generation.

## Acceptance Criteria

- Runtime and Cortex focused regression suites pass with the current code.
- The verification matrix explicitly covers repository, FSM, outbox, runtime handlers, Cortex bridge, watchdog/recovery, restart/pending input, and Cortex archive diagnostics.
- Source guards have been run and every hit is classified as safe, fixed, or converted into a follow-up child problem.
- No live finalize/session-ended mutation path accepts missing, bool, stale, or implicitly looked-up generation for active clearing, restart, or archive diagnostics.
- No old compatibility path silently defaults finalize/session-ended generation to `1` or current active generation.

## Verification Plan

- Run targeted `pytest` in `novaic-agent-runtime` with `PYTHONPATH=.:../novaic-common`.
- Run targeted `pytest` in `novaic-cortex` with `PYTHONPATH=.:../novaic-common:../novaic-logicalfs:../novaic-sandbox-sdk`.
- Use `rg` source guards over `queue_service`, `task_queue`, and Cortex lifecycle files.
- Record exact commands and outcomes in the result body.
- If any guard remains suspicious, create a follow-up child problem before marking this ticket complete.

## Risks

- Aggregate guards can surface false positives; each must be classified with file evidence, not hand-waved.
- Some compatibility names may remain in tests or comments; they are safe only if no live path consumes them.
- A broad test list can be slow, but this is the final verification layer and should bias toward certainty.

## Assumptions

- Historical backward compatibility for unsafe missing-generation finalize payloads is not required.
- Prior child problems already handled targeted implementation fixes; this ticket is allowed to add small fixes only if verification finds concrete residue.
- If the verification surface expands beyond bounded aggregate testing, the correct move is to split or spawn a child problem.
