# Cortex finalize mutation identity guards

## Problem Definition

`wake_finalize` can emit mutating tasks after a session has moved on. `session.ended` is now generation-checked, but the Cortex-facing and Business-status side effects still need explicit identity guards before they archive scopes, mark notifications processed, or move subagents to sleeping/completed. The dangerous cases are stale finalize tasks that still carry a valid old `scope_id` but should not mutate the current active wake/session.

## Proposed Solution

1. Inspect the live mutation handlers and payload builders:
   - `task_queue/sagas/wake_finalize.py`
   - `task_queue/handlers/cortex_handlers.py`
   - `task_queue/handlers/subagent_handlers.py`
   - relevant bridge/client payload contracts.
2. Split implementation if needed into the smallest guardable paths:
   - Cortex `scope_end` identity contract and handler checks.
   - Subagent sleeping/completed identity contract and handler checks.
   - Tests and residue guards for stale/missing identity.
3. Prefer explicit payload fields over hidden reads:
   - expected wake scope / root scope identity.
   - positive session generation where the downstream handler can verify it.
   - explicit skip/reject result when identity is missing or stale.
4. If a Cortex mutation is provably safe because it is structurally keyed by immutable scope path rather than active session state, document that evidence in the result/check instead of adding unnecessary plumbing.

## Acceptance Criteria

- `cortex.scope_end` cannot archive or acknowledge messages for a stale/missing identity payload.
- Subagent sleeping/completed handlers cannot mutate Business status from a stale/missing finalize identity payload, or the design documents why those mutations are harmless and structurally scoped.
- Tests cover missing identity and stale identity for every mutating path touched.
- No compatibility fallback silently treats missing generation/scope identity as valid.
- Source/residue guards show no old unguarded finalize mutation branch remains in the inspected paths.

## Verification Plan

- Focused unit tests for Cortex handler missing/stale identity.
- Focused unit tests for subagent handler missing/stale identity.
- Existing finalize suites:
  - `tests/test_pr254_finalize_ownership.py`
  - `tests/test_pr264_session_finalize_fsm_boundary.py`
  - `tests/test_pr48_turn_finalizer.py`
  - `tests/test_finalize_summary_boundary.py`
- Source grep for unguarded finalize mutation payload paths.

## Risks

- Cortex scope archive may be path-keyed and not have easy access to current session generation; if so, forcing a generation check in the wrong layer could add fake safety. Inspect first.
- Business subagent status may be coarse-grained rather than wake-scoped; stale finalize may still be observable as status flapping and likely needs a guard or redesign.
- This may need splitting because Cortex and Business status mutations have different authority boundaries.

## Assumptions

- Stale finalize side effects should fail closed or no-op; they should not be accepted through missing identity defaults.
- P349 already ensures upstream React-generated finalize contexts carry positive generation.
