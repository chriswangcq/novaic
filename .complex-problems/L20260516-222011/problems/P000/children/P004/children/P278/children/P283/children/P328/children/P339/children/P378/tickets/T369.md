# Runtime finalize generation regression test pass

## Problem Definition

Run a focused runtime regression pass proving finalize/session-ended generation ownership holds across repository, FSM, outbox, handler, recovery/watchdog, restart, pending input, and legacy cleanup paths.

## Proposed Solution

1. Run focused `py_compile` over changed runtime modules that participate in finalize/session-ended generation handling.
2. Run focused runtime pytest suites covering:
   - finalize ownership
   - session finalize FSM boundary
   - session recovery boundary
   - recovery outbox cutover
   - suspected-dead recovery
   - active inbox dispatch / restart behavior
   - attach and session outbox generation boundaries
   - saga compensation outbox cutover
3. Run runtime source guards for unsafe live patterns:
   - `int(generation)` and `int(session_generation)` coercion
   - missing/optional generation in finalize/session-ended mutation paths
   - direct active-session clearing outside repository/FSM ownership
   - legacy fallback generation defaults
4. Fix any concrete runtime issue found in this pass, or spawn a child problem if the fix is too large for this ticket.

## Acceptance Criteria

- Runtime focused tests pass.
- Runtime source guards are recorded and classified.
- No live runtime finalize/session-ended path accepts bool, missing, or stale generation for active clearing, restart, or archive-triggering outbox effects.
- Any remaining guard hits are explained with evidence or split into follow-up work.

## Verification Plan

- `python3 -m py_compile ...` in `novaic-agent-runtime`.
- `PYTHONPATH=.:../novaic-common pytest -q ...` focused runtime test list.
- `rg` source guards over `queue_service`, `task_queue`, and runtime tests.

## Risks

- Runtime tests may include safe compatibility fixtures that look like live residue; classify with file context before judging.
- A broad runtime test pass can uncover unrelated flakes; separate those as follow-up if not part of generation ownership.

## Assumptions

- Backward compatibility for missing-generation finalize payloads is not required.
- The runtime DB/state authority is already in `SessionRepository` and FSM components; this ticket verifies the boundary rather than redesigning it.
