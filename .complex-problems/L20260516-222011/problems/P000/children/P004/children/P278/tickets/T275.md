# Ticket: Session state SSOT outbox and generation boundary audit

## Problem Definition

Audit the queue session state ownership boundary after the FSM migration. The target is to verify whether `tq_session_state`, session input/inbox events, session outbox rows, generation checks, and any remaining active-session cache/view behavior match the intended model: explicit FSM decisions, durable inbox/outbox, generation-checked attach/finalize, and no hidden compatibility paths.

## Proposed Solution

Inspect the session schema, repository, ledger, outbox dispatcher, observed event handler, rebuild/recovery helpers, routes, and tests. Produce a boundary audit that maps state ownership and identifies any residual risks or cleanup candidates. If the audit finds concrete stale code or unsafe compatibility paths that can be removed safely, handle them through follow-up tickets rather than mixing broad cleanup into this audit ticket.

## Acceptance Criteria

- Map `tq_session_state`, session ledger/event/outbox, and any active-session cache/view roles with file references.
- Explain where session generation is created, checked, and advanced.
- Identify whether direct state mutation or side effects happen outside explicit FSM/outbox boundaries.
- Classify residual compatibility paths as safe, risky, or removable.
- Do not modify production code in this audit ticket unless the ledger later creates a concrete follow-up implementation ticket.

## Verification Plan

Use `rg`, schema inspection, and targeted reads over `novaic-agent-runtime/queue_service`, route code, runtime worker code, and queue/session tests. The result should be evidence-backed and conservative about any gaps.
