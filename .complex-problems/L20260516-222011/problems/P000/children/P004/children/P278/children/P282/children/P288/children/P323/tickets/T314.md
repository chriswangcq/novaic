# Rebuild and pending projection writer audit

## Problem Definition

Map session rebuild and pending-input projection writers to verify they update the intended session ledger/state model and cannot drift as independent authoritative state.

## Proposed Solution

Inspect `session_rebuild.py`, `session_projection.py`, and `SessionRepository` calls to pending projection/rebuild methods. Classify source events/tables and target state/projection.

## Acceptance Criteria

- Rebuild/projection functions are listed with file references.
- Their source tables/events and target state/projection are identified.
- Any independent cache/view that can drift from session ledger authority is flagged.

## Verification Plan

- Run source searches for rebuild/projection method names.
- Inspect source slices around writes.

## Risks

- Pending projection may be derived but still act like cache; inspect how it is written and consumed.

## Assumptions

- Rebuild should project running sagas into session_state; pending projection should be derived from input events and not an independent authority.
