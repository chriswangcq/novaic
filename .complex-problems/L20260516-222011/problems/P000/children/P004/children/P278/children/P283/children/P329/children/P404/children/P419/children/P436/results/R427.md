## Remaining Bridge Endpoints And Ownership

- `/v1/context/prepare_for_llm`: live authoritative LLM context preparation endpoint. It is backed by Cortex ContextEvent read model.
- `/v1/context/read`: materialized `context.jsonl` projection read endpoint for projection/context task maintenance and debugging, not LLM prompt assembly.
- `/v1/context/append`: materialized projection append endpoint used by runtime context task handlers for notification/projection maintenance, not authoritative LLM history.
- `/v1/context/batch`: materialized projection batch append endpoint used when initializing/writing wake projection context, not authoritative LLM history.
- `/v1/payload/*`: bounded explicit payload inspection/readback for CLI/debug use.
- `/v1/steps/read_formatted`: bounded tool-result projection endpoint for current-turn/historical tool result formatting.
