# Result: Finalize restart atomicity design

Design:
- Keep `session_ended(...)` Step 1 as the authoritative global transaction for finalize acceptance and pending projection derivation.
- If no pending projection exists, keep the current in-transaction `SESSION_CLOSED` transition and return `session_closed`.
- If pending projection exists, do not leave the transaction before recording restart intent.
- Inside the same transaction:
  - derive `pending` from projection;
  - allocate `new_scope_id` via injected provider;
  - compute `session_generation` via `self.session_ledger.next_generation(session_key)` after `record_session_finalized(...)` has advanced/cleared current generation within the same transaction;
  - build `restart_plan` with `build_restart_wake_plan(...)`;
  - record `SESSION_RESTART_PENDING` via `_record_session_transition_in_current_transaction(...)` with state `starting`, active scope `new_scope_id`, and `outbox_effect=restart_plan.outbox_effect`;
  - assert at least one outbox id was recorded;
  - record pending projection in the same transaction with `active_scope_id=new_scope_id` and marker `restart_pending:<scope>`.
- Return the same result shape with `delivery=outbox_pending` and `outbox_id` after transaction commit.

Preserved boundaries:
- No direct publish from `session_ended`; session outbox worker still owns wake creation side effect.
- No hidden clock/id: uses existing injected `self._clock()` and `self._new_scope_id()`.
- No compatibility branch: old after-transaction restart path should be removed.

Verification:
- Source guard should reject `_record_session_transition_after_transaction` in the pending restart block.
- Focused tests should include existing restart outbox cutover tests plus a new/updated test proving restart outbox exists immediately after `session_ended(...)` and state is `starting`, not externally dispatchable `no_active`.
