# Result: Session restart transaction flow audit

Audited restart and recovery-start paths.

Findings:
- Ordinary start and suspected-dead recovery start both go through `_queue_start_wake_transition(...)`, which builds the wake plan and records durable session outbox effects.
- Direct saga creation is not in `session_repo.py`; saga creation lives in `SessionOutboxDispatcher._publish_create_wake_saga(...)`, i.e. the durable session outbox side-effect boundary.
- Recovery archive side effect is represented as a session outbox effect; direct `queue_recovery_scope_end_task` path is absent from both session repo and saga repo.
- Finalize pending restart atomicity issue found in P303 has already been fixed by P306; restart path now records pending restart state/outbox inside the finalize transaction.

Source guard run:
- No `.orchestrator.create(` in session repo.
- `self.saga_orchestrator.create(` exists only in session outbox dispatcher.
- `queue_recovery_scope_end_task` absent from session repo and saga repo.
- `build_dispatch_wake_plan(` and `build_restart_wake_plan(` are present as explicit plan builders.

No new gap found in this restart/recovery audit slice.
