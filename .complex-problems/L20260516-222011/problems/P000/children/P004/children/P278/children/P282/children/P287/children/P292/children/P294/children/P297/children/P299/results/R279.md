# Result: Dispatch start-wake helper implementation

## Summary

Implemented the dispatch start-wake helper extraction in `novaic-agent-runtime/queue_service/session_repo.py`. Ordinary no-active dispatch and recovery dispatch now both call `_queue_start_wake_transition(...)`, and `build_dispatch_wake_plan(...)` appears only inside that helper.

## Changes Made

- Added `_queue_start_wake_transition(...)` to centralize:
  - `START_WAKE` FSM decision validation;
  - `build_dispatch_wake_plan(...)` call;
  - optional recovery archive effect creation;
  - durable `DISPATCH_WAKE_START_QUEUED` transition recording;
  - outbox id propagation into the returned result.
- Replaced ordinary no-active inline start-wake construction with a helper call using `record_in_current_transaction=True`.
- Replaced recovery/after-transaction inline start-wake construction with a helper call using `record_in_current_transaction=False`.
- Updated `_record_session_transition_in_current_transaction(...)` to return outbox IDs from `session_ledger.record_transition(...)`, so the helper can enforce required outbox recording in both modes.
