# Patch attach worker-only cutover test failures

## Problem Definition

The attach worker-only cutover has two focused failures: dispatch logging still expects `task_id` even though attach dispatch now returns an outbox-pending result, and a source-boundary test still asserts an older wake outbox wrapper shape. These failures prevent the cutover from being considered complete.

## Proposed Solution

Patch the stale attach dispatch logger to report `outbox_id` and `delivery`, update stale comments if they imply eager publish, and update the wrapper-boundary test to assert the current generic outbox wrapper shape. Then rerun correct py_compile and focused pytest commands from the `novaic-agent-runtime` directory.

## Acceptance Criteria

- Attach dispatch no longer reads `result["task_id"]` on the worker-only path.
- Attach dispatch result semantics remain `delivery=outbox_pending` plus `outbox_id`.
- Boundary tests reject old eager attach publish wrappers and assert the current generic outbox wrapper shape.
- Correct focused py_compile passes for `queue_service/session_repo.py` and `queue_service/session_outbox.py`.
- Focused tests pass for attach cutover and session outbox wrapper boundary.

## Verification Plan

- Run `python3 -m py_compile queue_service/session_repo.py queue_service/session_outbox.py` from `novaic-agent-runtime`.
- Run `PYTHONPATH=.:../novaic-common pytest -q tests/test_pr248_attach_outbox_cutover.py tests/test_pr281_session_outbox_wrapper_boundary.py` from `novaic-agent-runtime`.
- If those pass, run the broader session FSM/outbox focused suite before closing the parent verification.

## Risks

- Updating tests could accidentally weaken the intended guard instead of matching the current architecture.
- Removing `task_id` assumptions must not hide another legacy synchronous publish dependency.

## Assumptions

- Attach delivery should be owned by the durable session outbox worker, not by repository eager publish.
- The generic outbox wrapper accepting an effect list is the intended source shape for wake creation.
