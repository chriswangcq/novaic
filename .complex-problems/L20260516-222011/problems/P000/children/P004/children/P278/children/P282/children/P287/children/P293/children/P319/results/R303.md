# Old session wrapper and compatibility branch scan result

## Summary

Scanned old wrapper/helper names, active-session compatibility terms, generation guards, and mutation shortcuts. Dangerous publish wrapper names are gone from production behavior, but one production helper name remains misleading: `_publish_attach_request_after_transaction` no longer publishes externally and should be renamed to reflect durable outbox recording.

## Done

- Searched old wrapper/helper names: `_append_session_outbox_after_transaction`, `_publish_session_outbox_effect`, `_publish_wake_creation_outbox_effect`, `publish_attach_input_effect`, `publish_wake_creation_effect`, and `.append_effect(`.
- Searched active-session/generation compatibility terms across `queue_service` and `task_queue`.
- Inspected generation validation paths in `task_queue/handlers/session_handlers.py` and `task_queue/handlers/runtime_handlers.py`.
- Inspected `session_repo.py` around active state projection and attach outbox recording.

## Verification

- Old wrapper/helper names appear only in negative test guards (`test_pr281_session_outbox_wrapper_boundary.py`, `test_pr251_wake_creation_outbox_cutover.py`), not in production queue-service files.
- `tq_active_sessions` has no production queue-service/task_queue hits from P318 evidence.
- `generation is None` paths in handlers are strict validation (`Missing required field`) rather than fallback compatibility.
- `session_outbox.py` owns actual publish/ack/failure handling through `drain_pending`, `publish_effect`, `_publish_and_ack`, and `_publish_attach_input`.

## Known Gaps

- Production method `_publish_attach_request_after_transaction(...)` in `session_repo.py` is now a stale name. It records a transition and outbox effect, returns `delivery=outbox_pending`, and does not publish externally. The name preserves old mental model residue and should be renamed to an outbox-recording term.
