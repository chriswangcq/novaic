# Direct session SQL table access scan

## Problem Definition

Search for direct SQL/table access to session-related tables and classify whether matches belong to legitimate repository/adapter code or risky active paths outside the intended boundary.

## Proposed Solution

Use targeted `rg` patterns for session table names and SQL verbs, inspect production source slices for ambiguous hits, and record classification with file references.

## Acceptance Criteria

- Search patterns and match counts are recorded.
- Production matches are classified by intended owner module.
- Any risky production direct SQL outside repository/ledger/outbox adapter ownership is explicitly flagged.

## Verification Plan

- Run `rg` over `novaic-agent-runtime` for session table names and SQL verbs.
- Inspect relevant source files around matches.
- Classify docs/tests separately from production runtime files.

## Risks

- False positives in tests/docs can obscure active code; classify by path and source slice.

## Assumptions

- Direct SQL is legitimate inside session repository/ledger/outbox adapter layers, but not in unrelated business handlers or workers.
