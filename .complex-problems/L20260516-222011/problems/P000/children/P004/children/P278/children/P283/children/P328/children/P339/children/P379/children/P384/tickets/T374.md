# Classify Cortex archive diagnostics source guards

## Problem Definition

Run and classify Cortex source guards for archive diagnostics generation, hidden active-state lookup, and top-level versus nested remaining-stack semantics.

## Proposed Solution

1. Run `rg` guards over `novaic_cortex/api.py`, `context_event_writer.py`, `context_event_projection.py`, and focused tests for:
   - bool or `int(session_generation)` coercion
   - archive diagnostics generated from active/current session state
   - `remaining_stack` semantic/projection confusion
   - `archive_diagnostics` payload persistence and consumption
2. Inspect every hit and classify it as safe, fixed, or follow-up-worthy.
3. Fix small concrete residue if found and rerun focused tests.

## Acceptance Criteria

- Source guards are run and recorded.
- Every guard hit is classified with file-level evidence.
- No live Cortex archive diagnostics path synthesizes diagnostics from hidden active state.
- No live Cortex path coerces bool diagnostics generation.
- Projection keeps top-level `remaining_stack` semantic and nested `archive_diagnostics.remaining_stack` diagnostic-only.

## Verification Plan

- `rg` source guards in `novaic-cortex`.
- Targeted `sed`/`nl` context reads for hits.
- Focused pytest rerun only if code changes are needed.

## Risks

- `remaining_stack` appears in both semantic projection and diagnostics payload; classification must distinguish those deliberately.

## Assumptions

- Cortex persists diagnostics; runtime owns the source of the diagnostic values.
