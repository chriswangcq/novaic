# Ticket: Audit session schema and state ownership

## Problem Definition

Determine the authoritative session state model in Queue Service by inspecting schema definitions, repositories, projections, rebuild paths, and any active-session compatibility tables or views.

## Proposed Solution

Read queue schema and session repository/ledger/projection/rebuild code. Search for `tq_session_state`, `tq_active_sessions`, `session_input`, `pending`, and related active-session accessors. Produce a state ownership map and identify drift risk or cleanup candidates.

## Acceptance Criteria

- Session-related tables are listed with roles and file references.
- The authority relationship between `tq_session_state`, session events/input events/outbox, and `tq_active_sessions` is stated explicitly.
- Any duplicate live pointer/cache/view behavior is classified as required, safe cache/view, risky, or removable.
- No production code is changed by this audit ticket.

## Verification Plan

Use read-only `rg`, schema inspection, and targeted reads of queue schema, session ledger/repo/projection/rebuild files and relevant tests.
