# Ticket: Audit and clean test/migration compatibility residues

## Problem Definition

Tests, fixtures, migration-like files, and compatibility-named code can preserve unsafe old semantics even after runtime cleanup. Audit these residues so they do not document, assert, or revive missing-generation/stale-active compatibility behavior.

## Proposed Solution

- Inventory compatibility/generation/legacy hits in tests, fixtures, migration-like files, and compatibility-named modules.
- Delete or rewrite tests that assert missing/stale generation success.
- Classify retained historical references as non-live guard/documentation only.
- Add or strengthen source guards where a retained test should prevent reintroduction.
- Run focused tests/guards.

## Acceptance Criteria

- Test/fixture/migration-like hit inventory is saved.
- Any test asserting unsafe missing/stale generation success is removed or rewritten.
- Retained compatibility references are explicitly classified as non-live guard/docs/tests.
- Focused tests/guards pass.

## Verification Plan

- Run targeted `rg` scans over tests and migration-like paths.
- Inspect suspicious test names and assertions.
- Patch if live unsafe assertions are found; otherwise record classification with guard evidence.
