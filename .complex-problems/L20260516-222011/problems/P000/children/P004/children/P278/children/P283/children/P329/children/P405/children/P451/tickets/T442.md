# Ticket: Audit migration-like and compatibility-named source files

## Problem Definition

Find migration-like or compatibility-named source files that may retain old behavior or misleading names after runtime cleanup.

## Proposed Solution

- Inventory source paths and code hits for migration, legacy, compat, fallback, backcompat, old shape, and infer terms outside tests/ledger/caches.
- Classify each retained category as live current boundary, guard script, docs/comment, or unresolved risk.
- Patch/rename only if a live misleading compatibility path is found.

## Acceptance Criteria

- Source scan artifacts are saved.
- Retained migration/compatibility hits are classified.
- No unclassified live compatibility-named source remains.

## Verification Plan

- Inspect saved source scans and representative files.
- Record classification and residual risk.
