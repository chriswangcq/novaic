# Result: After-transaction publish/outbox audit found eager attach delivery residue

Mapped session side-effect delivery paths:
- Wake creation and recovery archive side effects are recorded into session outbox and delivered by `SessionOutboxDispatcher`, normally through `session_outbox_worker` or explicit drains in tests.
- Active attach path records a durable outbox row, but `SessionRepository._publish_attach_request_after_transaction(...)` then immediately calls `self.outbox_dispatcher.publish_attach_input_effect(outbox_id)`.

Safety assessment:
- This is not a data-loss bug: the outbox row is recorded before publish, and publish failure leaves the outbox pending/retryable.
- But it is still architectural residue: delivery has two owners, repository eager-drain and session-outbox-worker. That keeps business coordination code coupled to effect execution and makes worker ownership less crisp.

Gap:
- Remove repository eager attach publish. Dispatch should record durable attach outbox and return `delivery=outbox_pending`; session-outbox-worker should own publish/retry consistently.
