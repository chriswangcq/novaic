## Tables and Key Columns

`novaic-agent-runtime/queue_service/db/schema.py` defines the session ledger DDL around lines 342-411:

- `tq_session_events`
  - Role from schema comment: append-only session input/observation events.
  - Key ownership columns: `id`, `session_key`, `agent_id`, `subagent_id`, `user_id`, `generation`, `event_type`, `payload`, `idempotency_key`, `consumed_at`, `created_at`.
  - Indexes: session/generation/created_at, event_type/created_at, unique idempotency.
- `tq_session_state`
  - Role from schema comment: materialized session FSM state and the only session coordinator source of truth for active/no_active/ending/recovery state.
  - Key ownership columns: `session_key` primary key, `state`, `generation`, `agent_id`, `subagent_id`, `user_id`, `active_saga_id`, `active_scope_id`, `recovery_id`, `heartbeat_at`, `updated_at`.
  - Index: state/updated_at.
- `tq_session_outbox`
  - Role from schema comment: durable side-effect ledger created in the same transaction as state transition and dispatched by outbox worker.
  - Key ownership columns: `id`, `session_key`, `agent_id`, `subagent_id`, `user_id`, `generation`, `effect_type`, `payload`, `idempotency_key`, `status`, `attempts`, `last_error`, `published_at`, `acked_at`, `created_at`, `updated_at`.
  - Indexes: pending status/created_at, session/generation/created_at, unique idempotency.
