# React contract positive session generation result

## Summary

Implemented the P349 React contract hardening so React think/actions inputs require an explicit positive `session_generation` before they can build downstream saga payloads. The runtime no longer defaults missing React contract generation to zero, including direct dataclass construction paths.

## Done

- Added `novaic-agent-runtime/task_queue/contracts/session_generation.py` with shared positive generation parsing/validation helpers.
- Updated `novaic-agent-runtime/task_queue/contracts/react_think.py` to remove the `session_generation=0` default, reject missing/non-positive generation in `from_context()`, and validate direct constructor use in `__post_init__()`.
- Updated `novaic-agent-runtime/task_queue/contracts/react_actions.py` with the same explicit positive generation contract.
- Updated old test fixtures to provide explicit positive generation instead of relying on the removed fallback.
- Added tests for missing/zero rejection in both React contract factories and direct constructors.
- Added source guard tests that reject reintroducing `session_generation: int = 0` and `ctx.get("session_generation") or 0` in the React contracts.
- Added a finalize payload assertion proving positive generation is preserved into the wake-finalize trigger payload.

## Verification

- `python3 -m py_compile task_queue/contracts/session_generation.py task_queue/contracts/react_think.py task_queue/contracts/react_actions.py` passed.
- Focused and historical wrapper suite passed: `70 passed in 0.44s`.
- Source-only residue guard passed with no matches:
  - `task_queue/contracts/react_think.py`
  - `task_queue/contracts/react_actions.py`
- A deliberately broader historical test run initially found old fixtures missing generation; those fixtures were updated rather than adding production compatibility fallback.

## Known Gaps

- P350 still needs to handle stale finalize side effects in Cortex scope-end and subagent status mutation handlers.
- P351 still needs recovery/compensation identity preservation checks.
- This ticket does not claim the parent runtime finalize enforcement problem is fully solved; it only closes the React contract entry boundary.
