# Rebuild and pending projection writer audit result

## Summary

Rebuild and pending projection writers are derived adapters, not independent authoritative state. Rebuild projects running/launched saga rows into session_state under a global transaction; pending projection records diagnostic observed events derived from unconsumed input events and is explicitly non-critical.

## Done

- Mapped `rebuild_session_state_from_running_sagas` in `session_rebuild.py`.
- Mapped `build_pending_input_projection`, `pending_restart_source_from_projection`, and `build_pending_restart_saga` in `session_projection.py`.
- Mapped repository projection calls for buffering, finalize/close, restart pending, and non-critical after-transaction projection recording.
- Mapped `SessionLedgerRepository.record_pending_projection` as an event append of derived projection, not a state table authority.

## Verification

- Rebuild reads `tq_saga_state` joined to `tq_sagas` for running/pending/launched saga contexts and writes active rows via `session_ledger.record_active_session(...)` after marking active states no_active in one global transaction.
- Pending projection derives from `session_ledger.list_unconsumed_input_events(session_key)` and writes `PENDING_PROJECTION_OBSERVED` events with `projected_pending` payload.
- `_record_pending_projection_after_transaction` and `_record_pending_projection_in_current_transaction` comments explicitly mark projection as derived observability, not authoritative inbox, and swallow projection failures so durable input/session transitions remain authoritative.
