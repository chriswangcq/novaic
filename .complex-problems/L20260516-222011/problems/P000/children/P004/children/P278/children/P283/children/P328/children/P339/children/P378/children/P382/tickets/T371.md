# Classify runtime finalize source guards

## Problem Definition

Run and classify runtime source guards for stale finalize/session-ended generation residue that can survive passing tests.

## Proposed Solution

1. Run `rg` guards over `novaic-agent-runtime/queue_service`, `novaic-agent-runtime/task_queue`, and focused tests for:
   - `int(generation)` and `int(session_generation)` coercion
   - optional or default generation in finalize/session-ended mutation paths
   - direct active-session clearing or update helpers
   - fallback generation defaults and current-active lookups
   - remaining-stack/archive diagnostics synthesized from implicit state
2. Inspect every hit and classify it as:
   - safe test fixture/assertion
   - safe adapter signature that fails closed before mutation
   - safe non-finalize generation allocation path
   - unsafe live residue fixed in this ticket
   - unsafe or broad residue split into a follow-up child problem
3. Record the guard matrix with file evidence.

## Acceptance Criteria

- All guard commands and results are recorded.
- Every hit is classified with file-level evidence.
- No live runtime finalize/session-ended path remains that silently coerces bool/missing generation, defaults generation, or mutates active state without current-generation ownership.
- Unsafe hits are fixed or split before success.

## Verification Plan

- Use `rg -n` with targeted patterns.
- Use `sed -n` or `nl -ba` for local context on hits.
- Re-run focused tests if any source changes are made.

## Risks

- Guard scans can return noisy matches in tests and type signatures; do not claim success without manually classifying them.

## Assumptions

- Runtime source is the scope of this ticket; Cortex and cross-repo aggregate guards have separate ledger problems.
