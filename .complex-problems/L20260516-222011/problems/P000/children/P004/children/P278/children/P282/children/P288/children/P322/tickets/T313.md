# Active session read path audit

## Problem Definition

Map active session read/list APIs and confirm they derive from session_state SSOT rather than legacy active-session pointers.

## Proposed Solution

Search and inspect `get_active_session`, `list_active_session_states`, active-session state conversion helpers, and HTTP route callers.

## Acceptance Criteria

- Read/list methods and callers are listed with file references.
- Source of truth for active session reads is classified.
- Any stale pointer/cache read path is flagged.

## Verification Plan

- Run `rg` for active session read/list names and legacy active table names.
- Inspect source slices in `session_repo.py` and routes.

## Risks

- Helper names may still say active_session even when source is correct; classify behavior, not just name.

## Assumptions

- `SessionLedgerRepository.get_state/list_active_states` over `tq_session_state` is the intended SSOT read path.
