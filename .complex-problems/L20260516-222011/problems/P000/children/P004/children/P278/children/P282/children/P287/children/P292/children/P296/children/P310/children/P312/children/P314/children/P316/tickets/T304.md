# Patch session outbox boundary source guards

## Problem Definition

The wrapper-boundary test still asserts an older single-effect wake source shape even though the implementation uses a generic outbox wrapper shape. This makes the test fail for the wrong reason and weakens its value as an architecture guard.

## Proposed Solution

Update `test_pr281_session_outbox_wrapper_boundary.py` to assert the current generic outbox wrapper shape while preserving strict negative guards against repository-owned append/publish helpers and attach eager publish wrappers.

## Acceptance Criteria

- Test no longer asserts stale `outbox_effect=wake_plan.outbox_effect` if the implementation now uses `outbox_effects=outbox_effects`.
- Test still asserts repository code does not own outbox append/publish wrappers.
- Test still asserts `session_outbox.py` does not expose old synchronous wrapper methods.
- Test remains meaningful against accidental eager attach publish reintroduction.

## Verification Plan

- Inspect `session_repo.py` source shape and update the test to match the current intentional boundary.
- Run the boundary test in P317 together with attach cutover tests.

## Risks

- Over-relaxing the source guard could let legacy wrappers creep back in.

## Assumptions

- Generic outbox effect-list wiring is now the intended architecture for session transition outbox recording.
