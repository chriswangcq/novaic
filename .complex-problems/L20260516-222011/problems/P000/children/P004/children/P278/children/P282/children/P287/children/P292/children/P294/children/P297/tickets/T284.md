# Ticket: Refactor dispatch start-wake construction

## Problem Definition

Remove or reduce duplicate start-wake durable transition construction in `SessionRepository.dispatch()` so ordinary no-active start and suspected-dead recovery start share a single helper/shape and cannot drift silently.

## Proposed Solution

Split into a small design/read step, an implementation step, and a focused verification step. The implementation should preserve existing behavior and tests while making duplicate active construction detectable or impossible.

## Acceptance Criteria

- Ordinary start and recovery start use one shared helper/recording path or a clearly factored minimal difference.
- Durable outbox authority, `require_outbox`, input consumption, recovery archive effect, and result shape are preserved.
- Focused tests/guards pass and include a regression against duplicated start-wake construction.
- Old duplicate code is removed, not left unused.

## Verification Plan

Run focused queue session tests and a source guard scan after patching.
