# Attach outbox worker-only implementation result

## Summary

Implemented the first worker-only attach cutover attempt by removing repository eager attach publish and moving attach delivery expectation to durable session outbox drain. The attempt is intentionally recorded as partial because focused tests exposed two concrete cleanup gaps before the implementation can be considered closed.

## Done

- Updated `novaic-agent-runtime/queue_service/session_repo.py` so active-session attach dispatch returns `delivery=outbox_pending` and an `outbox_id` instead of synchronously publishing the attach effect from the repository.
- Removed `publish_attach_input_effect` from `novaic-agent-runtime/queue_service/session_outbox.py`, reducing the repository-to-worker duplicate ownership surface.
- Updated `novaic-agent-runtime/tests/test_pr248_attach_outbox_cutover.py` to expect dispatch to persist attach outbox work first and explicit `drain_pending(...)` to publish it.
- Updated `novaic-agent-runtime/tests/test_pr281_session_outbox_wrapper_boundary.py` to guard against the removed attach sync wrapper.

## Verification

- Ran focused command from `novaic-agent-runtime`: `python3 -m py_compile novaic-agent-runtime/queue_service/session_repo.py novaic-agent-runtime/queue_service/session_outbox.py; PYTHONPATH=.:../novaic-common pytest -q tests/test_pr248_attach_outbox_cutover.py tests/test_pr281_session_outbox_wrapper_boundary.py`.
- Verification did not pass. The py_compile paths were wrong for the current working directory, and pytest exposed three failures.

## Known Gaps

- `SessionRepository.dispatch()` still logs `result["task_id"]` after attach dispatch, but the worker-only result no longer includes `task_id`; this causes `KeyError` in both attach cutover tests.
- The wrapper-boundary test still asserts the older wake outbox source shape `outbox_effect=wake_plan.outbox_effect`, while the implementation now uses the newer `outbox_effects=outbox_effects` shape.
- A fresh focused py_compile command must use `queue_service/session_repo.py` and `queue_service/session_outbox.py` paths from the `novaic-agent-runtime` working directory.
