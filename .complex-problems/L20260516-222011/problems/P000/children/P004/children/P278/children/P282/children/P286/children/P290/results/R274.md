## Active Usage Classification

`SessionLedgerRepository` is the table adapter:

- `novaic-agent-runtime/queue_service/session_ledger.py` configures `FsmSqliteStoreConfig` with:
  - `events_table="tq_session_events"`
  - `state_table="tq_session_state"`
  - `outbox_table="tq_session_outbox"`
  - `machine_id_column="session_key"`
  - `state_conflict_columns=("session_key",)`
  - `event_consumed_column="consumed_at"`

Table roles:

- `tq_session_events`: event log and append-only inbox/observation ledger.
  - Active writes/read helpers go through `SessionLedgerRepository.append_event`, `append_input_received`, `list_unconsumed_input_events`, `mark_events_consumed`, `record_input_consumed`, `record_pending_projection`, `record_session_finalize_rejected`, and `record_session_finalized`.
  - Classification: active required event log / inbox ledger.
- `tq_session_state`: materialized session authority.
  - Active state writes go through `SessionLedgerRepository.upsert_state`, `record_active_session`, `record_session_finalized`, and `mark_active_states_no_active`.
  - Active reads go through `SessionLedgerRepository.get_state`, `list_active_states`, and `SessionRepository.get_active_session/list_active_session_states`.
  - `SessionRepository._active_session_state_in_current_transaction()` reads state via `session_ledger.get_state()` and `_active_session_from_session_state()` derives active pointer from `state == "active"` plus saga/scope fields.
  - Classification: authoritative materialized session state.
- `tq_session_outbox`: durable side-effect outbox.
  - Active writes go through `SessionLedgerRepository.append_outbox` and `record_transition(... outbox_effect[s] ...)`.
  - Active reads/acks/fails go through `list_pending_outbox`, `get_outbox`, `mark_outbox_published`, and `mark_outbox_failed`.
  - Classification: durable outbox authority.
- `tq_active_sessions`: retired/absent table.
  - `rg` found no production runtime references under `queue_service` or `task_queue`.
  - Remaining references are tests and guard assertions, including `test_pr257_remove_active_sessions_table.py`, `test_pr255_legacy_compat_cleanup.py`, `test_pr273_session_harness_final_residue_guard.py`, and `test_pr315_queue_fsm_final_residue_guard.py`.
  - Classification: safe test/documentation guard only, not active runtime code.

## Rebuild and Projection Notes

- `novaic-agent-runtime/queue_service/session_rebuild.py` reconstructs `tq_session_state` from active saga state on startup by calling `session_ledger.mark_active_states_no_active()` and `session_ledger.record_active_session(...)`.
- `novaic-agent-runtime/queue_service/session_projection.py` is pure helper logic for pending inbox projection and does not directly touch SQL tables.
- `SessionRepository` active-session APIs derive from session_state SSOT, not a separate active-session pointer table.

## Risks / Cleanup Candidates

No active `tq_active_sessions` runtime residue was found in the scanned production directories. The only notable residual risk is conceptual complexity: `SessionRepository` remains a dense coordinator around ledger operations, but table ownership itself is clean.
