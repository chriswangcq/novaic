# Old session wrapper and compatibility branch scan

## Problem Definition

Search for legacy session wrappers, compatibility branches, direct publish helpers, and mutation shortcuts that could keep old session harness logic alive outside the intended FSM/session outbox boundary.

## Proposed Solution

Run targeted searches for old helper names, active-session compatibility terms, generation fallback wording, direct publish/drain behavior, and session mutation shortcuts. Inspect ambiguous production hits and classify them.

## Acceptance Criteria

- Search patterns cover attach/wake publish wrappers, active session compatibility, generation fallback, and direct mutation shortcuts.
- Matches are classified as production active path, test guard, documentation, or no-match.
- Risky/removable production residue is flagged for cleanup follow-up.

## Verification Plan

- Run `rg` patterns over `novaic-agent-runtime` production and tests.
- Inspect `session_repo.py`, `session_outbox.py`, and relevant tests around hits.
- Record no-match evidence for dangerous helper names.

## Risks

- Some residue may survive as renamed behavior rather than old names; include behavior-oriented searches such as publish/drain and active session terms.

## Assumptions

- Negative test guards are acceptable if they intentionally prevent legacy reintroduction.
