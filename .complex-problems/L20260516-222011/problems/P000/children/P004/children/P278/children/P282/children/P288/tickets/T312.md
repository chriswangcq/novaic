# Audit session rebuild projection and state reads

## Problem Definition

Inspect rebuild, projection, and state read helpers to verify active session state is reconstructed/read from authoritative session state rather than stale pointers or caches.

## Proposed Solution

Split into focused child audits: active state read APIs, rebuild/projection writers, and coverage/guard classification.

## Acceptance Criteria

- Read/rebuild/projection methods are mapped with file references.
- Active session reads are proven to derive from `tq_session_state`/SessionLedgerRepository or any deviation is flagged.
- Tests/guards for rebuild/projection/state ownership are identified.
- Any drift-prone cache/view becomes a cleanup follow-up.

## Verification Plan

- Inspect `session_repo.py`, `session_rebuild.py`, `session_projection.py`, and tests.
- Run targeted source searches for active session read pointers and rebuild calls.
- Record classification before parent success.

## Risks

- A read path may look harmless but still derive from a stale projection; inspect call chain, not just table name.

## Assumptions

- `tq_session_state` is intended SSOT for active session read state.
