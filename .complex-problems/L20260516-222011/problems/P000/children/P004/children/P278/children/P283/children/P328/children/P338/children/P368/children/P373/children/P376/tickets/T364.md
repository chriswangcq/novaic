# Ticket: Persist Explicit Archive Diagnostics

## Problem Definition

Cortex receives explicit archive diagnostics from runtime but does not yet persist them in `WakeArchived` context events.

## Proposed Solution

Add an optional nested `archive_diagnostics` object to the `WakeArchived` payload when `ScopeEndRequest` includes diagnostics. Preserve the existing top-level semantic `remaining_stack` list. Extend `ContextEventWriter.wake_archived(...)`, `_append_wake_archived_event(...)`, and focused Cortex tests.

## Acceptance Criteria

- Diagnostic `ScopeEndRequest` archives a wake with exact nested `archive_diagnostics` containing `session_generation`, `finalize_reason`, `remaining_stack`, and `round_num`.
- Existing structural `WakeArchived` tests without diagnostics remain unchanged.
- Invalid diagnostic request validation still prevents metadata persistence.
- Focused Cortex tests pass.

## Verification Plan

- Run `python3 -m py_compile novaic_cortex/api.py novaic_cortex/context_event_writer.py`.
- Run focused Cortex tests for scope-end contract and context event lifecycle.

## Risks

- Accidentally changing top-level `remaining_stack` would corrupt context projection. Keep diagnostic data nested.
