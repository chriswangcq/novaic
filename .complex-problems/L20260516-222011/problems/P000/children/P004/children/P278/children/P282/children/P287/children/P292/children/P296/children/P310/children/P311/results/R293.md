# Result: Attach outbox worker-only design

Design:
- `SessionRepository._publish_attach_request_after_transaction(...)` should only record `PUBLISH_ATTACH_INPUT` outbox and return a result shaped like:
  - `action=attached_to_active`
  - `saga_id`, `scope_id`
  - `delivery=outbox_pending`
  - `outbox_id`
  - no synchronous `task_id`
- `SessionOutboxDispatcher.publish_attach_input_effect(...)` should be deleted if no production caller remains. Current search shows only `session_repo.py` calls it in production; tests assert its existence and must be updated/removed.
- Delivery should happen through `SessionOutboxDispatcher.drain_pending(...)`, normally via `session_outbox_worker`.
- Existing failure-retry semantics remain: publish failure during worker drain leaves outbox pending/retryable.

Verification plan:
- Source guard: no `publish_attach_input_effect` symbol in production runtime after cutover.
- Tests: update attach outbox cutover tests so dispatch returns pending, then explicit dispatcher drain publishes task.
- Remove/update wrapper-boundary test that currently requires the eager attach wrapper.
