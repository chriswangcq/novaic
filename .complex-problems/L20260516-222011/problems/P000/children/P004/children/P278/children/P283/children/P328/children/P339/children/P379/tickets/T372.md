# Cortex archive diagnostics aggregate regression

## Problem Definition

Verify Cortex scope archive diagnostics after the runtime generation boundary work. Cortex must persist explicit archive diagnostics from `scope_end` request payloads, reject unsafe generation coercion, and preserve context projection semantics without synthesizing diagnostics from hidden active state.

## Proposed Solution

1. Run focused Cortex lifecycle and projection tests for scope archive, context event API lifecycle, skill lifecycle, projection, model, and write authority.
2. Run source guards over Cortex lifecycle files for:
   - `int(session_generation)` or bool-coercing generation parsing
   - archive diagnostics built from current active/session state
   - remaining stack diagnostic fields synthesized outside explicit request payloads
   - projection accidentally consuming nested `archive_diagnostics.remaining_stack` as semantic stack content
3. Fix any concrete Cortex residue found; split a follow-up if broad.
4. Record exact commands, evidence, and residual risks.

## Acceptance Criteria

- Focused Cortex tests pass.
- Source guards are run and every hit is classified.
- `ScopeEndRequest` rejects bool diagnostics generation and enforces all-or-nothing diagnostics fields.
- `WakeArchived` persists explicit `archive_diagnostics` while preserving top-level semantic `remaining_stack` projection behavior.
- No Cortex archive diagnostics live path reads current active generation or active stack to synthesize finalize diagnostics.

## Verification Plan

- `python3 -m py_compile novaic_cortex/api.py novaic_cortex/context_event_writer.py novaic_cortex/context_event_projection.py`.
- Focused Cortex pytest suite with relevant lifecycle/projection/write authority tests.
- `rg` source guards over `novaic_cortex/api.py`, `context_event_writer.py`, `context_event_projection.py`, and tests.

## Risks

- Cortex projection intentionally has two stack concepts: top-level semantic `remaining_stack` and nested diagnostic `archive_diagnostics.remaining_stack`. The check must not conflate them.

## Assumptions

- Runtime is responsible for explicit finalize diagnostics inputs; Cortex persists and projects, but does not infer missing diagnostics from active runtime state.
