# Cortex scope_end identity contract result

## Summary

Implemented a fail-closed Cortex `scope_end` identity contract. Runtime structural scope archival still uses immutable scope identity/path for the actual archive, but the handler now requires an explicit positive `session_generation` before it performs any user-scoped lifecycle mutation such as archive, notification processed-transition, bridge removal, or participant completion. `wake_finalize` now propagates generation into the Cortex scope-end payload.

## Done

- Updated `novaic-agent-runtime/task_queue/handlers/cortex_handlers.py`:
  - `handle_cortex_scope_end()` now requires positive `session_generation` for user-scoped archival.
  - Missing/non-positive generation fails before meta read, archive, notification transition, bridge removal, or participant status updates.
  - Result/logs include `session_generation` for auditability.
- Updated `novaic-agent-runtime/task_queue/sagas/wake_finalize.py`:
  - `_build_cortex_scope_end_payload()` now includes `session_generation` using the same positive generation contract used by `session.ended`.
- Updated tests/fixtures to pass explicit positive generation in existing valid scope-end cases.
- Added `test_scope_end_requires_positive_session_generation_before_archive()` to verify missing/zero generation rejects before archive calls.
- Extended finalize ownership tests to prove wake-finalize scope-end payloads carry generation and reject missing/zero generation.
