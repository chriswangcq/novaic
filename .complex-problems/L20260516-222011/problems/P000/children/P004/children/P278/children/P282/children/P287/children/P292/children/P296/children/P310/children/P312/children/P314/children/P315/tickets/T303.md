# Patch attach dispatch logger contract

## Problem Definition

Attach dispatch now returns a durable outbox-pending result rather than a synchronous task result, but the dispatch logger still indexes `task_id` and crashes.

## Proposed Solution

Update the attach dispatch log line to use `outbox_id` and `delivery` from the new result shape, and refresh nearby comments so they no longer imply eager publish.

## Acceptance Criteria

- No `result["task_id"]` read remains in attach dispatch.
- Logging still includes useful saga/scope and delivery diagnostics.
- The attach result shape remains worker-owned and outbox-pending.

## Verification Plan

- Inspect the patched source for stale `task_id` references in the attach dispatch path.
- Run focused tests after P316 test guard patch is complete.

## Risks

- A too-broad source edit could affect start-wake task semantics unrelated to attach dispatch.

## Assumptions

- Start-wake paths may still use saga/task terminology where appropriate; this ticket only concerns active attach dispatch.
