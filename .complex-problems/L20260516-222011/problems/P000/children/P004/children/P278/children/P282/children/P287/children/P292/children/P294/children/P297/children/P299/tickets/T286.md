# Ticket: Implement dispatch start-wake helper extraction

## Problem Definition

Refactor `SessionRepository.dispatch()` so ordinary no-active start and recovery start share one start-wake durable transition construction helper, removing duplicated inline construction while preserving behavior.

## Proposed Solution

Add a private helper on `SessionRepository` that validates `START_WAKE`, builds `DispatchWakePlan`, appends optional recovery archive effect, records the transition in either current or new transaction, and returns the result. Replace both inline start-wake construction blocks with helper calls.

## Acceptance Criteria

- `session_repo.py` compiles.
- `build_dispatch_wake_plan(` is centralized in the helper rather than duplicated in `dispatch()`.
- Both ordinary and recovery start paths still return `wake_start_queued` with scope and outbox id.
- No unrelated code is modified.
