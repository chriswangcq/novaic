# Result: Cortex shell artifact seam inventory

## Summary

Cortex has direct coverage for the shell/device artifact-manifest contract and projection behavior. It proves `devicectl hd screenshot` uploads screenshot bytes to Blob and prints a `tool-output.v1` artifact manifest without inline base64, and it proves shell artifact manifests remain text in display/history projections. It does not by itself cover the runtime follow-up display call sequence.

## Verification

Evidence pointers:

- `test_devicectl_hd_screenshot_uploads_image_and_prints_tool_output_artifact` verifies screenshot bytes are uploaded to `runtime-artifact`, stdout is `tool-output.v1`, artifact has `access.display.args.file_url`, and the original base64 screenshot is absent from stdout.
- `test_devicectl_hd_file_pull_uploads_file_and_prints_tool_output_artifact` verifies the same artifact contract for pulled files.
- `test_cortex_payload_help_teaches_bounded_payload_inspection` and `test_devicectl_help_teaches_artifact_manifest_not_base64` verify CLI guidance against raw payload/base64 dumping.
- `step_result_projection.py::_parse_tool_output_v1` renders artifact manifests as text lines, including `display(file_url="blob://...")` access hints.
- `format_for_history_llm` and `format_for_current_tool_result_llm` call `_format_mcp_content(... include_display=False)`.
- `format_for_display_perception_llm` only emits image content for parsed `display_files`; `tool-output.v1` artifacts parse to text/artifacts and `display_files=[]`.
- `test_tool_output_v1_artifacts_render_as_manifest_text` and `test_tool_output_v1_display_perception_never_inlines_artifact_image` cover the manifest-only artifact behavior.
- `test_media_like_shell_step_payload_does_not_project_as_display_image` proves media-looking shell stdout is not treated as display media.
- `test_display_tool_step_payload_projects_llm_image_content` proves actual display tool payloads are visual only in explicit display perception mode, while history remains text.

## Known Gaps

- Cortex-side coverage is strong for shell artifact output and projection modes, but it stops before runtime assembles a subsequent assistant display call and the following historical replay.
- The missing near-integrated chain should be closed in the implementation child problem.
