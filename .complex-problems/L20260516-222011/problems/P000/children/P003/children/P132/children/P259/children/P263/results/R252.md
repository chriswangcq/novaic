# Cortex projection coverage inventory result

## Summary

Completed the Cortex projection coverage inventory. Existing focused tests already cover the core Cortex-side projection contract: `tool-output.v1` artifact manifests stay text/manifest-only, history/current non-display projections exclude image data, explicit display perception is the only projection mode that may include image media, payload inspection APIs are bounded, and step write/read formatting preserves payload refs instead of inline raw results.

## Done

- Inspected `novaic-cortex/novaic_cortex/step_result_projection.py` projection modes.
- Inspected `novaic-cortex/tests/test_tool_output_projection.py` coverage for artifact manifests, display perception, history/current non-display projections, media-like shell output, and `tool-step-payload.v1` raw payload isolation.
- Inspected `novaic-cortex/tests/test_step_result_projection.py` coverage for parser fallback, history truncation, and display perception data URL handling.
- Inspected `novaic-cortex/tests/test_payload_inspection_api.py` coverage for bounded read/search/summarize/qa behavior.
- Inspected `novaic-cortex/tests/test_context_event_api_steps_write.py` coverage for rejecting inline result writes, preserving payload refs, externalized payload resolution by stable step ref, and projection validation.

## Verification

Coverage map from inventory:

- Artifact manifest text-only: `test_tool_output_v1_artifacts_render_as_manifest_text`, `test_tool_output_v1_display_perception_never_inlines_artifact_image`.
- History/current non-display exclude images: `test_mcp_image_content_is_only_visual_in_display_perception_projection`, `test_explicit_projection_modes_control_parsed_display_files`, `test_display_tool_step_payload_projects_llm_image_content`.
- Shell media-like stdout does not become display media: `test_media_like_shell_step_payload_does_not_project_as_display_image`.
- Raw shell payload not projected from `tool-step-payload.v1`: `test_tool_step_payload_v1_projects_llm_content_not_raw_shell_payload`.
- Payload APIs bounded/redacted: `test_payload_read_returns_bounded_tail_slice`, `test_payload_search_returns_bounded_match_contexts`, `test_payload_summarize_uses_redacted_bounded_factory_input`, `test_payload_qa_requires_question_and_bounds_output`.
