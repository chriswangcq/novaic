# Cortex projection coverage inventory

## Problem

Inventory the current Cortex projection and payload tests against the desired contract before making changes.

## Success Criteria

- Existing test files covering `step_result_projection`, `tool_output_projection`, payload inspection, and context event step writes are identified.
- Coverage gaps are explicitly listed or the inventory proves no gap.
- The inventory distinguishes history/current non-display projection from explicit display perception projection.
