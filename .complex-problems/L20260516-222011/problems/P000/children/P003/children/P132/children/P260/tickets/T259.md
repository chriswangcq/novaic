# Agent-runtime current versus historical media regression coverage ticket

## Problem Definition

Agent-runtime owns context assembly and therefore the distinction between current explicit display perception and historical sanitized tool messages. It must prove that current display media can reach the provider as native media while later history replay uses placeholders/manifest text and shell output remains bounded terminal text.

## Proposed Solution

Audit runtime context/media tests, add missing focused coverage only if needed, and run a focused runtime regression suite. Split into inventory, implementation-gap closure, and verification so the result is not a superficial one-go pass.

## Acceptance Criteria

- Runtime context tests prove current display perception becomes provider-native image content when selected.
- Runtime history replay tests prove image data is replaced by placeholders or manifest text.
- Shell output tests prove terminal text stays bounded and is not image-sniffed into display media.
- Focused runtime tests pass.

## Verification Plan

Inspect and run tests under `novaic-agent-runtime/tests/unit/task_queue/test_no_historical_tool_image_injection.py`, `test_tool_handlers_display_chat_history.py`, `test_shell_output_contract.py`, `test_runtime_explicit_contracts.py`, and `test_factory_client_multimodal.py`. Add missing tests if inventory finds a gap.

## Risks

- Current-turn display media is intentionally allowed; tests must not confuse it with historical leakage.
- Runtime and factory provider conversion overlap; keep this problem focused on runtime context assembly.

## Assumptions

- Provider-specific conversion is covered elsewhere; this branch only needs to prove runtime preserves/sanitizes the right content shape.
