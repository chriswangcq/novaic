# Cortex projection regression coverage ticket

## Problem Definition

Cortex projection is the first contract boundary for stored tool results. It must prove by tests that artifact manifests, history projection, current non-display projection, explicit display perception projection, and payload pointer behavior do not accidentally reintroduce raw media into normal context.

## Proposed Solution

Audit the focused Cortex projection tests, add or adjust the smallest missing regression coverage, then run the focused Cortex projection/payload test set. Separate the work into inventory, implementation, and verification if the existing coverage is not obviously complete.

## Acceptance Criteria

- Cortex projection tests prove `tool-output.v1` artifact manifests are text/manifest-only.
- Cortex history/current non-display projections exclude image/base64 data.
- Cortex explicit display perception projection is the only mode that may include image media.
- Cortex payload/step APIs remain pointer/bounded instead of dumping raw payloads.
- Focused Cortex tests pass.

## Verification Plan

Inspect `novaic-cortex/novaic_cortex/step_result_projection.py`, related API/projection modules, and tests under `novaic-cortex/tests/test_*projection*.py`, `test_payload_inspection_api.py`, and `test_context_event_api_steps_write.py`. Add missing regression tests if coverage gaps are found, then run the focused pytest selection.

## Risks

- Some base64 test fixtures are intentional. Avoid deleting useful provider/display regression fixtures.
- Over-broad assertions can make explicit display perception look forbidden; preserve the contract exception.

## Assumptions

- Cortex should not decide provider-native adapter details except producing the explicit display-perception `_mcp_content` contract.
- Normal shell/tool history should remain text/manifest only.
