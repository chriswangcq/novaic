# Result: Runtime media seam inventory

## Summary

Runtime media handling has strong piecewise coverage, but no single runtime test fully stitches the screenshot artifact manifest flow into display current perception and historical replay. The current runtime side is therefore ready for a near-integrated regression in `P270`, not fully covered already.

  - `novaic-agent-runtime/tests/unit/task_queue/test_no_historical_tool_image_injection.py`

## Verification

Evidence pointers:

- `utils/step_result_client.py` selects `display_perception` only for current display tool messages and `history` for old display messages.
- `utils/context.py` preserves `_projection` through sanitization, converts only `display_perception` tool output into provider-native image user messages, and strips `_projection` before final request.
- `utils/multimodal.py` extracts images only from display `_mcp_content`, replaces display image payloads with placeholders in tool text, and bounds display text receipts.
- `test_no_historical_tool_image_injection.py` covers:
  - generic tool image output does not create image messages;
  - history projection does not inject images;
  - current display perception does inject provider-native image content;
  - display public wrapper stores raw media only in durable payload;
  - `prepare_llm_call` injects display image before following system message.
- `test_shell_output_contract.py` covers shell `tool-output.v1`, bounded large output, and media-like stdout staying terminal text.
- `test_factory_client_multimodal.py` confirms runtime factory client preserves structured `image_url` content instead of stringifying it.

## Known Gaps

- Runtime coverage is still piecewise for the actual user-visible screenshot path. The existing tests do not use one sequence where shell returns a screenshot artifact manifest, the next assistant calls display with that artifact, display is injected as current media, and later replay is sanitized.
- This gap should be closed in the near-integrated regression implementation ticket rather than by changing production code during inventory.
