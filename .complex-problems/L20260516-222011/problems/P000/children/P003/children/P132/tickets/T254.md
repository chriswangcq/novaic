# Cross-layer projection regression suite ticket

## Problem Definition

Projection correctness now spans multiple layers: shell/device CLI emits `tool-output.v1` artifact manifests, Cortex stores durable step payloads and projects step results, agent-runtime expands current display media into provider-native content, and later history replay must stay text/manifest-only. Existing local tests can pass while the integrated flow still drops current media or leaks raw base64 into history.

## Proposed Solution

Build and run a focused cross-layer regression suite that validates the end-to-end projection contract after the recent fixes. The suite should include:

1. Cortex projection tests proving `tool-output.v1` screenshot/artifact manifests remain text/manifest-only unless explicit display perception is invoked.
2. Runtime context tests proving explicit current display media is passed to the current LLM call as provider-native media while the corresponding tool message stored in history is sanitized.
3. At least one integrated or near-integrated regression that models the real flow: shell screenshot manifest -> display/current media projection -> later historical replay.
4. A final search/assertion proving raw media payload markers such as `/9j/`, `data:image`, and synthetic `SECRETBASE64` do not appear in historical text paths except in intentional tests/provider-native fixtures.

## Acceptance Criteria

- Focused Cortex projection tests pass.
- Focused agent-runtime context/projection tests pass.
- An integrated or near-integrated test covers shell screenshot artifact manifest followed by display/current media projection and later historical replay.
- The verification explicitly checks that raw base64 is absent from text history for display/tool results.
- Remaining risks are documented with evidence pointers.

## Verification Plan

Run targeted pytest selections across `novaic-cortex` and `novaic-agent-runtime`, then run a final repository scan for media/base64 markers in LLM-visible historical text paths. If no integrated regression exists, add the smallest focused test in the layer that can exercise the cross-layer contract without starting full services.

## Risks

- The truly end-to-end path may require service orchestration; if so, use a near-integrated unit/regression that exercises the exact projection functions and records the residual orchestration gap.
- Existing dirty worktree/submodule changes may include unrelated edits; avoid reverting them and keep any new test/code edits tightly scoped.
- Synthetic base64 fixtures are expected in tests and provider adapter paths; classify them rather than deleting useful regression fixtures.

## Assumptions

- Current architecture intentionally allows explicit current-turn display perception to carry provider-native image data.
- Historical replay and normal shell/tool results must stay bounded text/manifest-only.
- The regression suite can be focused rather than a full production deployment smoke test.
