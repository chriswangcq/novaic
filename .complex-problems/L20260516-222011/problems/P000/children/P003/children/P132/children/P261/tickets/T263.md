# Ticket: Near-integrated screenshot manifest to display replay regression

## Problem Definition

The isolated runtime and Cortex tests cover parts of the media contract, but the project still needs a near-integrated regression for the actual screenshot path:

1. A shell/device CLI returns terminal text containing a `tool-output.v1` image artifact manifest, not inline base64.
2. The agent selects the artifact URI and calls `display` for current perception.
3. The current `display` result becomes provider-native image content for the immediate LLM call.
4. Historical replay of the same flow is sanitized into text/manifest placeholders, with no raw base64 and no accidental provider-native image re-injection.

Without this test, future changes can pass unit slices while breaking the stitched user-visible behavior.

## Proposed Solution

First inventory existing runtime/Cortex tests to locate whether the full sequence is already covered. If not, add the smallest near-integrated regression at the boundary where prepared LLM messages are assembled.

The test should use explicit fixture data for:

- shell screenshot stdout containing a `tool-output.v1` artifact manifest;
- a display tool result for that artifact;
- a following assistant/system turn to catch ordering bugs;
- historical replay of the same tool result.

If the regression reveals an implementation gap, fix it with a boundary-respecting change and remove any obsolete branch that conflicts with the intended contract.

## Acceptance Criteria

- A regression covers the full logical sequence from screenshot artifact manifest to display current perception to sanitized history replay.
- Current display perception is provider-native media content only where intended.
- Shell output remains terminal text and bounded manifest text.
- Historical replay contains no raw base64 image payload and no accidental `data:image` provider content.
- Focused runtime/Cortex tests pass.
- Any discovered implementation gap is fixed in the smallest boundary-respecting way, with old/incorrect branches removed instead of left dormant.

## Verification Plan

- Run focused runtime/Cortex tests for media projection, tool output contracts, and shell output contracts.
- Add marker scans proving no raw screenshot base64 or unintended `data:image` appears in history replay.
- Confirm current display perception still uses provider-native media in the active call.
- Record exact commands and test counts in the result.
