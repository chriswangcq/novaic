# Result: Cross-layer coverage comparison

## Summary

Current coverage is not sufficient for the full near-integrated screenshot/display/replay contract. Runtime and Cortex each have strong piecewise coverage, but there is no single regression that stitches a shell screenshot artifact manifest into a display call, current provider-native image injection, and later sanitized history replay.

## Done

Compared the inventory results:

- `R260` runtime inventory: display current perception, history sanitization, shell bounded text, and factory multimodal preservation are covered piecewise.
- `R261` Cortex inventory: shell screenshot artifact manifest, blob upload, no stdout base64, and projection modes are covered piecewise.

## Known Gaps

`P270` should add a near-integrated regression, likely in runtime test coverage, that creates a message sequence containing:

1. assistant shell call;
2. shell tool result by `step_ref` whose Cortex formatted content is a `tool-output.v1` screenshot artifact manifest text;
3. assistant display call with that artifact URI;
4. display tool result by `step_ref` whose current projection returns `_mcp_content` image data;
5. a later round where the same display result is historical and must stay text/placeholder only.

This can likely be done with a fake Cortex bridge in `novaic-agent-runtime/tests/unit/task_queue/test_no_historical_tool_image_injection.py` without production changes unless the test exposes a defect.
