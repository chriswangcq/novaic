# Ticket: Production fix or no-op proof for near-integrated regression

## Problem Definition

After adding the near-integrated regression, determine whether production code needs any change. Passing the new regression may mean the current implementation is correct and only coverage was missing.

## Proposed Solution

Run the new regression and inspect the changed diff. If it passes without production changes and the assertions cover the intended behavior, record a no-op proof. If it fails or reveals a gap, implement the smallest boundary-respecting production fix and remove stale branches.

## Acceptance Criteria

- New regression is executed.
- If production code is unchanged, the result explicitly explains why no production fix is needed.
- If production code changes, focused tests pass and stale/conflicting code is removed.
- The conclusion does not hide residual verification work reserved for `P271`.

## Verification Plan

- Run the new test node or affected runtime test file.
- Inspect `git diff --stat`/targeted diff to confirm production versus test-only changes.
- Record the no-op or fix decision with evidence.
