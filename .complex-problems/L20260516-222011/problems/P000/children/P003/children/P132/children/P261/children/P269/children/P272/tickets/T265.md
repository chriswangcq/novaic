# Ticket: Runtime media seam inventory

## Problem Definition

Runtime needs a precise inventory of the code and tests responsible for display media handling so the near-integrated regression can target the right seam instead of duplicating irrelevant coverage.

## Proposed Solution

Read runtime source and tests for:

- display tool current perception expansion;
- historical display sanitization;
- shell text/output contract handling;
- factory/provider-native media preservation;
- active stack/system message ordering after display.

## Acceptance Criteria

- Runtime implementation files/functions are listed with file references.
- Runtime tests are listed with what behavior they prove.
- The result explicitly says whether runtime coverage is true near-integrated or piecewise.
- No production files are modified.

## Verification Plan

Use `rg`, `sed`, and targeted reads. Optionally run focused runtime tests only if needed to confirm test names or behavior.
