# Ticket: Add near-integrated screenshot/display regression test

## Problem Definition

Add a test that stitches the real logical flow: shell screenshot artifact manifest, display artifact access, current display image injection, and later display history replay sanitization.

## Proposed Solution

Extend `novaic-agent-runtime/tests/unit/task_queue/test_no_historical_tool_image_injection.py` with a fake bridge that returns formatted step content based on `(tool_call_id, projection)`. The test should call `prepare_llm_call` twice: once for current display perception and once for a later round where the same display step is historical.

## Acceptance Criteria

- The test includes a shell `tool-output.v1` artifact manifest with `display(file_url="blob://...")` access hint.
- The current call output contains exactly one provider-native image message for the display step.
- The shell step stays text-only and does not create provider-native image content.
- The later historical replay contains no provider-native image content and no raw display base64 in tool text.
- Test is focused and readable.

## Verification Plan

Run the affected runtime test file, or at minimum the new test node, and record the result in the ticket output.
