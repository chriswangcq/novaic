# Ticket: Implement near-integrated screenshot/display regression

## Problem Definition

The audit found no single regression for the real screenshot flow: shell emits a screenshot artifact manifest, the agent calls display with the artifact URI, current display perception becomes provider-native media, and later history replay stays text/placeholder-only.

## Proposed Solution

Add the smallest boundary-respecting regression, likely in `novaic-agent-runtime/tests/unit/task_queue/test_no_historical_tool_image_injection.py`, using a fake Cortex bridge that returns different formatted content by `tool_call_id` and projection.

The regression should prove:

- shell step formatted content is `tool-output.v1` manifest text with artifact access hints, not image media;
- display step current projection returns `_mcp_content` image data and becomes provider-native image content;
- a later round treats the same display step as history and does not inject media/base64;
- ordering remains valid when a following system message exists.

If the test exposes an implementation defect, fix it at the context/projection boundary and delete stale conflicting code if found.

## Acceptance Criteria

- A near-integrated regression is added or a no-op proof is recorded with strong evidence.
- The test covers shell manifest, display current perception, and historical replay in one logical sequence.
- No raw image base64 appears in tool text/history assertions.
- Provider-native image content appears only for current display perception.
- Focused tests pass or the issue is split further if not directly resolvable.
