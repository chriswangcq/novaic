# Audit runtime history/media perception boundary

## Problem Definition

Runtime must keep historical tool results text-only while allowing current display perception to become provider media content at the final model-call boundary.

## Proposed Solution

Inspect runtime context assembly, step result projection, multimodal conversion, and focused tests for current-vs-history display handling.

## Acceptance Criteria

- Runtime mapping identifies where display history is sanitized and where current perception becomes media content.
- Tests prove historical tool image data is not re-injected.
- Tests prove current display perception can become OpenAI/Anthropic media content only through the intended conversion path.

## Verification Plan

- Use `rg`/`nl` to map runtime files/functions.
- Run focused runtime multimodal/history tests.

## Risks

- The same `_mcp_content` shape can appear in multiple projections; the audit must distinguish history from current display perception.
