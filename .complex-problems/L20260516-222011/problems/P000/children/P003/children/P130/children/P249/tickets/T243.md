# Audit provider-native media adapter formats

## Problem Definition

LLM factory providers must encode image content in provider-native media fields. Data URL/base64 content is allowed here only as a media field, not as plain text message history.

## Proposed Solution

Inspect provider conversion code and tests for OpenAI-compatible, Anthropic, and Google/Gemini paths. Run focused factory tests that validate media payload shape.

## Acceptance Criteria

- Provider conversion functions for media are mapped with file/function pointers.
- Tests prove image content is represented as provider-native content parts.
- Gemini/Google data URL conversion uses `inlineData` rather than plain text.
- No provider request fixture expects base64 as a plain text-only message.

## Verification Plan

- Use `rg`/`nl` on `novaic-llm-factory/factory/providers.py` and tests.
- Run focused LLM factory tests covering provider media conversion.

## Risks

- Provider test suite may rely on mocked request capture; ensure assertions are on outgoing provider payload shape.

## Assumptions

- No live provider calls are required.
