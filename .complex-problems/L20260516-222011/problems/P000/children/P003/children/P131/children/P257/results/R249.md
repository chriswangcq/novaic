# Result: Factory log/detail large-output boundary hardened

## Summary
Factory log redaction/detail paths were scanned. Media redaction was already present for logged chat snapshots, and list queries omit large bodies by default. A remaining large-output gap was fixed: `/v1/logs/{log_id}` now bounds `request_body` and `response_body` by default and returns truncation metadata.

## Classification
| Cluster | Classification / Action | Evidence |
|---|---|---|
| chat request log snapshot | media/base64 redacted before persistence | `contracts.py:80-140`, `chat_routes.py:147-151` |
| response body logging | explicit optional response snapshot | `contracts.py:150-151`, `chat_routes.py:204-219` |
| logs list route | omits request/response bodies and returns body byte counts | `log_routes.py:36-61`, `db.py:552-603` |
| log detail route | fixed: default bounded body strings plus `*_chars` and `*_truncated` metadata | `log_routes.py` |
| log tests | redaction and bounded body regression tests | `tests/test_log_routes.py`, `tests/test_chat_routes.py` |

## Code Changes
- `novaic-llm-factory/factory/routes/log_routes.py`: added bounded body helpers and default `body_limit` for log detail route.
- `novaic-llm-factory/tests/test_log_routes.py`: added regression tests for default and explicit body truncation.
