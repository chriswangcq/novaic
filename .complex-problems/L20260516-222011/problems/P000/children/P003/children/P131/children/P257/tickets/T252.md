# Ticket: Factory log redaction and detail large-output sweep

## Problem Definition
LLM factory logs and log-detail APIs can expose large request/response bodies or raw multimodal media data if redaction and bounding are only applied on one path. Both persisted snapshots and detail responses must avoid raw base64/media leakage and unbounded payload display.

## Proposed Solution
Inspect factory contracts, chat route log snapshot creation, log route detail/list responses, and tests. Patch missing redaction or bounding if found, then run focused factory tests.

## Acceptance Criteria
- Request/response log storage and detail routes are classified with citations.
- Media/base64 request bodies are redacted in persisted/detail output.
- Large log/detail output has an explicit bound or documented storage/display boundary.
- Focused factory tests pass.

## Verification Plan
Run targeted `rg` scans for request_body/response_body/redaction/limit/detail, inspect routes/contracts, and run chat/log route tests.
