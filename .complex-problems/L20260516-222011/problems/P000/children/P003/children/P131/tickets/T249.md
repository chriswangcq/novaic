# Ticket: Large-output and base64 leakage sweep

## Problem Definition
Large tool outputs, logs, payloads, or media-like strings can still leak into LLM context, monitor views, or persisted logs even if the media/provider path is fixed. The codebase needs a broader sweep for unbounded serialization, raw payload dumps, and base64-like output in runtime/Cortex/common/factory surfaces.

## Proposed Solution
Audit large-output boundaries across runtime shell projection, Cortex step/payload projection, LLM factory log storage/detail APIs, and common tool contracts. Patch any path that lacks truncation, pointerization, redaction, or explicit artifact/blob handling.

## Acceptance Criteria
- Large-output risk clusters are identified and classified.
- Active LLM-context and log-detail paths have bounded text, redaction, or pointer contracts.
- Any stale raw-dump path is deleted or fixed.
- Focused tests cover the changed/verified boundaries.

## Verification Plan
Use targeted `rg` scans for unbounded serialization/dumps/truncation/redaction markers, inspect implementation slices, and run focused tests for shell output, step projection, payload inspection, and factory log redaction.
