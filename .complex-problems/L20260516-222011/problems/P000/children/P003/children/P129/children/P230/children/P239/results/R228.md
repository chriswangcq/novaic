# Result: LLM tool schema and policy payload boundary audit

## Summary

Audited the direct LLM tool surface, shell-capability migration policy, and payload-inspection schema boundaries. The active runtime tool policy is intentionally reduced to the final harness surface (`shell`, `display`, `skill_begin`, `skill_end`, `sleep`), with former interface tools classified as shell capabilities rather than direct LLM schemas.

## Done

- Confirmed `novaic-agent-runtime/task_queue/tool_surface_policy.py` explicitly separates `FINAL_HARNESS_TOOLS` from `MIGRATED_INTERFACE_TOOLS` and exposes only two target classes: `final_harness` and `shell_capability`.
- Confirmed device, IM, payload, audio, and subagent operations are not active direct executors; they are represented as shell capability commands in the shell schema description.
- Confirmed `novaic-cortex/tests/test_tool_schemas_limits.py` asserts shell documentation for `agentctl`, `devicectl`, `cortex payload`, media/audio, `desc`, stable Cortex paths, and sandbox backing-path warnings.
- Confirmed runtime boundary tests verify no non-final direct executors remain and no migrated schema names overlap the active direct executor surface.

## Verification

- `PYTHONPATH=novaic-agent-runtime:novaic-common:novaic-cortex:novaic-logicalfs:novaic-sandbox-sdk pytest -q novaic-agent-runtime/tests/test_tool_surface_boundary.py` -> passed, `5 passed in 0.07s`.
- `PYTHONPATH=novaic-cortex:novaic-common:novaic-logicalfs:novaic-sandbox-sdk pytest -q novaic-cortex/tests/test_tool_schemas_limits.py` -> passed, `13 passed in 0.06s`.
- Additional combined verification attempt failed before executing the Cortex tests because both `novaic-agent-runtime/tests` and `novaic-cortex/tests` define a top-level `tests` package; with runtime first on `PYTHONPATH`, pytest tried to import `tests.test_tool_schemas_limits` from the wrong package and raised `ModuleNotFoundError`.

## Known Gaps

- Cross-package pytest collection is not hygienic: multiple subprojects expose a top-level `tests` package, and Cortex tests import helpers via `from tests...`. This creates order-dependent imports when focused tests from multiple subprojects are executed in one pytest process. The schema boundary itself passed under package-local execution, but this verification gap should be closed rather than normalized.
