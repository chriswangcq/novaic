# Tighten shell capability guidance wording

## Problem Definition

Mapped shell guidance is mostly correct, but it should explicitly teach that payload reads/searches are bounded, summarize/qa use bounded redacted inputs, and device media outputs should remain artifact manifests instead of base64 in terminal/context.

## Proposed Solution

Update shell capability help/schema wording where needed and add focused tests so the contract remains visible and non-regressive.

## Acceptance Criteria

- Payload help explicitly says read/search are bounded and summarize/qa use bounded redacted inputs.
- Device artifact guidance explicitly tells agents to use blob/display manifests rather than printing base64 into terminal/chat.
- Tests assert the wording and no raw media payload contract regresses.

## Verification Plan

- Run focused shell capability blob contract tests.
- Run common/Cortex tool definition schema tests that assert shell guidance.
- Scan for stale wording that encourages raw payload/base64 dumping.

## Risks

- Overlong schema descriptions could violate tool schema length caps; keep wording concise.

## Assumptions

- Guidance can live in shell capability help plus canonical shell schema/tests without adding another user-facing tool.
