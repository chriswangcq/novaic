# Inventory generic test imports and marker files

## Problem Definition

We need a precise inventory of Cortex test imports and package markers that create `tests` namespace ambiguity before changing code.

## Proposed Solution

Search Cortex and related subprojects for generic `tests.*` imports and `tests/__init__.py` package markers, then record which files must change and which markers contribute to the mixed pytest failure.

## Acceptance Criteria

- Generic Cortex `tests.*` imports are listed with file paths.
- Cortex test package marker status is identified.
- Non-Cortex marker/import contributors to the observed mixed failure are identified.

## Verification Plan

- Run `rg` searches for `from tests` and `import tests` under relevant subprojects.
- Run `find` for `tests/__init__.py` files.
- Confirm the observed failure mode aligns with the inventory.

## Risks

- Search patterns may miss dynamic imports, so include both direct import scans and marker scans.

## Assumptions

- The failure is caused by normal Python package resolution rather than pytest plugin behavior.
