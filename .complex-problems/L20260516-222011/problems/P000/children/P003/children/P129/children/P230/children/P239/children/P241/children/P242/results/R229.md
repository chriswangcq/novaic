# Result: Cortex test namespace inventory

## Summary

Completed the read-only inventory. The mixed pytest failure is explained by empty `tests/__init__.py` package markers in both runtime and Cortex plus Cortex imports through generic `tests.*` helper paths.

## Done

- Found 40 Cortex test files importing `from tests.workspace_test_utils` or `from tests.blob_boundary_policy`.
- Found the Cortex helper files that need a package-specific namespace: `novaic-cortex/tests/workspace_test_utils.py` and `novaic-cortex/tests/blob_boundary_policy.py`.
- Confirmed `novaic-cortex/tests/__init__.py` is empty and only acts as a package marker.
- Confirmed `novaic-agent-runtime/tests/__init__.py` is also empty, which is why runtime-first `PYTHONPATH` can claim the generic `tests` package.
- Found two runtime unit tests that import `tests.threshold_policy`; those are not the direct cause of the Cortex schema test failure, but they prove the repo has more than one package relying on the same generic name.

## Verification

- `rg -n '^\s*(from|import)\s+tests(\.|\s|$)' novaic-cortex novaic-agent-runtime novaic-common novaic-logicalfs novaic-sandbox-sdk` identified the direct import inventory.
- `find ... -path '*/tests/__init__.py'` identified the package markers.
- `wc -l` confirmed both relevant `__init__.py` files are empty.
