# Check: LLM tool schema and policy boundary still has test-package hygiene gap

## Summary

The schema/policy evidence maps well to the intended tool boundary, but the problem is not fully closed because the verification harness has an order-dependent cross-package `tests` namespace conflict. A one-go audit cannot be accepted while a plausible combined verification path fails before executing the relevant Cortex test module.

## Blocking Gaps

- `R228` verifies runtime and Cortex schema tests when run separately, but the combined pytest command fails because `novaic-agent-runtime/tests` and `novaic-cortex/tests` both define the top-level package name `tests`.
- This makes package-local helper imports such as `from tests.workspace_test_utils ...` dependent on `PYTHONPATH` order. That is an explicit-boundary violation in the test layer and can mask future schema regressions.
- The original criteria require tool schema tests to verify payload tools and limits. A test command that breaks under normal cross-package focused selection leaves that criterion insufficiently proven.
