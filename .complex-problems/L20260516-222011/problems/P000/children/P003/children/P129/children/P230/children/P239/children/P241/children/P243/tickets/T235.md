# Migrate Cortex tests off generic `tests.*` imports

## Problem Definition

Cortex tests import shared helpers from the generic `tests.*` namespace, which collides with other subprojects that also expose a top-level `tests` package.

## Proposed Solution

Create a Cortex-specific test helper package, move the shared helper modules there, update all Cortex tests to import that package, and delete the stale `novaic-cortex/tests/__init__.py` marker so test collection no longer claims the generic `tests` package.

## Acceptance Criteria

- `workspace_test_utils.py` and `blob_boundary_policy.py` live under an unambiguous Cortex-specific helper namespace.
- All Cortex test imports are updated from `tests.*` to the new namespace.
- `novaic-cortex/tests/__init__.py` is deleted.
- No Cortex test file still imports generic `tests.*`.

## Verification Plan

- Run residue scans for `from tests` and `import tests` under `novaic-cortex/tests`.
- Run focused Cortex tests that use both moved helpers.
- Defer combined runtime+Cortex pytest verification to the sibling verification problem.

## Risks

- A mechanical rewrite could miss one import or create a helper package not available on `PYTHONPATH`.
- Removing the package marker changes pytest module names, so helper imports must be fully explicit.

## Assumptions

- `novaic-cortex` is on `PYTHONPATH` for Cortex tests, so a helper package placed under `novaic-cortex/` is importable.
