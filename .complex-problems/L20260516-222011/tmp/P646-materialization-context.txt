    27	# Prior to PR-46, ``handle_context_read`` merged input messages by
    28	# scanning a mutable UI receipt column — i.e. it trusted a display
    29	# flag as the source of truth for "what does this scope need to consume".

    35	# PR-46 moves the SSOT to ``scope.meta.input_message_ids`` (maintained
    36	# by PR-20's ``append_scope_input`` for both fresh-wake and buffered
    37	# paths). Runtime only consumes what the dispatch layer explicitly
    38	# wired into THIS scope.
