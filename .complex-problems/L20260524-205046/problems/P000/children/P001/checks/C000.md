## Stress Test

A failing runner returns only one failed quality gate result; tests confirm the final failed run keeps exactly that partial result and does not create a release pointer or advance branch head. This covers the most important auditability failure mode: a stopped release still has durable step evidence.

## Residual Risk

Persisting stdout/stderr can grow run JSON size or retain sensitive command output. This is non-blocking for this change because the controller already exposes the same payload in immediate responses; output redaction/truncation can be a later hardening task.
